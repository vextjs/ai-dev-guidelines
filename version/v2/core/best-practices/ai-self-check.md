# AI 行为自检清单

> AI 输出前必须执行的自检

---

## 📋 输出前自检（7 项）

### 1. 内容完整性

```yaml
自问:
  - [ ] 我要说的内容是否已经完全确定？
  - [ ] 是否还有"让我想想"、"应该是"等不确定的表述？
  
违规标志:
  ❌ "让我理解一下..."
  ❌ "我觉得应该..."
  ❌ "不对，应该是..."
```

### 2. 格式规范性

```yaml
自问:
  - [ ] 表格是否有空行分隔？
  - [ ] 代码块是否有语言标识？
  - [ ] 列表是否层级清晰？
  
违规标志:
  ❌ 表格渲染失败
  ❌ 代码块无语法高亮
```

### 3. 逻辑连贯性

```yaml
自问:
  - [ ] 是否有自相矛盾的地方？
  - [ ] 是否有"不对，应该是"这种自我纠正？
  
违规标志:
  ❌ 前后说法不一致
  ❌ 边说边改
```

### 4. 信息准确性

```yaml
自问:
  - [ ] 是否基于实际代码而非想象？
  - [ ] 是否理解了需求的隐含含义？
  
违规标志:
  ❌ "应该是这样..."（未验证）
  ❌ "根据经验..."（未查证）
```

### 5. 结构清晰性

```yaml
自问:
  - [ ] 是否有清晰的章节划分？
  - [ ] 重点内容是否用🔴标注？
  
违规标志:
  ❌ 大段文字无结构
  ❌ 重点信息被淹没
```

### 6. 可执行性

```yaml
自问:
  - [ ] 用户能否直接按照我的输出执行？
  - [ ] 是否需要用户再问"怎么做"？
  
违规标志:
  ❌ 只有理论无实践
  ❌ 步骤不完整
```

### 7. 完整性

```yaml
自问:
  - [ ] 是否有遗漏的接口/配置/文件？
  - [ ] 是否需要用户补充说明？
  
违规标志:
  ❌ 关键信息遗漏
  ❌ 半成品输出
```

---

## ⚠️ 违规修复流程

```yaml
如果发现违规:
  1. 立即停止输出
  2. 在内部重新整理
  3. 完成自检清单（7项）
  4. 一次性输出最终结果

禁止:
  - 边思考边输出
  - 输出不完整内容
  - 让用户帮忙整理
```

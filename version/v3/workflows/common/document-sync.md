# 文档同步机制

> 确保项目文档（STATUS.md / CHANGELOG.md / TASK-INDEX.md）与代码变更保持同步。
> 本文件由流程图节点 N12A 和 GIT 引用。

**版本**: v3.0.0
**最后更新**: 2026-03-10

---

## 概述

文档同步机制在每次代码变更后自动检测并生成项目文档更新预览，通过 CP6 确认后应用。

**流程位置**：N12（报告输出）→ **N12A**（判断是否涉及代码变更）→ **GIT**（执行同步+CP6）→ N13

---

## N12A 代码变更判断

### 判断逻辑

本次任务是否涉及**项目源码/配置文件**的新建、修改或删除？

| 判断结果 | 下一步 |
|---------|--------|
| **是** — 存在 `src/` `lib/` `app/` `config/` `package.json` 等文件变更 | → GIT 节点 |
| **否** — 仅分析/报告/记忆/文档/测试文件变更 | → 跳过 GIT，直接到 N13 |

### 豁免条件（即使有文件变更也跳过）

| 条件 | 说明 |
|------|------|
| 仅测试文件修改 | `test/**` · `__tests__/**` · `*.test.*` · `*.spec.*` |
| 仅文档文件修改 | `docs/**` · `*.md`（非源码目录下） |
| 仅 AI 规范文件修改 | `ai-dev-guidelines/**` |
| 用户在 CP6 选择 [S] | 明确跳过 |

---

## GIT 节点执行步骤

### 步骤总览

```text
GIT 节点:
  ① git status → 检查未提交变更
  ② 检查 CHANGELOG.md 是否需要新条目
  ③ 检查 STATUS.md 是否需要更新
  ④ 检查 TASK-INDEX.md 是否需要追加
  ⑤ 检查文档站是否需要同步（如有）
  ⑥ 生成更新预览 → CP6 确认
  ⑦ 按用户选择应用/编辑/跳过
```

## 必须同步的文件

### 1. STATUS.md（🔴 P0）

| 项 | 说明 |
|----|------|
| 位置 | 项目根目录 `STATUS.md` |
| 更新内容 | 版本号（如有变更）· 最后更新日期 · 进度统计 · 需求状态 |
| 检测方式 | 读取当前 STATUS.md → 对比变更 → 生成 diff |

**更新预览格式：**

```diff
# 项目状态

- **版本**: v1.2.0
+ **版本**: v1.3.0
- **最后更新**: 2026-02-10
+ **最后更新**: 2026-02-12

## 进度统计
- **已完成**: 15 → 16
- **进行中**: 3 → 2
```

### 2. CHANGELOG.md（🔴 P0）

| 项 | 说明 |
|----|------|
| 位置 | 项目根目录 `CHANGELOG.md` + `changelogs/vX.Y.Z.md` |
| 更新内容 | 版本概览表新增条目 + 详细变更文档 |
| 检测方式 | 收集本次任务变更 → 确定版本号 → 生成条目 |

**存储结构：**

```text
项目根目录/
├── CHANGELOG.md              # 版本概览（摘要表）
└── changelogs/
    ├── TEMPLATE.md           # 变更文档模板
    └── vX.Y.Z.md             # 版本详细变更
```

**CHANGELOG.md 概览表格式：**

```markdown
| 版本 | 日期 | 变更摘要 | 详细 |
|------|------|---------|------|
| [v1.1.0](changelogs/v1.1.0.md) | 2026-02-12 | 新功能描述 | [查看](changelogs/v1.1.0.md) |
```

**changelogs/vX.Y.Z.md 必需章节：**

1. 版本信息（版本号/日期/类型/风险/兼容性）
2. 变更摘要（一句话总结）
3. 背景说明（为什么变更）
4. 变更内容（改了什么）
5. 影响范围（影响哪些文件）
6. 详细变更清单（Added / Changed / Fixed / Removed）
7. 验证方法（如何验证）

### 3. TASK-INDEX.md（🔴 P0）

| 项 | 说明 |
|----|------|
| 位置 | `projects/<project>/TASK-INDEX.md` |
| 更新内容 | 追加本次任务记录行到对应类型表格 |
| 检测方式 | 任务完成时自动追加 |
| 参考模板 | `projects/_template/TASK-INDEX.md` |

### 4. README.md（🟡 P1 — 视情况更新）

| 更新条件 | 更新内容 |
|---------|---------|
| 新功能涉及用户可见变更 | 功能列表 |
| API 接口变更 | API 使用说明 |
| 配置项变更 | 配置说明 |
| 安装/使用方式变更 | 安装步骤 |

### 5. 文档站（🟡 P1 — 有文档站的项目）

| 项 | 说明 |
|----|------|
| 适用条件 | 项目存在 `website/` 或 `docs-site/` 目录 |
| 触发条件 | 公共 API 变更 · 配置项变更 · CLI 变更 · 中间件/插件接口变更 |
| 检查范围 | `website/docs/guide/` · `website/docs/api/` · `website/docs/examples/` |
| 检测方式 | grep 搜索相关关键词 → 列出受影响页面 |
| 豁免 | 纯内部实现重构 · 仅测试变更 · 仅性能优化（不改接口） |

**文档站同步输出格式：**

```text
📖 文档站同步检查:
  - ✅ website/docs/guide/config.md — 已同步
  - ⚠️ website/docs/api/config.md — 需更新（缺少新接口）
  - ➡️ 记录为后续待办
```

## CP6 确认

### 触发条件（满足任一）

- 代码文件被修改/新建/删除
- 配置文件被修改
- 新功能开发完成
- Bug 修复完成

### 输出格式

```text
📋 项目文档更新确认

## STATUS.md 更新
- 版本: v1.2.0 → v1.3.0
- 最后更新: [当前日期]
- 进度: +1 个已完成需求

## CHANGELOG.md 新增条目
### v1.3.0 (YYYY-MM-DD)
#### 新功能
- [变更描述]

## TASK-INDEX.md
- 追加: [任务类型] | [任务描述] | [日期] | [状态]

---

❓ 是否应用项目文档更新？

[C] Continue - 确认应用更新
[M] Modify - 手动编辑更新内容
[S] Skip - 跳过更新（需确认原因）
```

### 用户响应处理

| 选项 | 行为 |
|:----:|------|
| **[C]** | 应用所有更新 → 继续到 N13 |
| **[M]** | 等待用户编辑后应用 → 继续到 N13 |
| **[S]** | 记录跳过原因 → 继续到 N13 |

## 版本号规则（SemVer）

### 语义化版本格式

```text
MAJOR.MINOR.PATCH

MAJOR: 不兼容的 API 变更（破坏性变更）
MINOR: 新功能（向后兼容）
PATCH: Bug 修复（向后兼容）
```

### 版本号更新建议

| 任务类型 | 版本变更 | 示例 |
|---------|---------|------|
| Bug 修复 | PATCH +1 | 1.2.3 → 1.2.4 |
| 性能优化 | PATCH +1 | 1.2.3 → 1.2.4 |
| 新功能 | MINOR +1 | 1.2.3 → 1.3.0 |
| 重构 | MINOR +1 | 1.2.3 → 1.3.0 |
| 破坏性变更 | MAJOR +1 | 1.2.3 → 2.0.0 |

## Git 操作规范（约束 #14 · #21）

### Conventional Commits 格式

```text
<type>(<scope>): <description>
```

| type | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | Bug 修复 |
| `refactor` | 重构（不改功能） |
| `perf` | 性能优化 |
| `docs` | 文档 |
| `chore` | 构建/工具/依赖 |
| `test` | 测试 |

### 主动触发规则（约束 #21）

| 条件 | 行为 |
|------|------|
| 任务涉及源码变更 | AI 在任务完成时（阶段 4 之前）主动执行 `git status` |
| 有未提交变更 | 输出 commit 计划 → 等待用户确认（约束 #14） |
| 仅分析/报告 | 不触发 git 操作 |
| 仅 ai-dev-guidelines 文件 | 不触发 git 操作 |

---

## 失败处理

| 情况 | 处理方式 |
|------|---------|
| STATUS.md 不存在 | 提示创建（提供模板） |
| CHANGELOG.md 格式不匹配 | 尝试智能解析，失败则提示 |
| 版本号冲突 | 询问用户处理 |
| TASK-INDEX.md 不存在 | 提示创建（参考 `_template/`） |

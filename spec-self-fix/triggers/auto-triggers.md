# 自动触发规则

> AI 在执行任务时自动触发规范检测的规则

**版本**: v1.0  
**创建日期**: 2026-02-12

---

## 🎯 触发场景

### 场景 1: 预检查阶段

```yaml
触发时机: 工作流执行前（workflows/00-pre-check/）

检测内容:
  - 工作流文件是否完整
  - 工作流是否有验证清单
  - 模板文件是否存在

执行方式: 自动检测，发现问题立即提示

示例:
  AI 收到: "添加限流功能"
  预检查: 读取 workflows/01-requirement-dev/README.md
  检测: 文件是否包含必要章节
  如缺失: "检测到工作流文件缺少验证清单，是否补充？"
```

---

### 场景 2: 发现规范冲突

```yaml
触发时机: 执行任务时发现矛盾规范

检测内容:
  - 通用规范 vs 项目规范冲突
  - 规范前后矛盾

执行方式: 立即暂停，提示用户

示例:
  AI 读取: code-standards.md 要求驼峰命名
  AI 读取: projects/user/profile/03-代码风格.md 要求下划线
  触发: "检测到规范冲突，请确认使用哪个？"
```

---

### 场景 3: 链接失效

```yaml
触发时机: 读取规范文件时发现死链接

检测内容:
  - Markdown 链接指向不存在的文件
  - 引用的文件已删除

执行方式: 记录问题，任务完成后报告

示例:
  AI 读取: README.md
  发现: [老文档](./old.md) 指向不存在的文件
  记录: 标记为待修复
  任务后: "检测到 3 个死链接，是否修复？"
```

---

### 场景 4: 任务无法执行

```yaml
触发时机: 规范要求与实际情况不符

检测内容:
  - 规范要求生成 5 个文档，但模板只有 3 个
  - 规范要求使用工具，但工具不可用

执行方式: 立即暂停，分析根因

示例:
  规范: "生成 01-05 共 5 个文档"
  实际: templates/ 只有 3 个模板
  触发: "规范与模板不匹配，请修复规范或补充模板"
```

## 🔧 触发机制实现

### 检测点注入

```yaml
在关键流程中注入检测点:
  
  执行任务:
    → 预检查阶段
      ↓
      [检测点 1: 工作流完整性]
      ↓
    → 读取规范
      ↓
      [检测点 2: 规范冲突检测]
      [检测点 3: 链接有效性]
      ↓
    → 执行步骤
      ↓
      [检测点 4: 可执行性验证]
      ↓
    → 完成任务
      ↓
      [检测点 5: 汇总问题]
```

### 检测频率

```yaml
实时检测（每次任务）:
  - 工作流完整性
  - 规范冲突
  - 任务可执行性

延迟检测（任务后汇总）:
  - 链接失效
  - 格式问题
  - 小的优化建议

定期检测（每月）:
  - 版本过时
  - 全局冗余
  - 结构优化
```

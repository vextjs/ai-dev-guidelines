## 📦 变更内容

### 核心变更

#### 1. 新增约束 #17"报告+记忆自动输出"

```yaml
规则:
  - 每次会话必须自动写入报告文件（reports/ 下）+ 更新记忆（.ai-memory/ 下）
  - 报告和记忆的写入是 AI 的自动行为，绝不询问用户
  - 对话中只输出结论摘要 + 报告文件路径

❌ 绝对禁止:
  - "需要我写入报告吗？"
  - "要不要我保存记忆？"
  - 在对话中堆砌大段分析内容而不写报告文件
  - 会话结束时没有任何报告文件产出
```

#### 2. 报告为主体、记忆为索引架构

```yaml
之前:
  - 报告是"临时可选"文件
  - 记忆文件堆砌完整分析细节（臃肿）
  - 对话中直接输出大段分析（会话关闭后丢失）

之后:
  - 报告文件（reports/）= 完整分析内容（主体）
  - 记忆文件（.ai-memory/）= 摘要 + 报告链接（索引）
  - 对话输出 = 结论摘要 + 报告路径

关系图:
  用户提问 → AI 分析 → 写入报告文件（完整内容）
                     → 更新记忆文件（追加报告链接）
                     → 对话输出结论摘要 + 报告路径

  下次会话 → 读记忆 → 跟链接读报告 → 完整恢复上下文
```

## 💡 设计决策记录

### 为什么报告为主体、记忆为索引？

| 问题 | 之前的方案 | 新方案 |
|------|-----------|--------|
| 对话输出不可追溯 | 无解决方案 | 完整分析写入报告文件 |
| 记忆文件臃肿 | 记忆堆砌完整分析内容 | 记忆只存摘要+报告链接 |
| 跨会话上下文丢失 | 依赖记忆中的详细描述 | 读记忆→跟链接读报告 |
| token 浪费 | 记忆文件重复写分析内容 | 记忆精简，报告承载内容 |

### 为什么禁止询问用户？

1. 报告和记忆是 AI 内部工作机制，不需要用户决策
2. 询问会打断用户思路，增加不必要的交互轮次
3. 遗忘写入 = 下次会话丢失上下文 = 重复劳动
4. Zed Agent 面板不支持交互式按钮，每次确认都需要用户发消息

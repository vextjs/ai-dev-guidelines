## 📝 变更摘要

约束体系从 14 条扩展至 15 条，重新编号 #9（文件名带日期+序号，Agent 由目录隔离保证）和 #10（多 Agent 目录隔离），新增 #15（任务完成验证）。

---

## 🎯 背景说明

### 为什么需要这次变更？

v2.3.0 引入了约束 #12~#14（文件名带日期+Agent、Agent 标识必填、交叉验证），但存在以下问题：
- 约束 #9 原本要求文件名中包含 Agent 前缀，但 v1.3 的 task-memory 已改用目录隔离方案，Agent 不应出现在文件名中
- 约束 #10 原本是"主动合理性分析"，编号与功能不匹配
- AI 在声称"任务完成"时没有强制验证机制，可能遗漏输出文件或记忆更新

### 目标

- 调整约束 #9 为"文件名带日期+序号"（去掉 Agent 前缀，改由目录隔离保证）
- 调整约束 #10 为"多 Agent 目录隔离"
- 新增约束 #15"任务完成验证"，确保声称完成前必须验证输出完整性
- 总约束数从 14 条扩展至 15 条

---

## 📦 变更内容

### 核心变更

#### 1. 约束 #9 调整 — 文件名带日期+序号

```yaml
旧规则 (#9): 文件名带日期+Agent（YYYYMMDD-<agent>-<描述>.md）
新规则 (#9): 文件名带日期+序号（YYYYMMDD-NN-<描述>.md）

变更原因:
  - Agent 由目录隔离保证（clients/<agent>/），不再写入文件名
  - 新增 NN 当日序号（01、02、03...），确保同天文件有明确先后顺序

示例:
  ✅ 正确: 20260227-01-analysis-deep-review.md
  ✅ 正确: 20260227-02-fix-spec-verify.md
  ❌ 错误: 20260227-zed-copilot-analysis-xxx.md  # Agent 不应在文件名中
  ❌ 错误: 20260227-analysis-xxx.md               # 缺少序号 NN
```

#### 2. 约束 #10 调整 — 多 Agent 目录隔离

```yaml
旧规则 (#10): 主动合理性分析（现重编号为 #12）
新规则 (#10): 多 Agent 目录隔离

规则内容:
  - 记忆文件写入 clients/<当前agent>/ 目录
  - 绝不写入其他 Agent 的 clients/ 目录
  - 全局 SUMMARY.md 仅追加关键决策（标注来源 Agent）
  - 恢复任务时只扫描本 Agent 目录
```

#### 3. 新增约束 #15 — 任务完成验证

```yaml
触发条件: AI 准备声称"任务完成"之前
执行时机: 所有修改/生成操作完成后

验证清单:
  1. 输出文件存在性:
     - 所有声称创建的文件确实存在？（用工具验证）
     - 文件路径正确？（在预期目录下）
     - 文件名符合命名规范？

  2. 内容结构完整性:
     - 文档头部包含所有必填字段？（日期、Agent、状态）
     - 日期字段是真实日期，非占位符？
     - 没有中途截断的内容？

  3. 记忆文件完整性:
     - .ai-memory 中的任务记忆已更新？
     - SUMMARY.md 已追加/更新对应条目？

  4. 关联文件一致性（如修改了规范文件）:
     - 已执行交叉验证清单？
     - 所有引用处已同步更新？

🔴 未通过验证项必须修复后才能声称完成。
```

## 🎯 背景说明

### 为什么需要这次变更？

基于 v1.1.0 的对比分析，发现 ai-dev-guidelines 在以下方面需要补充：
1. 确认点数量不足（5个 vs 7+2）
2. 缺乏文档同步机制
3. 模板不够完善
4. 代码分析原则不够强制

### 现状问题

- 缺少项目文档更新确认点（CP6）
- 缺少测试/文档产物选择询问点
- 没有 STATUS/CHANGELOG 同步规范
- 缺少前端对接和 API 文档模板

### 目标

- 补充确认点至完整覆盖
- 建立文档同步机制
- 完善模板库
- 强化代码分析原则

---

## 📦 变更内容

### 核心变更

1. **确认点机制增强**
   - 新增 CP6 项目文档更新确认
   - 新增 IP1 测试产物询问点
   - 新增 IP2 文档产物询问点

2. **文档同步规范**
   - 参考 monSQLize 建立 changelogs/ 分层结构
   - 定义 STATUS.md / CHANGELOG.md 同步触发条件
   - 提供完整的模板和示例

3. **模板优化**
   - 新增前端对接文档模板
   - 新增 API 接口文档模板
   - 重构 CHANGELOG 模板为分层结构

4. **代码分析原则**
   - 在 analysis-methods.md 开头增加强制原则
   - 明确禁止行为清单

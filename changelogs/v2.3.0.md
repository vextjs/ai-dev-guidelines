## 📝 变更摘要

预检查从 3 项升级为 4 项必做（新增 Agent 标识），引入日期强制必填规范确保所有输出文件可追溯，新增 `analysis-lite.md` 深度分析精简模板，全面对齐多编辑器/多 Agent 支持策略。

---

## 🎯 背景说明

### 为什么需要这次变更？

- **多编辑器场景混乱**: 用户同时使用 Zed + WebStorm 等多个编辑器的 AI 助手，但生成的文档和记忆文件无法区分来源
- **文件缺少日期**: AI 生成的报告和记忆文件经常不带日期前缀，导致无法判断哪个是最新版本，也无法按时间排序
- **缺少深度分析模板**: 用户经常需要对项目进行深度分析/架构分析，但没有对应的精简模板

### 目标

- 预检查中强制识别并输出 Agent 标识
- 所有输出文件名必须以 `YYYYMMDD-` 开头，文档头部日期字段禁止留占位符
- 提供 `analysis-lite.md` 精简模板用于深度分析场景

---

## 📦 变更内容

### 核心变更

#### 1. 预检查新增第 4 项 — Agent 标识

```yaml
预检查输出格式:
  1. 工作区: [当前路径]
  2. 任务类型: [需求/Bug/优化/分析/...]
  3. 输出位置: [projects/<project>/...]
  4. Agent: [zed-copilot / webstorm-copilot / cursor / vscode-copilot / ...]  ← 🆕

Agent 标识表:
  | 编辑器 + AI        | 标识值              |
  |-------------------|---------------------|
  | Zed + Copilot     | zed-copilot         |
  | WebStorm + Copilot| webstorm-copilot    |
  | VS Code + Copilot | vscode-copilot      |
  | Cursor            | cursor              |
  | VS Code + Cline   | vscode-cline        |
  | Windsurf          | windsurf            |
  | 其他              | <editor>-<provider> |

检测优先级:
  1. 工具环境/系统信息推断
  2. 用户在对话中提及的编辑器名称
  3. 无法确定时使用 unknown-agent 并提示用户确认
```

#### 2. 日期强制必填规范

```yaml
文件名日期前缀:
  ✅ 正确: 20260226-01-analysis-v3-architecture-deep-review.md
  ❌ 错误: v3-architecture-deep-analysis.md       # 缺少日期前缀
  ❌ 错误: 2026-02-26-analysis-xxx.md              # 日期格式错误（有分隔符）

文档头部日期字段:
  🔴 禁止留 "YYYY-MM-DD" 占位符
  🔴 必须通过 now() 工具获取真实日期
  🔴 Agent 字段必须填写当前编辑器/AI 标识

AI 自检清单（生成文档前必须确认）:
  - [ ] 日期字段不包含 "YYYY" 占位符？
  - [ ] 日期通过 now() 工具获取？
  - [ ] Agent 字段已填写？
  - [ ] 文件名以 YYYYMMDD- 开头？
```

#### 3. 新增 analysis-lite 深度分析模板

新增 `templates/lite/analysis-lite.md`，用于深度分析/架构分析场景：

```yaml
适用场景:
  - 项目架构深度分析
  - 代码质量评估
  - 规范一致性审查
  - 技术债务盘点

模板结构:
  - 分析概览（项目、范围、方法）
  - 检查结果（按严重度分类: 🔴/🟡/💡/❌）
  - 建议与优化方向
  - 总结与优先级
```

#### 4. 多 Agent 支持对齐

在以下文件中统一对齐多编辑器/多 Agent 支持策略：
- `copilot-instructions.md` — Agent 标识入口说明
- `workflows/00-pre-check/README.md` — Agent 检测流程
- `workflows/common/task-memory.md` — 记忆文件中标注 Agent
- `QUICK-REFERENCE.md` — Agent 标识速查表

# v2.2.0 - 项目规范模块化重构 + 规范体系清理

> **发布日期**: 2026-02-24  
> **变更类型**: Enhancement + Refactor  
> **风险级别**: P1 (High) — 大量文件结构变更  
> **向后兼容**: ⚠️ 部分不兼容（旧 PROJECT.md 结构已废弃）

## 📝 变更摘要

所有项目规范统一为 `profile/` 模块化子目录结构，新增任务记忆机制（`.ai-memory/`）和临时报告规范（`reports/`），全面清理旧文件和冗余内容，copilot-instructions.md 精简为纯入口文件。

---

## 🎯 背景说明

### 为什么需要这次变更？

v2.0.0 引入了统一的 `PROJECT.md` 模板，但实践中暴露出以下问题：

- **单文件过长**: chat 项目的 PROJECT.md 超过 500 行，AI 读取耗费大量 token
- **模块耦合**: 修改代码风格需要打开包含架构约束、项目信息等无关内容的文件
- **跨会话丢失**: AI 会话断开后，新会话无法知道之前做了什么
- **报告无规范**: 各项目 `reports/` 目录文件平铺、命名混乱、无法区分新旧
- **入口文件臃肿**: copilot-instructions.md 包含大量规范细节，应精简为纯索引

### 目标

- 项目规范拆分为 `profile/` 模块化文件，按需加载
- 建立跨会话任务记忆机制（`.ai-memory/`）
- 统一临时报告的存储和命名规范
- copilot-instructions.md 精简为入口指引
- 清理所有过时文件

---

## 📦 变更内容

### 核心变更

#### 1. 项目规范模块化 — `profile/` 结构

所有项目规范从单一 `PROJECT.md` 拆分为 `profile/` 子目录下的独立模块文件：

```
projects/<project>/
├── README.md                    # 项目顶层索引
├── TASK-INDEX.md                # 任务索引
└── profile/                     # 🔴 模块化规范
    ├── README.md                # 规范入口 + AI 遵循声明
    ├── 01-项目信息.md            # 项目路径、技术栈、目录结构
    ├── 02-架构约束.md            # 强制禁止项（P0）
    ├── 03-代码风格.md            # 命名规范、缩进格式
    ├── 04-错误处理与多语言.md     # 推荐
    ├── ...                      # 更多模块（按需）
    └── CHANGELOG.md             # 版本变更日志
```

**优势**:
- AI 按需加载，修改代码风格只需读 `03-代码风格.md`（~50 行），无需读完整规范（500+ 行）
- 模块独立维护，减少合并冲突
- 新项目只需复制 `_template/profile/`，按需填充

#### 2. TASK-INDEX.md 纳入标准结构

每个项目根目录新增 `TASK-INDEX.md`，作为任务追溯索引：
- 需求开发、Bug 修复、性能优化等历史任务在此记录
- AI 在任务完成时自动更新
- 模板位于 `_template/TASK-INDEX.md`

#### 3. 任务记忆机制 — `.ai-memory/`

新增 `workflows/common/task-memory.md`，定义跨会话记忆机制：
- 存储位置: `projects/<project>/.ai-memory/`
- 核心文件: `SUMMARY.md`（任务摘要）+ `tasks/<id>.md`（单任务记忆）
- 采用"先写后干"策略，任务开始前先创建记忆，避免 token 耗尽丢失上下文
- `.ai-memory/` 加入 `.gitignore` 忽略

#### 4. 临时报告规范 — `reports/`

新增 `workflows/common/temp-reports.md`，统一报告存储和命名：
- 存储位置: `projects/<project>/reports/`
- 命名格式: `YYYYMMDD-<类型>-<简述>.md`
- 分类子目录: `diagnostics/`、`bugs/`、`requirements/`、`analysis/`、`.temp/`
- `reports/` 加入 `.gitignore` 忽略

#### 5. copilot-instructions.md 精简

入口文件从 ~300 行精简为 ~80 行，仅保留：
- 预检查格式和必做项
- 规范体系入口表（指向各子文件）
- 关键原则和禁止行为
- 所有规范细节移至对应子文件

#### 6. README.md 精简

删除与 QUICK-REFERENCE.md 重复的 ~180 行内容（预检查格式、任务映射表、确认点等），避免双份维护。

#### 7. 清理旧文件

- 删除 `_template/` 下 13 个旧模板文件（已被 `profile/` 模块化结构替代）
- 删除 `outputs/` 弃用目录（已被 `projects/` 替代）
- 删除 v1.x 过时的工作流文件
- 全部项目（chat/dev-docs/user-service）迁移到新结构

## 🔄 迁移指南

### 从 v2.0/v2.1 升级

1. **项目规范迁移**:
   ```bash
   # 在各项目目录下创建 profile/ 子目录
   mkdir -p projects/<project>/profile
   
   # 将 PROJECT.md 内容拆分到模块文件
   # 01-项目信息.md — 项目路径、技术栈、目录结构
   # 02-架构约束.md — 强制禁止项
   # 03-代码风格.md — 命名规范、缩进格式
   # profile/README.md — 规范索引 + AI 遵循声明
   
   # 删除旧 PROJECT.md
   rm projects/<project>/PROJECT.md
   ```

2. **新增任务索引**:
   ```bash
   cp projects/_template/TASK-INDEX.md projects/<project>/TASK-INDEX.md
   ```

3. **创建 .ai-memory 目录**: AI 首次执行任务时自动创建，无需手动操作

4. **创建 reports 目录**: AI 首次生成报告时自动创建，无需手动操作

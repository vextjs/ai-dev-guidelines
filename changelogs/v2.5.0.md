## 📝 变更摘要

新增规范修改交叉验证清单（约束 #14），确保修改任何规范文件后自动检查所有引用处是否需要同步更新；同时引入日期强制必填规范，禁止输出文件名和文档头部留日期占位符。

---

## 🎯 背景说明

### 为什么需要这次变更？

ai-dev-guidelines 规范体系由 6+ 个核心文件组成，同一规则在多处引用（如预检查项数同时出现在 `copilot-instructions.md`、`00-pre-check/README.md`、`QUICK-REFERENCE.md` 三个文件中）。此前多次出现"只改了一处、其他地方没同步"的不一致问题，导致 AI 执行时获得矛盾的指令。

此外，AI 生成的报告和记忆文件经常留有 `YYYY-MM-DD` 占位符，无法判断文档新旧版本。

### 现状问题

- 修改规范文件后未同步其他引用处，导致预检查项数、约束条数、文件命名格式等在不同文件间不一致
- AI 生成的文档日期字段留占位符，无法追溯创建时间
- 输出文件名缺少日期前缀，文件管理器中无法按时间排序

### 目标

- 建立强制性的交叉验证机制，修改规范文件后必须逐个检查引用处
- 所有输出文件名强制以 `YYYYMMDD-` 开头
- 文档头部日期字段禁止留占位符

## 📦 变更内容

### 核心变更

#### 1. 新增约束 #14 — 规范修改需交叉验证

修改任何规范文件后，必须逐个检查所有引用该规范的文件是否需要同步更新。

交叉检查对照表：

| 当你修改了... | 必须同步检查 | 检查什么 |
|-------------|------------|---------|
| `copilot-instructions.md` | `00-pre-check/README.md`、`QUICK-REFERENCE.md` | 预检查项数、禁止行为、格式示例 |
| `00-pre-check/README.md` | `copilot-instructions.md`、`task-memory.md` | 预检查项数、记忆写入路径、Agent 检测逻辑 |
| `task-memory.md` | `copilot-instructions.md`、`00-pre-check/README.md` | 文件名格式、Agent 方案、恢复流程 |
| `QUICK-REFERENCE.md` | `copilot-instructions.md` | 约束条数、关键原则 |
| `CONSTRAINTS.md` | `QUICK-REFERENCE.md`、`decision-tree.yaml` | 约束条数、编号、详细描述 |

检查清单（每次修改后逐条确认）：

```yaml
□ 预检查项数各处一致？
□ 文件命名格式各处一致？
□ 多 Agent 方案各处一致？
□ 约束条数与实际条目数一致？
□ 修改文件的版本号和日期已更新？
□ 所有引用链接指向的文件确实存在？
```

#### 2. 日期强制必填规范

```yaml
文件名规则:
  - 所有输出文件必须以 YYYYMMDD- 开头
  - 日期必须是真实日期，禁止留占位符
  - 日期通过 now() 工具获取

文档头部规则:
  - 创建日期字段必须替换为真实日期
  - Agent 字段必须填写当前编辑器/AI 标识
  - 项目名称必须为真实项目名
```

#### 3. QUICK-REFERENCE.md 新增交叉验证段落

在速查手册中新增"规范修改交叉验证清单"完整段落，包含：
- 必须交叉检查的文件对照表
- 每次修改后的检查清单模板
- 何时触发的说明

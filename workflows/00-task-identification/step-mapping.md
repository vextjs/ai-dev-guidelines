## 🔵 需求开发（Intent-01）

### 完整步骤序列

```
预检查 → 需求确认 → 项目分析 → 数据库分析* → 深度分析 → 
方案设计 → [CP2:方案确认] → 代码实现 → [CP3:代码确认] → 
测试生成 → 测试执行 → [CP4:测试确认] → 文档生成 → 
[CP5:文档确认] → 三轮验证 → 完成报告

*: 条件步骤，涉及数据库时执行
```

### 详细步骤

| 步骤 | 名称 | 输入 | 输出 | 确认 |
|-----|------|------|------|------|
| 1 | 预检查 | 用户请求 | 预检查结果 | - |
| 2 | 需求确认 | 用户需求 | 需求复述 | CP1 |
| 3 | 项目分析 | 项目代码 | 分析报告 | - |
| 4 | 数据库分析* | Schema | 字段清单 | - |
| 5 | 深度分析 | 相关代码 | 改造清单 | - |
| 6 | 方案设计 | 分析结果 | 技术方案 | CP2 |
| 7 | 代码实现 | 技术方案 | 代码变更 | CP3 |
| 8 | 测试生成 | 代码 | 测试用例 | - |
| 9 | 测试执行 | 测试用例 | 测试结果 | CP4 |
| 10 | 文档生成 | 全部内容 | 文档 | CP5 |
| 11 | 三轮验证 | 全部产出 | 验证报告 | - |
| 12 | 完成报告 | 验证结果 | 最终报告 | - |

```
├── README.md
```

## ⚪ 简单问答（Intent-09）

### 简化步骤序列

```
预检查（简化）→ 直接回答 → 完成
```

### 说明

```yaml
跳过步骤:
  - 项目分析
  - 方案设计
  - 代码实现
  - 测试生成
  - 文档生成

保留步骤:
  - 预检查（仅检查项 1、3、4）
  - 直接回答问题
```

## 🔴 紧急修复（Intent-13）

### 加速步骤序列

```
预检查 → 问题确认 → 快速分析 → [CP1:确认] → 
紧急修复 → [CP2:代码确认] → 快速验证 → [CP3:回滚确认] → 完成
```

### 说明

```yaml
简化步骤:
  - 跳过: 测试用例生成（使用快速验证替代）
  - 简化: 文档归档（仅关键信息）
  
强化步骤:
  - 必须: 回滚方案
  - 必须: 影响评估
```

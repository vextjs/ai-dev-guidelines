# 风险等级定义

> P0/P1/P2 三级风险的详细定义和处理规则

---

## 📊 风险等级概览

| 等级 | 名称 | 影响范围 | 确认点 | 特殊要求 |
|-----|------|---------|-------|---------|
| P0 | Critical（关键） | 可能造成数据丢失、安全问题、生产事故 | 5 个 | 必须回滚方案 |
| P1 | High（高） | 影响核心功能，需要谨慎处理 | 4 个 | 标准流程 |
| P2 | Low（低） | 低风险，影响有限 | 0-2 个 | 可简化流程 |

---

## 🔴 P0 - Critical（关键）

### 定义

```yaml
影响: 可能造成数据丢失、安全问题、生产事故
触发意图:
  - 数据库变更
  - 安全修复
  - 部署相关
  - 数据迁移
  - 紧急修复
  - 架构重构
```

### 确认点（5 个）

```yaml
CP1: 需求/问题确认
  - 确认操作范围
  - 确认影响评估

CP2: 方案确认
  - 展示完整方案
  - 🔴 展示 P0 清单（危险操作列表）
  - 用户逐项确认

CP3: 实施方案确认
  - 展示实施方案变更清单
  - 展示影响范围

CP4: 验证确认
  - 展示测试结果
  - 确认可以继续

CP5: 发布确认
  - 最终确认
  - 回滚方案确认
```

### P0 清单内容

```yaml
必须展示的内容:
  - 将删除的数据/文件
  - 将修改的核心逻辑
  - 数据库结构变更
  - 配置变更
  - 影响范围
  - 回滚方案

展示格式:
  🔴 P0 操作清单
  ════════════════════════════════════════
  
  ⚠️ 以下操作需要您逐项确认：
  
  1. [数据库] 将添加索引 users.email
     影响: 索引创建期间可能短暂阻塞写操作
     回滚: db.users.dropIndex("email_1")
     确认: [ ]
  
  2. [配置] 将修改 Redis 连接超时
     原值: 5000ms
     新值: 10000ms
     回滚: 改回 5000ms
     确认: [ ]
  
  ════════════════════════════════════════
  请确认以上所有项目: [全部确认/取消]
```

### 特殊要求

```yaml
强制要求:
  - ✅ 必须有回滚方案
  - ✅ 必须展示 P0 清单
  - ✅ 用户必须逐项确认
  - ✅ 所有操作记录到审计日志
  - ✅ 测试环境先验证

禁止行为:
  - ❌ 跳过任何确认点
  - ❌ 自动执行危险操作
  - ❌ 不提供回滚方案
```

## 🟡 P1 - High（高）

### 定义

```yaml
影响: 影响核心功能，需要谨慎处理
触发意图:
  - 需求开发
  - Bug 修复
  - 性能优化
  - API 开发
  - 配置变更
  - 依赖更新
```

### 确认点（4 个）

```yaml
CP1: 需求确认
  - 复述需求
  - 确认理解正确

CP2: 方案确认
  - 展示技术方案
  - 用户确认方向

CP3: 实施方案确认
  - 展示实施方案变更清单
  - 用户确认后才执行

CP4: 文档确认
  - 展示生成的文档
  - 用户确认完整
```

### 要求

```yaml
标准要求:
  - ✅ 完整的三轮验证
  - ✅ 测试覆盖 >= 80%
  - ✅ 文档同步更新
  - ✅ 代码审查

可选:
  - ⚠️ 回滚方案（根据影响范围）
```

---

## 🟢 P2 - Low（低）

### 定义

```yaml
影响: 低风险，影响有限
触发意图:
  - 简单问答
  - 仅测试
  - 仅文档
  - 代码审查
  - 技术调研
```

### 确认点（0-2 个）

```yaml
简单问答: 0 个确认点
  - 直接回答

仅文档: 1 个确认点
  - 文档内容确认

技术调研: 2 个确认点
  - 调研范围确认
  - 结论确认
```

### 要求

```yaml
可简化:
  - ⚠️ 可简化验证流程
  - ⚠️ 文档可选
  - ⚠️ 测试可选

仍需遵守:
  - ✅ 预检查
  - ✅ 基础代码规范
```

## 📋 风险判断流程

```
用户请求
    ↓
识别任务意图
    ↓
查询意图矩阵
    ↓
获取风险等级
    ↓
┌─────────────────────────────────────┐
│                                     │
│  P0?  ──YES──> 展示 P0 清单         │
│   │            逐项确认             │
│   │            强制回滚方案         │
│   NO                                │
│   ↓                                 │
│  P1?  ──YES──> 标准流程             │
│   │            4 个确认点           │
│   │            完整验证             │
│   NO                                │
│   ↓                                 │
│  P2   ──────> 简化流程              │
│               0-2 个确认点          │
│               基础验证              │
│                                     │
└─────────────────────────────────────┘
```

## 🔄 风险升级规则

### 自动升级

```yaml
P1 → P0 的情况:
  - 涉及数据删除
  - 涉及生产环境
  - 涉及安全相关
  - 影响支付/交易

P2 → P1 的情况:
  - 涉及核心逻辑
  - 涉及多个模块
  - 涉及配置变更
```

### 降级（需用户确认）

```yaml
P0 → P1 的情况:
  - 仅影响测试环境
  - 用户明确表示低风险
  
  ⚠️ 必须用户明确确认才能降级
```

# 性能优化流程

> **任务类型**: 性能优化  
> **使用场景**: 提升系统性能、解决性能瓶颈  
> **输出目录**: `projects/<project-name>/optimization/<opt-id>/`  

---

## 📋 流程概览

```
性能基线 → 瓶颈识别 → 优化方案 → 代码实施 → 效果验证 → 对比报告
```

---

## 🎯 执行步骤

### Step 1: 建立性能基线

**目标**: 记录优化前的性能指标

**AI 执行**:
1. 询问用户当前性能问题
2. 确定需要优化的指标（响应时间、吞吐量、资源使用等）
3. 收集当前性能数据
4. 建立性能基线

**输出**: `01-baseline.md`

**内容要点**:
- 优化目标
- 当前性能指标
- 性能问题描述
- 测试环境说明

---

### Step 2: 瓶颈识别

**目标**: 找出性能瓶颈

**AI 执行**:
1. 分析代码
2. 检查数据库查询
3. 分析网络请求
4. 识别性能瓶颈点

**输出**: `02-analysis.md`

**内容要点**:
- 瓶颈位置
- 瓶颈原因
- 优化优先级
- 预期改进空间

---

### Step 3: 优化方案设计

**目标**: 设计优化方案

**AI 执行**:
1. 提出多个优化方案
2. 对比方案优劣
3. 选择最佳方案
4. 评估实施风险

**输出**: `03-solution.md`

**模板**: `templates/core/optimization-template.md`

**内容要点**:
- 优化方案设计
- 技术实现要点
- 风险评估
- 预期效果

---

### Step 4: 代码实施

**目标**: 实施优化

**AI 执行**:
1. 修改代码
2. 优化数据库查询
3. 添加缓存（如需要）
4. 记录所有变更

**输出**: `04-implementation.md`

**模板**: `templates/core/implementation-template.md`

**内容要点**:
- 代码修改清单
- 数据库优化（索引、查询等）
- 缓存策略
- 配置变更

---

### Step 5: 效果验证

**目标**: 验证优化效果

**AI 执行**:
1. 运行性能测试
2. 收集优化后的性能指标
3. 对比优化前后数据
4. 验证功能正确性

**输出**: `05-validation.md`

**内容要点**:
- 优化后性能指标
- 性能提升百分比
- 对比图表
- 功能验证结果

---

### Step 6: 生成对比报告

**目标**: 总结优化效果

**AI 执行**:
1. 汇总优化前后数据
2. 生成对比报告
3. 总结经验教训
4. 提出后续优化建议

**输出**: 更新所有文档，确保完整性

---

## 🚨 注意事项

### 性能测试
```yaml
必须包含:
  1. 测试环境说明
  2. 测试数据量
  3. 并发数
  4. 多次测试取平均值
```

### 数据库优化
```yaml
如果优化数据库:
  1. 添加索引必须先分析执行计划
  2. 记录索引大小和维护成本
  3. 验证是否影响写入性能
```

### 缓存策略
```yaml
如果添加缓存:
  1. 说明缓存失效策略
  2. 说明缓存更新机制
  3. 评估缓存命中率
  4. 考虑缓存穿透/雪崩问题
```

### 性能指标
```yaml
常见优化指标:
  - 响应时间（P50, P95, P99）
  - QPS/TPS
  - CPU 使用率
  - 内存使用量
  - 数据库连接数
```

## ✅ 完成检查清单

在向用户报告完成前，确认：

- [ ] 所有文档已生成（01-05）
- [ ] 性能基线已建立
- [ ] 瓶颈已明确识别
- [ ] 优化方案已详细记录
- [ ] 代码变更已记录
- [ ] 性能测试已完成
- [ ] 有对比数据证明优化效果
- [ ] 脚本文件已生成（如需要）

---

## ✅ 性能优化专属验证

完成优化后，执行以下验证：

```yaml
性能优化验证清单:
  数据验证:
    ✅ 基线数据: 有优化前的性能数据
    ✅ 对比数据: 有优化后的性能数据
    ✅ 多次测试: 数据来自多次测试取平均
  
  效果验证:
    ✅ 达到目标: 性能提升达到预期
    ✅ 稳定性: 多次测试结果一致
    ✅ 无退化: 其他指标未明显下降
  
  功能验证:
    ✅ 功能正常: 优化未破坏原有功能
    ✅ 边界情况: 极端情况下仍能正常工作
```

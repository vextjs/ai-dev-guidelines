# 安全修复流程

> **任务类型**: 安全修复  
> **使用场景**: 漏洞修复、安全加固、合规整改  
> **输出目录**: `projects/<project-name>/security/<sec-id>/`  

---

## 📋 流程概览

```
漏洞识别 → 风险评估 → 修复方案 → 代码修复 → 安全测试 → 安全审计
```

---

## 🎯 执行步骤

### Step 1: 漏洞识别
- 分析安全漏洞
- 确定漏洞类型（SQL注入、XSS等）
- 评估影响范围

**输出**: `01-vulnerability.md`

### Step 2: 风险评估
- 评估安全风险等级
- 分析潜在危害
- 确定修复优先级

**输出**: 更新 `01-vulnerability.md`

### Step 3: 修复方案
- 设计修复方案
- 考虑防御深度
- 制定加固措施

**输出**: `02-solution.md`

### Step 4: 代码修复
- 实施安全修复
- 添加安全检查
- 更新安全配置

**输出**: `03-implementation.md`

### Step 5: 安全测试
- 执行安全扫描
- 进行渗透测试
- 验证修复效果

**输出**: `04-testing.md`

### Step 6: 安全审计
- 复查代码
- 更新安全文档
- 总结经验教训

**输出**: 更新所有文档

---

projects/payment-service/security/SEC-sql-injection-fix-20260211/

## 🚨 安全注意事项

### 敏感信息保护
- 不要在文档中记录真实的密钥、密码
- 使用 `<REDACTED>` 替代敏感信息
- 漏洞细节仅在内部文档中记录

### 常见安全漏洞
- SQL 注入
- XSS 跨站脚本
- CSRF 跨站请求伪造
- 文件上传漏洞
- 权限绕过

### 安全修复原则
- 输入验证
- 输出编码
- 最小权限原则
- 防御深度

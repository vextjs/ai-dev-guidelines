# 开发过程问题追踪

> **用途**：记录开发过程中发现的所有问题，供自我审查（11-self-audit）时核对状态并触发修复  
> **维护者**：用户 + AI Agent  
> **核对时机**：每次执行 `11-self-audit` 工作流时，必须读取本文件并逐条核对

---

## 📋 使用说明

1. **谁来写入**：用户在开发过程中发现的问题直接写入本文件
2. **格式要求**：按下方表格格式填写，每个问题一行
3. **自我审查时**：AI 必须逐条核对所有 `🔴 未解决` 和 `🟡 部分解决` 状态的问题
4. **修复触发**：
   - `🔴 未解决` → 分析原因 → 触发规范自我修复（self-fix）→ 更新状态
   - `🟡 部分解决` → 验证剩余部分 → 继续修复 → 更新状态
   - `✅ 已解决` → 验证修复有效性 → 确认状态
   - `❌ 不修复` → 记录不修复原因 → 跳过

---

## 🏷️ 状态定义

| 状态 | 含义 |
|------|------|
| 🔴 未解决 | 问题已发现，尚未处理 |
| 🟡 部分解决 | 已有部分修复，但未完全解决 |
| ✅ 已解决 | 问题已修复并验证通过 |
| ❌ 不修复 | 经评估决定不修复（需注明原因） |

# ai-dev-guidelines 设计理念

> 本文档阐述 ai-dev-guidelines 的设计哲学、架构决策和演进路线

**版本**: v1.0
**最后更新**: 2026-02-27

---

## 一、核心定位

### 不是什么

- ❌ 不是"提示词模板"——不是给 AI 几条规则让它照做
- ❌ 不是"编码规范"——不是 ESLint/Prettier 那样约束代码风格
- ❌ 不是"项目管理工具"——不是 Jira/Linear 的替代品

### 是什么

> **AI 工作操作系统 (AI Work OS)** — 给 AI Agent 一个完整的工程任务执行框架

就像操作系统给应用程序提供文件系统、内存管理、进程调度一样，ai-dev-guidelines 给 AI Agent 提供：

| 操作系统概念 | 对应本项目 | 说明 |
|------------|-----------|------|
| **文件系统** | 报告 + 记忆 | 结构化的持久化存储 |
| **内存管理** | 消息驱动 5 阶段 | 上下文的自动加载/保存/恢复 |
| **进程调度** | 10 种工作流 + 决策树 | 任务类型识别和执行步骤编排 |
| **权限控制** | 20 条约束 + 确认点 | AI 行为的边界和安全阀 |
| **多进程** | 多 Agent 目录隔离 | 多个 AI 实例并发协作 |
| **自我修复** | spec-self-fix 模块 | 系统自维护和健康检查 |
| **驱动程序** | 模板体系 | 标准化的输出接口 |

## 二、设计原则

### 原则 1：AI 是第一用户

所有设计决策以 "AI 能否无歧义执行" 为标准，而非"人类是否好读"。

```yaml
体现:
  - 入口文件用 ⛔/🔴 等强信号标记（AI 对 emoji 敏感度高于字体加粗）
  - 约束用编号+速查表双重索引（AI 擅长查表，不擅长搜索散文）
  - 示例用 ✅/❌ 正反对照（AI 从对比中学习效率最高）
  - 数值类规则在多处冗余引用（AI 上下文窗口有限，减少跳转）
```

### 原则 2：三层递进架构

控制 AI 的 Token 成本，按需加载信息。

```
第 1 层: 入口（39 行）— AI 必读，~300 tokens
第 2 层: 速查（494 行）— 日常执行，~4000 tokens
第 3 层: 详情（3000+ 行）— 按需加载

总计 ~4000 行，但 AI 日常只需加载 ~500 行
```

### 原则 3：有状态 > 无状态

每次对话都从零开始是巨大的浪费。消息驱动记忆让 AI 拥有"工作记忆"。

```yaml
无状态模式（其他项目）:
  用户: "继续上次的工作"
  AI: "抱歉，我不记得上次做了什么"

有状态模式（本项目）:
  用户: "继续"
  AI: |-
    📋 预检查:
    5. 上次记忆: 20260227.md §会话11 (✅ 完成)
    → 读取记忆 → 读取报告 → 完整恢复上下文
```

### 原则 4：防复现 > 事后修复

同类 Bug 反复出现的根因是"没有机制防止它再次发生"。

```yaml
防复现三件套:
  1. META.yaml — 单一真相源（版本号/约束条数只在此定义）
  2. bump-version.js — 自动同步（修改 META → 自动更新所有引用文件）
  3. 引用清单 — 显式列出所有引用位置（版本号 8 文件 + 约束条数 11 文件）

结果: 版本号遗漏从 BUG-023~031（9 个）→ 之后再未发生
```

### 原则 5：约束是护栏而非枷锁

20 条约束不是限制 AI 的能力，而是防止 AI 犯人类会犯的错。

```yaml
约束设计哲学:
  #1~#3: "需确认" — 防止 AI 未经授权执行破坏性操作
  #5~#6: "代码质量" — 防止 AI 生成低质量代码
  #9~#10: "命名和隔离" — 防止多 Agent 混乱
  #14~#15: "交叉验证" — 防止改了一处忘了其他处
  #16~#17: "确认+自动" — 平衡安全性和效率
  #20: "拆分" — 防止 AI 生成巨型文件
```

## 四、演进路线

### 已完成阶段

| 阶段 | 版本 | 里程碑 |
|:----:|:----:|--------|
| 1 | v1.0~v1.3 | 基础框架：工作流 + 模板 + 约束 |
| 2 | v2.0~v2.5 | 多 Agent 支持 + 预检查 + Agent 标识 |
| 3 | v2.6~v2.8 | 记忆机制成熟 + 报告自动输出 + 防复现 |
| 4 | v2.9~v2.11 | 消息驱动记忆 + 自修复机制 + 深度审计 |

### 当前状态 (v2.11.0)

```
规范文件: ~4000 行    工作流: 10 种    模板: 29 个
约束: 20 条          工具: 7 个       项目: 5+ 个
```

### 未来方向

#### 阶段 5: 独立 Agent 化 (v3.x)

将 ai-dev-guidelines 从"规范文件"升级为"可执行 Agent"：

```yaml
现状 (v2.x — 规范驱动):
  - AI 读取规范文件 → 按规范执行
  - 依赖 AI 自觉遵守规则
  - 不同编辑器/模型的执行力差异大

未来 (v3.x — Agent 驱动):
  - 独立 Agent 进程，通过 MCP/API 与编辑器 AI 交互
  - 规范内置为 Agent 的核心逻辑，不再依赖"读文件"
  - 统一的执行力，不受编辑器/模型差异影响
```

**Agent 架构设想**:

```
┌─────────────────────────────────────────┐
│           ai-dev-agent (v3.x)            │
│                                          │
│  ┌──────────┐  ┌──────────┐             │
│  │ 规范引擎  │  │ 记忆引擎  │             │
│  │ (约束检查) │  │ (持久化)  │             │
│  └──────────┘  └──────────┘             │
│                                          │
│  ┌──────────┐  ┌──────────┐             │
│  │ 工作流引擎 │  │ 模板引擎  │             │
│  │ (步骤编排) │  │ (文档生成) │             │
│  └──────────┘  └──────────┘             │
│                                          │
│  ┌──────────────────────────┐           │
│  │     MCP Server / API      │           │
│  └──────────────────────────┘           │
└────────────┬────────────────────────────┘
             │ MCP Protocol / REST API
             ▼
┌────────────────────────────────────────┐
│  编辑器 AI (Copilot / Claude / GPT)     │
│  → 只负责代码生成和对话                  │
│  → 规范/记忆/工作流由 Agent 管理         │
└────────────────────────────────────────┘
```

**Agent 化的优势**:

| 维度 | 规范驱动 (v2.x) | Agent 驱动 (v3.x) |
|------|:---------------:|:-----------------:|
| 执行一致性 | 🟡 依赖模型能力 | ✅ Agent 统一执行 |
| 约束强制性 | 🟡 靠 AI 自觉 | ✅ 代码级强制 |
| 记忆可靠性 | 🟡 AI 可能遗漏 | ✅ Agent 自动管理 |
| Token 成本 | 🟡 需加载规范文件 | ✅ Agent 内置，零 Token |
| 跨编辑器 | ✅ 已支持 | ✅ MCP 统一接口 |
| 可测试性 | 🟡 人工审查 | ✅ 自动化测试 |

#### 阶段 6: 生态扩展 (v4.x)

```yaml
可能的扩展方向:
  - npm 包化: `npm install ai-dev-guidelines` → 任何项目即用
  - 插件系统: 第三方可扩展工作流/模板/约束
  - 多语言支持: 规范文件支持英文/日文等
  - 可视化仪表盘: 查看所有项目的 AI 活动/报告/记忆
  - CI/CD 集成: 在 PR 中自动执行规范检查
```

## 五、与市面项目的本质区别

```
市面项目:  提示词 → AI → 代码
           (无状态，无流程，无记忆)

本项目:    规范体系 → AI → 完整工程交付
           (有状态，有流程，有记忆，有自修复)

未来:      Agent → 编辑器AI → 完整工程交付
           (有状态，有流程，有记忆，有自修复，代码级强制)
```

## 一、跨会话恢复（"继续"指令）

当你说"继续"、"接着"、"恢复"时，AI 会执行以下恢复流程：

```
Step 1: 确定 Agent 标识（当前编辑器 + AI 组合）

Step 2: 扫描当前 Agent 的 .ai-memory
  → 路径: ai-dev-guidelines/projects/*/.ai-memory/clients/<agent>/SUMMARY.md
  → 目标: 找到状态为 🔄 的未完成任务
  → 兜底: 如果 clients/<agent>/ 不存在，检查全局 SUMMARY.md（向后兼容旧格式）

Step 3: 读取最新记忆
  → 找到最新日期文件（YYYYMMDD.md）
  → 读取末尾最新会话的 📨 对话记录 → 快速回忆对话脉络
  → 跟链接读报告文件 → 恢复详细上下文

Step 4: 输出恢复摘要
  → 上次做了什么 + 做到哪里 + 接下来该做什么
  → 等你确认后继续
```

### 边界情况处理

| 情况 | AI 行为 |
|------|--------|
| 找到 0 个未完成任务 | 检查其他 Agent 是否有未完成任务，提示你确认 |
| 找到 1 个未完成任务 | 直接恢复 |
| 找到多个未完成任务 | 列出清单，让你选择恢复哪个 |
| 记忆文件标记 ✅ 但你说"继续" | 理解为你对结果不满意，想要调整 |
| 其他 Agent 的未完成任务 | 不自动恢复，提示你确认 |
| 记忆文件存在但内容很少（只有初始记忆） | 说明 Token 在任务刚开始时就耗尽了，AI 从头开始但保留已有信息 |
| 找不到任何 .ai-memory 目录 | 该项目从未使用过任务记忆，AI 提示你这是全新开始 |
| 用户指定了具体项目（如"继续 vext 的任务"） | 只扫描该项目的 .ai-memory |

### Token 耗尽恢复

当 AI 在执行过程中 Token 耗尽，下次你说"继续"时：

1. AI 读取记忆文件，找到最后一次更新的会话
2. 从 📨 对话记录中恢复对话脉络
3. 从关联报告文件中恢复详细上下文
4. 输出恢复摘要，列出已完成和待完成的工作
5. 等你确认后从断点继续

> **为什么能恢复？** AI 遵循"先写后干"原则 — 开始执行前先创建初始记忆，过程中在每个消息事件增量更新。所以即使 Token 中途耗尽，至少有一份快照记录。

## 二、快速模式 vs 完整模式

### 模式对比

| 维度 | 快速模式（默认） | 完整模式 |
|------|----------------|---------|
| **适用** | 简单需求、< 5 个文件 | 复杂需求、核心功能 |
| **流程** | 5 阶段 | 7 阶段（+文档归档 +验证检查） |
| **模板** | `core/templates/lite/` | `core/templates/core/` |
| **预计时间** | 15-30 分钟 | 45-90 分钟 |
| **触发词** | "快速"、"简单"（或默认） | "完整"、"详细"、"需要文档" |
| **输出产物** | 代码实现 + 实施方案 | 完整文档套件 + 代码实现 |

### 自动判断

AI 根据以下条件自动选择模式：

| 条件 | 快速模式 | 完整模式 |
|-----|---------|---------|
| 需求描述 | < 100 字 | >= 100 字 |
| 涉及文件 | < 5 个 | >= 5 个 |
| 复杂度 | 简单改动 | 多模块协作 |
| 用户关键词 | "快速"、"简单" | "完整"、"详细" |

### 手动切换

你可以随时说：

- "切换到完整模式" → 使用完整流程
- "只要代码，不要文档" → 跳过文档生成
- "需要完整文档" → 补充生成文档

## 三、你会看到的输出格式

### 预检查输出

每次会话开始时你会看到：

```
📋 预检查:
1. 工作区: E:\Worker\chat-service
2. 任务类型: Bug 修复
3. 输出位置:
   报告: ai-dev-guidelines/projects/chat/reports/bugs/zed-copilot/20260228/
   产物: ai-dev-guidelines/projects/chat/bugs/消息丢失问题/
4. Agent: zed-copilot
5. 上次记忆: projects/chat/.ai-memory/clients/zed-copilot/tasks/20260227.md §会话03 (✅ 完成)
6. 📝 记忆已创建: .ai-memory/clients/zed-copilot/tasks/20260228.md §会话01 (🔄)
```

### 继续任务的预检查输出

当你说"继续"时，预检查格式略有不同：

```
📋 预检查（继续任务）:
1. 工作区: E:\Worker\chat-service
2. 任务类型: 继续任务
3. Agent: zed-copilot
4. 恢复来源: projects/chat/.ai-memory/clients/zed-copilot/tasks/20260228.md §会话02 (🔄)
5. 📝 记忆已更新: §会话03 (🔄)

📋 恢复摘要:
- 上次任务: Bug 修复 — 消息丢失问题
- 做到哪里: CP2 已确认，实施方案已生成，待 CP3 确认
- 接下来: 请确认实施方案后继续执行
```

### 确认点输出

每个确认点你会看到类似：

```
📋 需求理解:
- 核心需求: [一句话描述]
- 涉及模块: [模块1, 模块2]
- 关键改动:
  1. [改动点1]
  2. [改动点2]

请确认以上理解是否正确？[确认/修改]
```

### 进度报告

执行阶段你会看到：

```
🔄 实施进度: [2/5]
✅ 已完成: model/User.ts, service/UserService.ts
🔄 正在处理: controller/UserController.ts
⏳ 待处理: config/auth.ts, test/user.test.ts
```

### 完成报告

任务结束时你会看到：

```
✅ 任务完成

## 修改汇总
| 文件 | 操作 | 说明 |
|-----|------|------|
| model/User.ts | 新增 | 用户积分模型 |
| service/UserService.ts | 修改 | 新增获取积分方法 |

## 验证结果
- [x] 功能验证通过
- [x] 编译/类型检查通过

📄 完整报告: reports/requirements/zed-copilot/20260228/01-req-用户积分接口.md
```

### 合理性评估输出（仅在有建议时）

当 AI 认为有更好的方案时：

```
💡 合理性评估:
你的需求是 [复述需求]，但我注意到：

1. [建议1]: [原因]
2. [建议2]: [原因]

建议: [具体建议]

是否按建议执行？还是按原需求继续？
```

# 开发执行约束

> 本规范定义 AI 在 dev-docs 框架下执行开发任务时的约束条件

---

## 🎯 核心约束（4 条）

### 1. 删除操作需确认

删除现有代码/文件/配置前必须用户确认：

```yaml
需确认删除:
  - 文件删除
  - 函数/方法删除
  - 配置项删除
  - 路由删除

操作方式:
  AI: "需要删除 userService.js，是否确认？[Y/N]"
  用户: "Y"
  AI: "已删除"
```

### 2. Git 操作需确认

执行 git commit/push 前必须用户确认：

```yaml
需确认操作:
  - git commit
  - git push
  - git merge

操作方式:
  AI: "代码完成，是否提交？
        分支: feature/xxx
        变更: 3 个文件
        [Y/N]"
```

### 3. 方案选择需确认

多方案时必须用户决策：

```yaml
需确认场景:
  - 技术方案选择
  - 依赖版本选择
  - 架构设计决策

操作方式:
  AI: "[A] 方案 A - 优点/缺点
        [B] 方案 B - 优点/缺点
        建议 A，请选择 [A/B]"
```

### 4. 多任务需拆分

同时收到多个任务时建议分开执行：

```yaml
检测到多任务:
  AI: "检测到 2 个任务：
        1. 添加限流功能（需求开发）
        2. 优化查询性能（性能优化）
        建议分开执行，先做哪个？[1/2]"
```

## 📝 代码输出约束（3 条）

### 5. 必须包含错误处理

```typescript
// ✅ 正确
async function getUser(id: string) {
  try {
    if (!id) throw new Error('ID required');
    return await db.users.findById(id);
  } catch (error) {
    logger.error('getUser failed', { id, error });
    throw error;
  }
}

// ❌ 错误 - 无错误处理
async function getUser(id: string) {
  return await db.users.findById(id);
}
```

### 6. 禁止硬编码敏感信息

```typescript
// ✅ 正确
const API_KEY = process.env.API_KEY;

// ❌ 错误
const API_KEY = 'sk-abc123...';
```

### 7. 使用结构化输出

输出使用表格/列表/代码块，避免大段叙述：

```markdown
✅ 正确:
| 步骤 | 操作 | 文件 |
|-----|------|------|
| 1 | 添加方法 | userService.ts |

❌ 错误:
"这个功能需要修改用户服务，首先我们要添加一个新的方法..."
```

## ⚠️ 失败处理

任何失败必须明确报告，不能静默跳过：

```yaml
报告格式:
  AI: "❌ [失败类型]
        错误: [错误信息]
        建议:
        1. [建议1]
        2. [建议2]"
```

## 🔴 报告验证约束（新增）

生成分析报告或问题清单时，**必须逐项验证**：

```yaml
禁止:
  - 基于推断列出问题（未实际查看文件）
  - 复制之前报告内容而不重新验证
  - 假设某内容应该存在但未确认

必须:
  - 每个问题都读取实际文件验证
  - 区分: ✅已验证存在 / ❌验证后不存在
  - 只报告验证确认的问题

详见: best-practices/AI-CODE-STANDARDS.md#报告必须验证
```

# 问题分析: Session 丢失

> **Bug ID**: BUG-001  
> **项目**: user-service  
> **日期**: 2026-02-12  
> **严重程度**: P1

---

## 📋 问题描述

### 用户反馈
用户登录后 session 经常丢失，导致需要重新登录，影响用户体验。

### 复现步骤
1. 用户正常登录系统
2. 使用 5-10 分钟后
3. 刷新页面或进行操作
4. 系统提示需要重新登录

### 影响范围
- 影响用户: 所有登录用户
- 影响功能: 所有需要认证的功能
- 发生频率: 约 30% 的用户报告此问题

---

## 🔍 根因分析

### 排查过程

1. **检查 Session 配置**
   ```javascript
   // config/session.js
   {
     secret: process.env.SESSION_SECRET,
     resave: false,
     saveUninitialized: false,
     cookie: {
       maxAge: 1000 * 60 * 30  // 30 分钟
     }
   }
   ```
   - ❌ 发现问题: `maxAge` 设置为 30 分钟，过短

2. **检查 Redis 连接**
   ```bash
   # Redis 连接正常
   PING -> PONG
   ```
   - ✅ Redis 连接正常

3. **检查服务器时间同步**
   ```bash
   # 服务器时间
   date -> 2026-02-12 10:30:00
   ```
   - ✅ 时间同步正常

### 根本原因

**Session 过期时间配置过短**（30分钟），且没有实现自动续期机制。

用户在活动时 session 不会自动延长，导致 30 分钟后强制过期。

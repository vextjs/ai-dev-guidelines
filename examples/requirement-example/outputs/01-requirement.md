## 📋 需求概述

### 背景
user-service 是一个用户管理微服务，随着用户量增长，需要添加 API 限流功能，防止滥用和保护服务稳定性。

### 需求描述
在 user-service 项目添加限流功能，每个用户每分钟最多 100 个请求。

### 核心目标
1. 防止 API 滥用
2. 保护服务稳定性
3. 合理分配资源

---

## 📊 需求详情

### 功能需求

| # | 需求项 | 优先级 | 描述 |
|---|--------|--------|------|
| 1 | 用户级限流 | P0 | 每个用户每分钟最多 100 次请求 |
| 2 | IP 级限流 | P1 | 未登录用户按 IP 限流 |
| 3 | 限流配置化 | P0 | 支持不同环境不同配置 |
| 4 | 友好响应 | P1 | 超限时返回 429 和重试时间 |

### 非功能需求

| # | 需求项 | 描述 |
|---|--------|------|
| 1 | 性能 | 限流检查延迟 < 5ms |
| 2 | 可靠性 | Redis/MongoDB 故障时降级放行 |
| 3 | 可观测 | 记录限流触发日志 |

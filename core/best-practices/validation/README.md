# 验证机制

> AI 输出质量保证的核心机制

---

## 🎯 模块定位

| 项目 | 说明 |
|-----|------|
| **定位** | AI 生成内容的质量验证体系 |
| **核心功能** | 三轮验证 + 自动修复 |
| **使用场景** | 方案设计后、代码生成后、提交前 |
| **与其他模块关系** | 被所有工作流（01-08）在验证阶段调用 |

---

## 📋 概述

验证机制是确保 AI 生成的方案和代码质量的关键环节。本目录包含完整的三轮验证规范。

## 🎯 验证时机

| 时机 | 验证内容 | 验证轮次 | 强制输出进度 |
|-----|---------|---------|------------|
| 方案设计后 | 验证方案逻辑和完整性 | 三轮全部 | ✅ 必须 |
| 代码生成后 | 验证代码质量和正确性 | 三轮全部 | ✅ 必须 |
| 测试生成后 | 验证测试覆盖和有效性 | 第一轮 + 第二轮 | - |
| 文档生成后 | 验证文档完整性 | 第三轮 | - |
| 提交前 | 最终完整性验证 | 三轮全部 | - |

## 📊 三轮验证概览

### 第一轮：逻辑验证（6 项）

| # | 验证项 | 检查内容 |
|---|-------|---------|
| 1 | 需求覆盖 | 所有需求点是否有对应实现 |
| 2 | 边界处理 | null/undefined/空值/极值是否处理 |
| 3 | 错误处理 | async 函数是否有 try-catch |
| 4 | 逻辑完整 | 条件分支是否完整（if-else） |
| 5 | 流程正确 | 业务流程是否符合需求 |
| 6 | 返回值 | 返回值类型和结构是否正确 |

### 第二轮：技术验证（7 项）

| # | 验证项 | 检查内容 |
|---|-------|---------|
| 7 | 代码规范 | 是否符合项目代码规范 |
| 8 | 性能考量 | 是否有性能问题（N+1、大循环等） |
| 9 | 安全检测 | 是否有安全风险（注入、XSS等） |
| 10 | 并发安全 | 是否考虑并发问题 |
| 11 | 依赖声明 | 新依赖是否正确声明 |
| 12 | 测试覆盖 | 是否有足够的测试覆盖 |
| 13 | 日志完整 | 关键操作是否有日志 |

### 第三轮：完整性验证（10 项）

| # | 验证项 | 检查内容 |
|---|-------|---------|
| 14 | 文件完整 | 所有必需文件是否生成 |
| 15 | 配置完整 | 配置项是否完整 |
| 16 | 禁止删除 | 是否误删除了现有代码 |
| 17 | 文档关联 | 相关文档是否更新 |
| 18 | 环境变量 | 新环境变量是否在 .env.example 中 |
| 19 | 版本一致 | 版本号是否一致 |
| 20 | 依赖锁定 | package-lock.json 是否更新 |
| 21 | 类型定义 | TypeScript 类型是否完整 |
| 22 | 导入导出 | import/export 是否正确 |
| 23 | 回滚方案 | 是否有回滚方案（P0 操作） |

## ✅ 验证输出格式

### 验证进度输出（方案/代码验证必须）

```text
🔍 [验证中] 三轮验证

[1/3] 逻辑验证中...
  ✅ 需求覆盖检查完成
  ✅ 边界处理检查完成
  ✅ 错误处理检查完成（自动修复 2 处）
  ✅ 逻辑完整检查完成
  ✅ 流程正确检查完成
  ✅ 返回值检查完成

[2/3] 技术验证中...
  ✅ 代码规范检查完成
  ⚠️ 性能考量检查发现 1 个问题（已自动优化）
  ✅ 安全检测检查完成
  ✅ 并发安全检查完成
  ✅ 依赖声明检查完成
  ✅ 测试覆盖检查完成
  ✅ 日志完整检查完成

[3/3] 完整性验证中...
  ✅ 文件完整检查完成
  ✅ 配置完整检查完成
  ✅ 禁止删除检查完成
  ✅ 文档关联检查完成
  ...

✅ 三轮验证全部通过，继续下一步
```

### 验证发现问题

```text
🔍 [验证中] 三轮验证

[1/3] 逻辑验证中...
  ❌ 需求覆盖检查发现问题:
     - 缺少: 用户认证功能未实现
     - 建议: 补充认证逻辑
  
[问题汇总]
发现 1 个问题需要人工处理

❓ 是否修复后继续？ [Y/N]
```

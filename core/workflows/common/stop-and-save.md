# Stop & Save 机制

> ⚠️ **已弃用（Deprecated since v2.6.0）**
> 
> 本机制已被 [`task-memory.md`](./task-memory.md) 的 **"先写后干"（Write-First）策略** 取代。
> 新机制的优势：
> - **自动触发**：无需用户主动选择，任务开始时自动创建记忆
> - **增量更新**：关键节点自动快照，token 耗尽也不丢失上下文
> - **多 Agent 支持**：通过 `clients/<agent>/` 目录隔离，支持多编辑器并行
> - **统一存储**：记忆统一存放在 `.ai-memory/clients/<agent>/tasks/`
>
> **迁移指南**：
> - 原 `projects/<project>/tasks/<id>/PROGRESS.md` → 改用 `.ai-memory/clients/<agent>/tasks/<YYYYMMDD>-<NN>-<TYPE>-<id>.md`
> - 原"用户主动触发保存" → 改为"先写后干"自动记忆
> - 原"读取 PROGRESS.md 恢复" → 改为读取 `clients/<agent>/SUMMARY.md` + 扫描 🔄 状态任务
>
> 以下为历史文档，仅供参考。

---

## 📋 概述

Stop & Save 是一种允许用户在任务执行过程中暂停并保存当前进度的机制。用户可以在任意确认点选择此选项，稍后继续执行。

> ⛔ **请使用新机制**：详见 [`task-memory.md §先写后干`](./task-memory.md)

---

## 🎯 触发方式

### 在确认点选择

```yaml
任何确认点都支持 Stop & Save:

❓ 方案是否确认？

[✅ 确认] [✏️ 修改] [❌ 取消] [📦 Stop & Save]
```

### 直接请求

```yaml
用户随时可以说:
  - "保存进度"
  - "暂停"
  - "stop and save"
  - "先到这里"
```

## 📦 保存内容

### 进度文件结构

```yaml
保存位置: projects/<project>/tasks/<task-id>/PROGRESS.md

保存内容:
  1. 任务基本信息
     - 任务 ID
     - 任务类型
     - 开始时间
     - 当前步骤
  
  2. 已完成的步骤
     - 预检查结果
     - 需求确认结果
     - 方案设计（如已完成）
     - 代码变更（如已完成）
  
  3. 未完成的步骤
     - 待执行步骤清单
     - 下一步是什么
  
  4. 上下文信息
     - 用户确认的决策
     - 关键发现
     - 注意事项
```

### 进度文件示例

```markdown
# 任务进度文件

## 基本信息
- **任务 ID**: REQ-user-20260212-rate-limit
- **任务类型**: 需求开发
- **开始时间**: 2026-02-12 09:00
- **当前步骤**: CP2 - 方案确认
- **进度**: 4/10 步骤

## 已完成步骤

### ✅ 预检查
- 工作区: E:\Worker\user-service
- 项目规范: 已加载
- 风险等级: P1

### ✅ 需求确认
核心目标: 在 user 项目集成限流功能
用户已确认: 2026-02-12 09:15

### ✅ 项目分析
- 技术栈: Express + TypeScript + MongoDB
- 入口文件: src/app.ts
- 中间件目录: src/middleware/

### ✅ 方案设计
方案文件: 02-technical.md
状态: 等待用户确认

## 未完成步骤
- [ ] 方案确认（当前）
- [ ] 代码实现
- [ ] 测试生成
- [ ] 测试执行
- [ ] 文档生成
- [ ] 完成报告

## 下一步
等待用户确认方案后，执行代码实现

## 关键决策记录
1. 使用 flex-rate-limit 库
2. 全局限流 + API 级别限流
3. 配置存储在 config/rate-limit.config.ts

---
**保存时间**: 2026-02-12 09:30
```

## 🔄 恢复执行

### 恢复方式

```yaml
用户说:
  - "继续上次的任务"
  - "恢复 REQ-user-20260212-rate-limit"
  - "继续"

AI 响应:
  1. 读取进度文件
  2. 展示上次进度
  3. 询问是否继续
```

### 恢复流程

```yaml
步骤 1 - 读取进度:
  - 查找 PROGRESS.md 文件
  - 解析已完成和未完成步骤

步骤 2 - 展示进度:
  📋 任务恢复
  
  任务: REQ-user-20260212-rate-limit
  进度: 4/10 步骤
  
  已完成:
  ✅ 预检查
  ✅ 需求确认
  ✅ 项目分析
  ✅ 方案设计
  
  当前步骤: CP2 - 方案确认
  
  是否继续？ [✅ 继续] [🔄 重新开始]

步骤 3 - 继续执行:
  - 从断点继续
  - 保持之前的上下文
```

## ⚠️ 注意事项

### 进度文件有效期

```yaml
建议:
  - 进度文件有效期 7 天
  - 超过 7 天建议重新开始
  - 项目代码变化较大时建议重新开始
```

### 无法恢复的情况

```yaml
无法恢复:
  - 进度文件被删除
  - 项目结构大幅变化
  - 代码冲突严重

处理:
  - 提示用户重新开始
  - 保留已生成的文档供参考
```

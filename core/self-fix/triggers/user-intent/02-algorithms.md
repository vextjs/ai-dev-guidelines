## 🔧 检测算法实现

### 算法 1: 关键词匹配

```yaml
高优先级关键词（直接触发）:
  - "明明"、"不是"、"不对"、"错了"
  - "规范"、"要求"、"应该"
  - "为什么不"、"怎么不"

中优先级关键词（综合判断）:
  - "理解"、"意思"、"意图"
  - "重复"、"再说一次"

🆕 v2.0 升级触发关键词（直接触发模式 5）:
  - "又出现了"、"每次都"、"总是"、"再次"
  - "重复同样的问题"、"上次修过了"
  - "怎么又"、"还是一样"
  - "没有真正修复"、"修了等于没修"
  - "后续都会重复复现"

🆕 v2.1 规范补充触发关键词（直接触发模式 6）:
  - "没有定义"、"缺少规范"、"缺失"、"规范中没有"
  - "不合理"、"太严格"、"太宽松"、"不够清晰"
  - "需要新增"、"应该加上"、"建议增加"、"需要定义"
  - "定义不清楚"、"歧义"、"容易误解"
  - "如果没有需要定义清楚"

触发逻辑:
  if 包含高优先级关键词:
    立即触发检测（模式 1~4）
  elif 包含升级触发关键词:
    立即触发升级分析（模式 5 — 🆕 v2.0）
  elif 包含规范补充触发关键词:
    立即触发规范审查（模式 6 — 🆕 v2.1）
  elif 包含中优先级关键词 + 负面语气:
    触发检测
```

### 算法 2: 语气分析

```yaml
负面语气特征:
  - 问号多（？？？）
  - 感叹号（！）
  - 重复字符（我说的是xxx，xxx！）
  - 短句（语气急促）

🆕 v2.0 升级语气特征:
  - 反问句（"不是修过了吗？"）
  - 无奈语气（"每次都..."）
  - 质疑机制本身（"这样修有用吗？"）

🆕 v2.1 建设性反馈语气特征:
  - 建议语气（"应该..."、"需要..."、"建议..."）
  - 条件句（"如果没有...需要..."）
  - 分析语气（"对AI不友好"、"这样不合理"）

综合判断:
  负面语气 + 关键词 → 触发检测
  升级语气 + 升级关键词 → 触发模式 5 升级分析
  建设性语气 + 规范补充关键词 → 触发模式 6 规范审查
```

### 算法 3: 对话轮次分析

```yaml
检测重复:
  1. 提取最近 5 轮对话的用户输入
  2. 提取核心关键词（去除停用词）
  3. 计算关键词重叠度
  4. 如果连续 3 轮重叠度 > 60% → 触发检测

示例:
  轮次 1: "添加限流功能"  关键词: [添加, 限流, 功能]
  轮次 2: "我说的是限流"  关键词: [说, 限流]
  轮次 3: "限流！"        关键词: [限流]
  
  重叠: 3/3 轮都有"限流" → 触发
```

### 算法 4: 修复历史匹配（🆕 v2.0）

```yaml
检测反复问题:
  1. 扫描 core/self-fix/records/ 目录下所有修复记录
  2. 提取每条记录的问题类型标签
  3. 与当前用户反馈的问题类型进行匹配
  4. 统计同类问题出现次数
  5. 次数 ≥ 2 → 触发升级分析

示例:
  records/2026-02-27-version-sync-gap-and-stage0-timing.md
    问题类型: "版本号全量同步缺失"
    出现次数: 第 3 次（BUG-023~031）
  
  用户说: "版本号又不一致了"
  匹配: 问题类型 = "版本号不一致"
  历史: 已出现 3 次
  结果: 🔴 触发升级分析（模式 5）

检测知识库断层:
  1. 读取最新修复记录中的"修改文件清单"
  2. 检查修改是否覆盖了以下全部层面:
     - 执行流程入口（QUICK-REFERENCE/CONSTRAINTS/task-memory/00-pre-check）
     - spec-self-fix 知识库（detection/triggers/repair）
     - 修复记录（records/summary.md）
  3. 缺少任一层面 → 标记为"断层"
  4. 如果用户指出修复未生效 → 优先检查断层

示例:
  修复记录: 2026-02-27-version-sync-gap-and-stage0-timing.md
  修改文件清单:
    ✅ QUICK-REFERENCE.md
    ✅ CONSTRAINTS.md
    ✅ task-memory.md
    ✅ 00-pre-check/README.md
    ❌ detection/conflict-detection.md  ← 未更新
    ❌ triggers/auto-triggers.md        ← 未更新
    ❌ repair/repair-patterns.md        ← 未更新
  断层: 知识库未同步 → 用户反馈"修复后还会复现"时优先修复此断层
```

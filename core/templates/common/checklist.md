# 通用检查清单

## 文档完整性检查

```markdown
### 文档生成检查清单

- [ ] 文档头部信息完整（项目、任务ID、日期等）
- [ ] 所有必填章节已填写
- [ ] 代码示例已测试验证
- [ ] 图表清晰可读
- [ ] 链接有效可访问
- [ ] 文档尾部变更历史已更新
```

## 代码质量检查

```markdown
### 代码变更检查清单

- [ ] 代码符合项目规范
- [ ] 已添加必要的注释
- [ ] 已编写单元测试
- [ ] 测试覆盖率达标
- [ ] 无 ESLint/TSLint 错误
- [ ] 已通过代码审查
```

## 部署前检查

```markdown
### 部署前检查清单

- [ ] 测试环境验证通过
- [ ] 配置文件已更新
- [ ] 数据库迁移脚本已准备
- [ ] 回滚方案已制定
- [ ] 监控告警已配置
- [ ] 部署文档已更新
```

## 任务完成检查

```markdown
### 任务完成检查清单

- [ ] 所有文档已生成
- [ ] 代码已提交
- [ ] 脚本已编写（如需要）
- [ ] 测试已通过
- [ ] 文档已归档到正确目录
- [ ] 用户已确认完成
```

---

## 使用方法

在文档中引用检查清单：

```markdown
## 完成检查

{{checklist:document-completeness}}
```

或直接复制粘贴相关章节到文档中。

## v1 待解决问题（重新设计时参考）

1. **插件机制**：框架缺少插件系统，无法让第三方或内部模块以统一方式扩展框架能力
2. **请求上下文传递**：requestId / 当前用户等请求级信息如何贯穿 route → service → model
3. **service 纯单例局限**：企业级场景 service 可能需要访问请求级上下文，纯单例做不到
4. **路由元信息的消费端**：meta/docs 收集后如何自动生成 OpenAPI 文档
5. **TypeScript 类型安全**：services/schemas 作为 `any` 注入，缺少类型提示

## 🏗️ 项目定位

vext 是一个自研 Node.js Web 框架，核心设计目标：

- **Adapter 架构**: 底层框架可替换（当前基于 Hono，后期可替换为自研内核）
- **三段式路由**: `path, options, handler` — options 为声明式配置对象，暴露给文档生成工具
- **插件系统**: 插件可在 app 对象上注册方法/属性/路由
- **请求上下文封装**: req/res 对象封装请求和响应，handler 直接使用
- **服务自动注入**: 所有 service 文件注入到 `app.services.<name>` 中访问
- **schema-dsl 校验**: 参数校验采用 schema-dsl 模块

## 🚨 AI 助手规范遵循声明（强制）

### 强制要求

**当你收到涉及 vext 项目的任何开发任务时，必须严格遵守以下规则**：

1. **🔴 优先级**: 本 Profile 规范 > 通用规范 > 原有项目代码模式

2. **🔴 禁止行为**:
   - ❌ **禁止模仿原有项目代码**：即使项目中存在不符合规范的代码，也不得模仿
   - ❌ **禁止"保持一致性"**：不得以"与现有代码风格保持一致"为由违反规范
   - ❌ **禁止跳过验证**：不得跳过架构约束检查

3. **✅ 正确行为**:
   - ✅ **遵循新规范**：所有新代码必须符合本 Profile 规范
   - ✅ **重构旧代码**：修改旧代码时，必须按新规范重构
   - ✅ **遵循架构约束**：严格遵守 [02-架构约束.md](./02-架构约束.md) 的禁止项

### 新旧代码区分

| 场景 | 处理方式 |
|------|---------|
| 新增代码 | 🔴 必须完全遵循本规范 |
| 修改旧代码 | 🔴 修改部分必须按新规范重构 |
| 仅阅读旧代码 | ⚠️ 不要被旧模式误导 |

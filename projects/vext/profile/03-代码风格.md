## 📝 命名规范

### 文件命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 路由文件 | `kebab-case.ts` | `user-profile.ts` |
| 服务文件 | `kebab-case.ts` | `user-service.ts` |
| 中间件文件 | `kebab-case.ts` | `auth-guard.ts` |
| 插件文件 | `kebab-case.ts` | `cors-plugin.ts` |
| 工具文件 | `kebab-case.ts` | `string-utils.ts` |
| 配置文件 | `kebab-case.ts` | `rate-limit.config.ts` |
| 类型定义 | `kebab-case.ts` | `user.types.ts` |

### 变量 / 函数命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 变量 | `camelCase` | `userName`, `isActive` |
| 函数 | `camelCase` | `getUser()`, `validateInput()` |
| 类名 | `PascalCase` | `UserService`, `AuthPlugin` |
| 常量 | `UPPER_SNAKE_CASE` | `MAX_RETRY`, `DEFAULT_PORT` |
| 接口/类型 | `PascalCase` | `UserProfile`, `RouteOptions` |
| 枚举 | `PascalCase` | `HttpStatus`, `UserRole` |
| 枚举值 | `UPPER_SNAKE_CASE` | `HttpStatus.NOT_FOUND` |
| 私有属性 | `camelCase` + `#` 前缀 | `#db`, `#config` |

### 目录命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 顶层目录 | `kebab-case` | `routes`, `services`, `middlewares` |
| 子目录 | `kebab-case` | `user-management`, `error-handlers` |

## 🛣️ 路由代码风格

### 路由定义（三段式）

```typescript
// ✅ 正确: path, options, handler 三段式
export default defineRoute('/users/:id', {
  method: 'GET',
  middleware: [authGuard],
  validate: {
    params: z.object({ id: z.string().uuid() }),
  },
  docs: {
    summary: '获取用户详情',
    tags: ['用户'],
  },
}, async (req, res) => {
  const user = await req.app.services.user.getById(req.params.id)
  res.json(user)
})
```

### Handler 函数风格

```typescript
// ✅ 正确: 简洁，业务逻辑委托给 service 层
async (req, res) => {
  const result = await req.app.services.user.create(req.body)
```

## 🔧 服务层代码风格

### 服务类定义

```typescript
// ✅ 正确: 类的方式，通过 app 访问依赖
export class UserService {
  #app: App

  constructor(app: App) {
    this.#app = app
  }

  async getById(id: string) {
    const user = await this.#app.db.collection('users').findOne({ _id: id })
    if (!user) {
      throw new AppError('USER_NOT_FOUND', 404)
    }
    return user
  }

  async create(data: CreateUserInput) {
    // 业务逻辑在此实现
    return await this.#app.db.collection('users').insertOne(data)
  }
}
```

## 📐 TypeScript 规范

### 类型标注

```typescript
// ✅ 正确: 明确的参数和返回类型
async function getUser(id: string): Promise<User | null> {
  return await db.findOne({ _id: id })
}

// ❌ 错误: 缺少类型标注
async function getUser(id) {
  return await db.findOne({ _id: id })
}
```

### 类型导入

```typescript
// ✅ 正确: 使用 type 关键字导入纯类型
import type { User, CreateUserInput } from '../types'

// ❌ 错误: 类型和值混合导入（当只需要类型时）
import { User, CreateUserInput } from '../types'
```

### 避免 `any`

```typescript
// ✅ 正确: 使用 unknown + 类型收窄
function processData(data: unknown): string {
  if (typeof data === 'string') return data
  if (typeof data === 'object' && data !== null) return JSON.stringify(data)
  return String(data)
}

// ❌ 错误: 使用 any
function processData(data: any): string {
  return data.toString()
}
```

## 🚫 禁止事项

| 禁止 | 原因 | 替代方案 |
|------|------|---------|
| `var` 声明 | 作用域混乱 | 使用 `const` / `let` |
| `==` 比较 | 隐式类型转换 | 使用 `===` |
| 嵌套回调 | 可读性差 | 使用 `async/await` |
| `console.log` 调试 | 生产环境残留 | 使用 `app.logger` |
| 魔法数字 | 含义不明 | 定义命名常量 |
| `export default` 匿名 | 调试困难 | 使用命名导出或命名函数 |
| 修改函数参数 | 副作用 | 创建新对象 |

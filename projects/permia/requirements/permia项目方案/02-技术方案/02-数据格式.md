## 一、FileAdapter 磁盘存储格式（permia-data.json）

```json
{
  "roles": {
    "<roleId>": {
      "id": "<roleId>",
      "label": "显示名称",
      "parent": "<parentRoleId 或 null>",
      "description": "描述",
      "createdAt": 1709424000000,
      "updatedAt": 1709424000000
    }
  },
  "userRoles": {
    "<userId>": ["<roleId1>", "<roleId2>"]
  },
  "rules": {
    "<roleId>": [
      { "type": "allow", "action": "invoke", "resource": "GET:/api/users" },
      { "type": "deny",  "action": "delete", "resource": "db:users" }
    ]
  }
}
```

**三个顶层键说明**：

| 键 | 类型 | 说明 |
|----|------|------|
| `roles` | `Object<roleId, RoleData>` | 所有角色定义 |
| `userRoles` | `Object<userId, string[]>` | 用户→角色绑定（一个用户可绑定多角色） |
| `rules` | `Object<roleId, PermissionRule[]>` | 每个角色自身的权限规则（不含继承） |

## 三、数据量估算

| 规模 | 角色数 | 用户数 | 规则数/角色 | 文件大小估算 |
|------|--------|--------|------------|-------------|
| 小型 | 5~10 | ≤100 | ~5 | ~15 KB |
| 中型 | 20~50 | ≤1,000 | ~10 | ~100 KB |
| 大型 | 50~100 | ≤10,000 | ~20 | ~700 KB |

> **超大规模**（角色 100+，用户 10,000+）建议切换到 MonSQLizeAdapter，FileAdapter 不适合多进程部署。

## 四、数据存储说明

| 存储层 | 格式 | 时机 |
|--------|------|------|
| 内存 `_data` | 同上 JSON 结构对应的 Object | 启动时加载，运行时读写 |
| 磁盘文件 | 美化 JSON（`JSON.stringify(_data, null, 2)`） | debounce 100ms 后异步写盘 |
| 反向索引 `_roleUsers` | `Map<roleId, Set<userId>>` | 内存中维护，不落盘，启动时重建 |

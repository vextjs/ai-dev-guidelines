## 四、资源路径模型

```
格式:  <scope>:<resource>[:<field>]

数据库/集合级:  'db:users'                   集合整体的增删改查权限
字段级:         'db:users:email'              单字段读写权限
               'db:users:*'                  该集合所有字段
接口级:         'api:users:create'            对应 POST /users 接口权限
               'api:users:*'                 users 资源下所有接口
               'api:*'                        所有接口
全局通配:       '*'                           所有资源

操作类型(action):
  db: 资源 → read | write | create | update | delete | *
  api: 资源 → invoke | *
  通用:      *

说明:
  - "栏目"和"按钮"权限本质是接口（API）级鉴权
  - 前端按钮/栏目是否显示由前端根据接口权限自行判断，permia 不处理 UI 渲染
  - 原 'ui:dashboard:btn-publish' → 'api:articles:publish'
```

## 五、核心 API 设计

### 5.1 主类 Permia

```javascript
const { Permia, MemoryAdapter } = require('permia');

const permia = new Permia({
  storage: new MemoryAdapter(),        // 默认；可替换为 MonSQLizeAdapter 等
  cache: { enabled: true, ttl: 300000 },
  strict: true                         // deny 规则严格优先（默认 true）
});

// 角色管理
await permia.roles.create('admin',  { label: '管理员' });
await permia.roles.create('editor', { label: '编辑', parent: 'viewer' });

// 权限规则
await permia.roles.allow('admin',  '*',                    '*');
await permia.roles.allow('editor', ['read','write'],       'db:articles');
await permia.roles.allow('editor', 'read',                 'db:users:email');
await permia.roles.deny( 'editor', 'delete',               'db:users');
await permia.roles.allow('editor', 'view',                 'ui:dashboard:btn-publish');

// 用户绑定
await permia.users.assign('uid-1', 'editor');
await permia.users.revoke('uid-1', 'editor');

// 鉴权
const ok = await permia.can('uid-1', 'write', 'db:articles');   // → boolean
await permia.assert('uid-1', 'delete', 'db:users');              // 失败抛 PermiaError

// 字段过滤
const safe = await permia.filterFields('uid-1', 'read', 'db:users', record);

// 链式 API
const ctx = permia.for('uid-1');
await ctx.can('read', 'db:articles');
await ctx.filterFields('read', 'db:users', record);
```

### 5.2 存储适配器接口

```javascript
// 用户实现此接口可对接任意存储后端
class StorageAdapter {
  async getRoles()                           // → Map<id, RoleData>
  async setRole(id, data)
  async deleteRole(id)
  async getUserRoles(userId)                 // → string[]
  async setUserRoles(userId, roleIds)
  async getPermissions(roleId)               // → PermissionRule[]
  async setPermissions(roleId, rules)
}

// 内置：MemoryAdapter（默认）
// 官方可选：MonSQLizeAdapter（基于 monsqlize，支持 MongoDB，后期可切换其他数据库）
```

### 5.3 错误码

```javascript
PermiaError.codes = {
  PERMISSION_DENIED,      // 鉴权拒绝
  ROLE_NOT_FOUND,         // 角色不存在
  USER_NOT_FOUND,         // 用户不存在
  ROLE_ALREADY_EXISTS,    // 角色已存在
  CIRCULAR_INHERITANCE,   // 循环继承
  INVALID_RESOURCE_PATH,  // 资源路径格式错误
  INVALID_ACTION,         // 非法操作类型
  STORAGE_ERROR,          // 存储层错误
}
```

## 七、存储适配器策略

| 适配器 | 包 | 说明 |
|--------|-----|------|
| `MemoryAdapter` | 内置（零依赖） | 默认，开箱即用，适合开发/测试 |
| `MonSQLizeAdapter` | `permia-adapter-monsqlize`（独立包，后期） | 生产推荐，基于 monSQLize 持久化，后期可无缝切换 MySQL/PostgreSQL 等 |
| 自定义 | 用户实现 `StorageAdapter` 接口 | 任意存储后端 |

> **设计原则**: permia 核心包零强依赖；monsqlize 为可选适配器，独立包发布，避免强耦合。

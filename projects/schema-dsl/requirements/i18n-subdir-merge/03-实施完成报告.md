## Bug 修复说明（3 个测试污染）

**根因**：初版 `i18n-subdir.test.js` 使用了根级别 `beforeEach(() => Locale.reset())`，
在 mocha 全量跑时该 hook 会作用于所有后续文件的每个 it，清除了其他文件 `before()` 中注册的自定义 key。

**修复**：将所有测试包裹在 `describe('i18n 子目录合并')` 内，改为 describe 作用域内的 `afterEach(Locale.reset)`，
状态清理严格限制在本文件范围内，不再泄漏至其他测试文件。

| 受影响测试 | 根因 | 状态 |
|-----------|------|------|
| `custom-features` Dynamic Locale `min` key | beforeEach reset 清掉了 `before()` 注册的 `min: 太短` | ✅ 已修复 |
| `error-message-interpolation` enum 中文消息 | zh-CN enum key 被 reset 清掉 | ✅ 已修复 |
| `label-translation` label.test_field | before() 注册的 label key 被 reset 清掉 | ✅ 已修复 |

## 核心实现摘要

### `loadLocalesFromDir(dirPath, options)` 函数逻辑

```
1. 路径存在性检查 → 不存在 console.warn，不抛错
2. readdirSync(dirPath, { withFileTypes: true })
3. 遍历 entries：
   ├── 目录 → 递归调用（子目录名只作组织层）
   └── 文件 + (.js|.json) + 语言代码格式正则校验
       → 校验通过：冲突检测 → Locale.addLocale
       → 校验失败：静默跳过
4. 冲突检测：
   → strict=true：throw Error（含 key 名、文件路径）
   → strict=false（默认）：console.warn
```

### 兼容性保证

- 所有现有用法（字符串路径/对象语言包/localesPath 对象）100% 兼容
- `strict` 默认 `false`，不影响任何现有行为
- 全量测试从 14 个失败降至 0 个失败，含 3 个跨文件状态污染 bug 修复

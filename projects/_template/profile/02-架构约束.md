## 🚫 架构约束（强制）

### 1. 禁止在 Controller 层直接操作数据库

Controller 仅处理请求和响应，业务逻辑和数据访问必须在 Service 层。

### 2. 禁止在 Service 层使用 ctx.request/ctx.response

Service 应独立于 HTTP 层，便于测试和复用。

### 3. 禁止绕过中间件直接返回响应

统一响应格式由 `responseHelper` 中间件处理。

### 4. 禁止在 Model 层编写业务逻辑

Model 仅定义数据结构和验证规则。

### 5. 禁止硬编码数据库连接字符串

通过配置中心管理。

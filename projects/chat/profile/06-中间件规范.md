# 06 - 中间件规范

> **模块**: 中间件加载顺序与使用规范  
> **版本**: v1.0  
> **最后更新**: 2026-02-24  
> **视角**: 本文档使用"你"指代 AI 助手（规范执行者）

---

## 📋 中间件加载顺序

```typescript
config.middleware = [
    'language',            // 1. 多语言（最先，确保 i18n 可用）
    'responseHelper',      // 2. 统一响应格式（提供 ctx.success）
    'validatorHelper',     // 3. 参数校验辅助
    'schemaDslValidator',  // 4. schema-dsl 验证与错误处理（核心）
    'httpHelper',          // 5. HTTP 请求工具（提供 ctx.http）
    'crudHelper',          // 6. CRUD 辅助
    'exceptions',          // 7. 全局异常处理
    'visitorLimit',        // 8. 访客限制
];
```

---

## 1. language.ts - 多语言

**用途**: 解析请求头 `language` 字段，设置当前语言环境

**必须放在第一位**，确保其他中间件可以使用 `ctx.i18n`。

---

## 2. responseHelper.ts - 统一响应格式

**用途**: 提供 `ctx.success()` 方法，统一响应格式

```typescript
// Controller 层
ctx.success(data);
// 输出: { code: 0, message: '操作成功', data: {...}, timestamp: '...' }
```

---

## 3. schemaDslValidator.ts - Schema DSL 验证与错误处理

**用途**: 提供参数验证和错误处理功能

**核心方法**:
- `ctx.dsl({...})` - 定义 Schema DSL 规则
- `ctx.validateDsl(schema, source)` - 验证参数（`'query'` / `'body'` / `'params'`）
- `ctx.dsl.error.throw(errorCode)` - 抛出业务错误（try 块中使用）
- `ctx.dsl.error.create(errorCode)` - 创建系统错误对象（catch 块中使用）

**详见**: [05-参数校验规范.md](./05-参数校验规范.md)、[04-错误处理与多语言.md](./04-错误处理与多语言.md)

---

## 4. httpHelper.ts - HTTP 请求工具

**用途**: 提供 `ctx.http` 对象，封装跨服务 HTTP 调用

**核心方法**:

| 方法 | 说明 |
|------|------|
| `ctx.http.postJSON(tag, url, body, init?)` | POST JSON 数据 |
| `ctx.http.getJSON(tag, url, init?)` | GET 并解析 JSON |
| `ctx.http.fetchJSON(tag, url, init?)` | 通用请求（推荐） |

**🔴 重要**: 所有方法的第一个参数是 `tag`（日志标识），用于日志追踪。

**详见**: [09-微服务通信.md](./09-微服务通信.md)

---

## 5. userAuth.ts - JWT 鉴权

**用途**: JWT + 单点登录鉴权，解析用户信息到 `ctx.state.user`

**使用方式**: 在路由层作为中间件挂载

```typescript
const userAuth = app.middleware.userAuth({ level: 'basic' });
sub.get('/some-api', userAuth, controller.home.someController.method);
```

---

## 6. internalAuth.ts - 内部通信鉴权

**用途**: 内部服务间通信鉴权（Token 校验）

```typescript
const internalAuth = app.middleware.internalAuth();
sub.post('/internal/some-api', internalAuth, controller.internal.someController.method);
```

---

## 7. exceptions.ts - 全局异常处理

**用途**: 统一捕获所有未处理的异常，返回标准错误响应

**必须放在中间件列表的靠后位置**。

---

## 8. visitorLimit.ts - 访客限制

**用途**: 限制特定路径的访客访问频率

**忽略路径**:
```typescript
config.visitorLimit = {
    ignore: ['/open', '/health', '/internal'],
};
```

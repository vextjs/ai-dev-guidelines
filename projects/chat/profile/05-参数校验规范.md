# 05 - 参数校验规范

> **模块**: schema-dsl 参数校验与 Controller 内联校验模式  
> **重要性**: 🔴 P0 - 所有新代码必须遵守  
> **版本**: v1.1  
> **最后更新**: 2026-03-06  
> **视角**: 本文档使用"你"指代 AI 助手（规范执行者）

---

## 📋 唯一推荐方式

所有新代码**必须**使用 `schema-dsl`，**禁止**使用 Joi 编写新的校验逻辑。

---

## 🚀 标准模式：Route → Controller（内联校验）→ Service

参数校验**直接在 Controller 方法内完成**，不需要创建独立的 Validator 类文件。

### 架构层次

```
Route（路由注册）
  → Controller（参数校验 + 调用 Service + 响应）
      → Service（业务逻辑 + 错误处理）
```

### ❌ 禁止方式：独立 Validator 类

```typescript
// ❌ 禁止：创建独立的 Validator 文件
// app/validator/home/blog_validator.ts
import { BaseContextClass } from 'egg';
export default class BlogValidator extends BaseContextClass {
    async create() { /* ... */ }
}

// ❌ 禁止：Controller 中实例化 Validator
export default class BlogController extends Controller {
    private validator: BlogValidator;
    constructor(ctx: any) {
        super(ctx);
        this.validator = new BlogValidator(ctx);
    }
    async create() {
        await this.validator.create();  // ❌ 不要这样做
    }
}
```

> **说明**：项目中 `app/validator/admin/` 目录下存在历史遗留的独立 Validator 类，这是旧代码模式。
> **所有新代码**必须使用下方的 Controller 内联校验方式。

## ✅ 正确方式：Controller 内联校验

### Step 1: Controller 层（参数校验 + 调用 Service + 响应）

```typescript
// app/controller/home/BlogController.ts
import { Controller } from 'egg';

export default class BlogController extends Controller {

    async create() {
        const { ctx } = this;

        // 1. 参数校验（内联，校验失败自动返回多语言错误响应）
        const schema = ctx.dsl({
            title: 'string:1-200!',
            content: 'string:1-50000!',
            summary: 'string:0-500',
            cover_image: 'url',
            tags: 'array',
            status: 'string',
        });
        await ctx.validateDsl(schema, 'body');

        // 2. 获取参数
        const form = ctx.state.form;
        const userId = ctx.state.user._id;

        // 3. 调用 Service（不需要 try-catch）
        const result = await ctx.service.blog.blogService.createBlog(form, userId);

        // 4. 成功响应
        ctx.success(result);
    }

    async update() {
        const { ctx } = this;

        // 路径参数校验
        const paramsSchema = ctx.dsl({
            blogId: 'objectId!',
        });
        await ctx.validateDsl(paramsSchema, 'params');

        // 请求体校验（所有字段可选）
        const bodySchema = ctx.dsl({
            title: 'string:1-200',
            content: 'string:1-50000',
            summary: 'string:0-500',
            cover_image: 'url',
            tags: 'array',
            status: 'string',
        });
        await ctx.validateDsl(bodySchema, 'body');

        const blogId = ctx.params.blogId;
        const form = ctx.state.form;
        const userId = ctx.state.user._id;

        const result = await ctx.service.blog.blogService.updateBlog(blogId, form, userId);
        ctx.success(result);
    }

    async delete() {
        const { ctx } = this;

        const schema = ctx.dsl({
            blogId: 'objectId!',
        });
        await ctx.validateDsl(schema, 'params');

        const blogId = ctx.params.blogId;
        const userId = ctx.state.user._id;

        const result = await ctx.service.blog.blogService.deleteBlog(blogId, userId);
        ctx.success(result);
    }
}
```

### Step 2: Service 层（业务逻辑 + 错误处理）

```typescript
// app/service/blog/BlogService.ts
import { Service } from 'egg';

export default class BlogService extends Service {
    async createBlog(form: any, authorId: string) {
        const { ctx } = this;
        try {
            // 业务校验
            if (form.tags && form.tags.length > 10) {
                ctx.dsl.error.throw('BLOG_TAGS_LIMIT');
            }

            // 自动摘要
            form.summary = form.summary || form.content.slice(0, 200);

            const doc = {
                ...form,
                author_id: authorId,
                view_count: 0,
                del_flag: 0,
                published_at: form.status === 'published' ? new Date() : null,
                created_at: new Date(),
                updated_at: new Date(),
            };

            return await ctx.model.Blog.create(doc);
        } catch (error: any) {
            if (error.code) throw error;
            throw ctx.dsl.error.create('BLOG_CREATE_FAILED');
        }
    }
}
```

## 📖 DSL 语法规则

### 基本格式

```
type:min-max[!]
```

### 类型

| 类型 | 说明 | 示例 |
|------|------|------|
| `string` | 字符串 | `string:2-50` |
| `int` | 整数 | `int:1-100` |
| `float` | 浮点数 | `float:0-` |
| `number` | 数字（整数或浮点） | `number:0-999999` |
| `email` | 邮箱 | `email!` |
| `url` | URL | `url` |
| `objectId` | MongoDB ObjectId | `objectId!` |
| `array` | 数组 | `array` |
| `boolean` | 布尔值 | `boolean` |
| `enum` | 枚举 | `enum:draft\|published!` |

### 必填标记

| 格式 | 说明 | 示例 |
|------|------|------|
| `!` | 必填字段 | `email!` |
| 省略 | 可选字段 | `string:2-50` |

### 常用场景

```typescript
// 分页查询（游标分页）
const schema = ctx.dsl({
    limit: 'int:1-50',
    after: 'string',
    keyword: 'string:0-100',
});
await ctx.validateDsl(schema, 'query');

// 路径参数
const schema = ctx.dsl({
    blogId: 'objectId!',
});
await ctx.validateDsl(schema, 'params');

// 创建/更新
const schema = ctx.dsl({
    title: 'string:1-200!',
    description: 'string:0-1000',
    status: 'enum:draft|published',
});
await ctx.validateDsl(schema, 'body');
```

---

## 📐 校验来源（validateDsl 第二参数）

| 来源 | 说明 | 适用场景 |
|------|------|---------|
| `'body'` | `ctx.request.body` | POST / PUT 请求体 |
| `'query'` | `ctx.query` | GET 请求查询参数 |
| `'params'` | `ctx.params` | 路由路径参数（如 `:blogId`） |

> 校验通过后，校验过的参数会自动写入 `ctx.state.form`，Controller 中通过 `ctx.state.form` 获取。

---

## 🔑 分层职责划分

| 层 | 负责的校验 | 示例 |
|---|-----------|------|
| **Controller** | 格式校验（类型、长度、必填、枚举） | `title: 'string:1-200!'`、`blogId: 'objectId!'` |
| **Service** | 业务校验（业务规则、权限、存在性） | `tags.length > 10`、`author_id !== userId`、文章是否存在 |

> **原则**：Controller 只做「参数格式是否正确」，Service 做「业务逻辑是否允许」。

## ⚠️ 常见错误

### ❌ 错误 1: 业务错误使用 throw new Error()

```typescript
// ❌ 错误
if (!user) throw new Error('User not found');

// ✅ 正确
if (!user) ctx.dsl.error.throw('USER_NOT_FOUND');
```

### ❌ 错误 2: Controller 层手动 try-catch

```typescript
// ❌ 错误
async create() {
    try {
        const result = await ctx.service.xxx.create(data);
        ctx.success(result);
    } catch (error) {
        ctx.fail(error.message);
    }
}

// ✅ 正确（不需要 try-catch，由 exceptions 中间件统一处理）
async create() {
    const schema = ctx.dsl({ title: 'string:1-200!' });
    await ctx.validateDsl(schema, 'body');
    const result = await ctx.service.xxx.create(ctx.state.form);
    ctx.success(result);
}
```

### ❌ 错误 3: 创建独立的 Validator 文件

```typescript
// ❌ 错误：在 app/validator/ 下新建 Validator 类
// app/validator/home/blog_validator.ts  ← 禁止新建

// ✅ 正确：直接在 Controller 方法内校验
async create() {
    const schema = ctx.dsl({ title: 'string:1-200!' });
    await ctx.validateDsl(schema, 'body');
    // ...
}
```

### ❌ 错误 4: 在 Service 层做格式校验

```typescript
// ❌ 错误：Service 层做格式校验
async createBlog(form: any) {
    if (!form.title || form.title.length > 200) {
        ctx.dsl.error.throw('INVALID_TITLE');
    }
}

// ✅ 正确：格式校验在 Controller，Service 只做业务校验
// Controller:
const schema = ctx.dsl({ title: 'string:1-200!' });
await ctx.validateDsl(schema, 'body');
// Service:
if (form.tags && form.tags.length > 10) {
    ctx.dsl.error.throw('BLOG_TAGS_LIMIT');
}
```

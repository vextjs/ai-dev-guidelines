## 🔥 错误处理规范

### 🔴 强制规则

1. **业务错误**（try 块中）：使用 `ctx.dsl.error.throw(errorCode)`
2. **系统错误**（catch 块中）：使用 `throw ctx.dsl.error.create(errorCode)`
3. **Controller 层禁止 try-catch**：错误由 `exceptions` 中间件统一捕获

### ❌ 禁止方式

```typescript
// ❌ 禁止使用 throw new Error()
throw new Error('User not found');

// ❌ 禁止使用 ExError（旧方式）
throw new ExError('FAILED');

// ❌ 禁止使用 ctx.fail（旧方式）
throw ctx.fail('ERROR_CODE', 404);

// ❌ 禁止手动拼装错误响应
ctx.body = { code: -1, success: false, message: '...' };
```

### ✅ 正确方式

```typescript
// ✅ 业务错误（在 try 块中）
ctx.dsl.error.throw('TRIP_NOT_FOUND');

// ✅ 系统错误（在 catch 块中）
throw ctx.dsl.error.create('QUERY_FAILED');
```

### 完整 Service 层示例

```typescript
export default class TripProjectService extends Service {
    async createTripProject(data: any) {
        const { ctx } = this;
        try {
            const existing = await ctx.model.TripProject.findOne({
                title: data.title, del_flag: 0
            });
            if (existing) {
                // ✅ 业务错误
                ctx.dsl.error.throw('PROJECT_ALREADY_EXISTS');
            }
            return await ctx.model.TripProject.create(data);
        } catch (error: any) {
            // 业务错误直接抛出
            if (error.code) throw error;
            // ✅ 系统错误包装
            throw ctx.dsl.error.create('PROJECT_CREATE_FAILED');
        }
    }
}
```

### 响应格式

```typescript
// ✅ 成功响应（统一使用 ctx.success）
ctx.success(data);
// 输出: { code: 0, message: '操作成功', data: {...}, timestamp: '...' }

// ❌ 禁止直接赋值 ctx.body（旧方式）
ctx.body = { code: 0, desc: '', data: {...} };
```

> **说明**: 现有旧代码中存在 `ctx.body = { code: 0, ... }` 和 `throw ctx.fail()` 的用法，
> 这是历史遗留。**所有新代码**必须使用 `ctx.success()` + `ctx.dsl.error.throw/create`。

## 🌐 多语言错误码配置

### 🔴 唯一正确位置：`config/dsl/`

```
config/dsl/
├── index.ts       # 导出 dslLocales
├── zh.ts          # 简体中文（必须）
├── en.ts          # 英文（必须）
└── hk.ts          # 繁体中文（必须）
```

### 配置格式（必须包含 code 和 message）

```typescript
// config/dsl/zh.ts
export default {
    'PROJECT_ALREADY_EXISTS': {
        code: 40101,
        message: '项目已存在'
    },
    'PROJECT_CREATE_FAILED': {
        code: 50101,
        message: '项目创建失败'
    },
    'TRIP_NOT_FOUND': {
        code: 40102,
        message: '行程不存在'
    },
};
```

```typescript
// config/dsl/en.ts
export default {
    'PROJECT_ALREADY_EXISTS': {
        code: 40101,
        message: 'Project already exists'
    },
    'PROJECT_CREATE_FAILED': {
        code: 50101,
        message: 'Project creation failed'
    },
    'TRIP_NOT_FOUND': {
        code: 40102,
        message: 'Trip not found'
    },
};
```

```typescript
// config/dsl/hk.ts
export default {
    'PROJECT_ALREADY_EXISTS': {
        code: 40101,
        message: '項目已存在'
    },
    // ...
};
```

### 🔴 禁止事项

- ❌ 禁止在 `config/err_codes.ts` 中配置新错误码（已废弃）
- ❌ 禁止在 `config/error_messages.ts` 中配置（已废弃）
- ❌ 禁止在代码中硬编码错误消息字符串
- ❌ 禁止使用简单字符串格式（必须 `{ code, message }` 对象格式）

### 错误码分段建议

| 段 | 范围 | 用途 |
|---|------|------|
| 4xxxx | 40000-49999 | 业务错误（客户端错误） |
| 5xxxx | 50000-59999 | 系统错误（服务端错误） |
| 8xxxx | 80000-89999 | 参数验证错误 |

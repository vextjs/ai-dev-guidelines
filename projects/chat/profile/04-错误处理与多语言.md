## 🔥 错误处理规范

### 🔴 强制规则

1. **业务错误**（try 块中）：使用 `ctx.dsl.error.throw(errorCode)`
2. **系统错误**（catch 块中）：使用 `throw ctx.dsl.error.create(errorCode)`
3. **Controller 层禁止 try-catch**：错误由 `exceptions` 中间件统一捕获

### ❌ 禁止方式

```typescript
// ❌ 禁止使用 throw new Error()
throw new Error('User not found');

// ❌ 禁止使用 ExError（旧方式）
throw new ExError('FAILED');

// ❌ 禁止使用 ctx.fail（旧方式）
throw ctx.fail('ERROR_CODE', 404);

// ❌ 禁止手动拼装错误响应
ctx.body = { code: -1, success: false, message: '...' };
```

### ✅ 正确方式

```typescript
// ✅ 业务错误（在 try 块中）
ctx.dsl.error.throw('BLOG_NOT_FOUND');

// ✅ 系统错误（在 catch 块中）
throw ctx.dsl.error.create('BLOG_CREATE_FAILED');
```

### 完整 Service 层示例

```typescript
export default class BlogService extends Service {
    async createBlog(form: any, authorId: string) {
        const { ctx } = this;
        try {
            if (form.tags && form.tags.length > 10) {
                // ✅ 业务错误
                ctx.dsl.error.throw('BLOG_TAGS_LIMIT');
            }
            return await ctx.model.Blog.create({ ...form, author_id: authorId });
        } catch (error: any) {
            // 业务错误直接抛出
            if (error.code) throw error;
            // ✅ 系统错误包装
            throw ctx.dsl.error.create('BLOG_CREATE_FAILED');
        }
    }
}
```

### 响应格式

```typescript
// ✅ 成功响应（统一使用 ctx.success）
ctx.success(data);
// 输出: { code: 0, message: '操作成功', data: {...}, timestamp: '...' }

// ❌ 禁止直接赋值 ctx.body（旧方式）
ctx.body = { code: 0, desc: '', data: {...} };
```

> **说明**: 现有旧代码中存在 `ctx.body = { code: 0, ... }` 和 `throw ctx.fail()` 的用法，
> 这是历史遗留。**所有新代码**必须使用 `ctx.success()` + `ctx.dsl.error.throw/create`。

## 🌐 多语言错误码配置

### 🔴 唯一正确位置：`config/dsl/`

### 目录结构：模块化子目录（v1.1 新增）

chat 服务使用**模块化子目录**组织 DSL 多语言配置，每个业务模块独立一个子目录：

```
config/dsl/
├── core/                    ← 通用 key（通用验证、字段标签、全局错误）
│   ├── zh-CN.ts
│   ├── en-US.ts
│   └── zh-HK.ts
├── trip_square/             ← 行程广场模块专属 key
│   ├── zh-CN.ts
│   ├── en-US.ts
│   └── zh-HK.ts
├── blog/                    ← Blog 博客模块专属 key
│   ├── zh-CN.ts
│   ├── en-US.ts
│   └── zh-HK.ts
├── <新模块>/                ← 未来新增模块，同结构
│   ├── zh-CN.ts
│   ├── en-US.ts
│   └── zh-HK.ts
└── index.ts                 ← 路径扫描 + 合并导出
```

> **子目录扫描**：`index.ts` 通过 schema-dsl v1.2.3+ 的 `dsl.config({ dirs: [...] })` 自动扫描各子目录，
> 同时合并导出 locale 对象供 `error_catch.ts` 使用。

### 🔴 新增模块时的操作步骤

1. 在 `config/dsl/` 下新建模块子目录（如 `config/dsl/blog/`）
2. 创建三个语言文件：`zh-CN.ts`、`en-US.ts`、`zh-HK.ts`
3. 在 `config/dsl/index.ts` 中追加新目录路径和合并导出
4. **禁止**直接在 `core/` 子目录中追加模块专属错误码

### 历史遗留说明

> ⚠️ 项目当前实际仍为**扁平文件结构**（`zh.ts` / `en.ts` / `hk.ts`），尚未迁移到子目录结构。
> 迁移工作已在行程广场需求的 Phase 0 中规划（将旧文件迁移到 `core/` 子目录，删除旧文件）。
> **所有新模块的 DSL 错误码必须按子目录结构规划**，在 Phase 0 迁移完成后直接落地。

## 📐 配置格式（必须包含 code 和 message）

### 🔴 强制格式：`{ code, message }` 对象

每个 DSL 错误条目**必须**包含 `code`（数字错误码）和 `message`（多语言文案）两个字段。

### 🔴 禁止事项

- ❌ 禁止在 `config/err_codes.ts` 中配置新错误码（已废弃）
- ❌ 禁止在 `config/error_messages.ts` 中配置（已废弃）
- ❌ 禁止在代码中硬编码错误消息字符串
- ❌ 禁止使用简单字符串格式（必须 `{ code, message }` 对象格式）
- ❌ 禁止跨模块 code 码重复

```typescript
// ❌ 禁止：简单字符串格式
'BLOG_NOT_FOUND': '文章不存在',

// ✅ 正确：{ code, message } 对象格式
'BLOG_NOT_FOUND': {
    code: 40301,
    message: '文章不存在',
},
```

### 历史遗留代码说明

> ⚠️ 现有 `config/dsl/zh.ts` / `en.ts` / `hk.ts` 中的错误码使用的是**简单字符串格式**
>（如 `'SELL_CONFIG_PROJECT_NOT_FOUND': '行程不存在'`），这是历史遗留。
> **所有新增的错误码**必须使用 `{ code, message }` 对象格式。
> 旧代码将在子目录迁移时统一改造。

## 🔢 错误码分段规则（code 码分配）

### 分段总则

| 前缀 | 范围 | 用途 |
|:----:|------|------|
| 4xxxx | 40000 - 49999 | **业务错误**（客户端错误、参数不合法、权限不足、资源不存在等） |
| 5xxxx | 50000 - 59999 | **系统错误**（服务端异常、数据库写入失败、外部服务调用失败等） |

### 模块分段表

每个业务模块分配独立的 code 码段，**禁止跨模块占用**：

| 模块 | 业务错误段 | 系统错误段 | 说明 |
|------|:---------:|:---------:|------|
| **core**（通用） | 40001 - 40099 | 50001 - 50099 | 通用/全局错误（登录、Token、权限等） |
| **sell_config**（售卖配置） | 40101 - 40199 | 50101 - 50199 | 行程售卖配置相关 |
| **trip_square**（行程广场） | 40201 - 40299 | 50201 - 50299 | 行程广场、点赞、限流等 |
| **blog**（博客） | 40301 - 40399 | 50301 - 50399 | 博客 CRUD 相关 |
| _(预留)_ | 40401 - 40499 | 50401 - 50499 | 未来新增模块 A |
| _(预留)_ | 40501 - 40599 | 50501 - 50599 | 未来新增模块 B |

### 🔴 分段使用规则

1. **新增模块**：在上表中申领下一个可用段（如 404xx），更新本文档
2. **段内递增**：同一模块内的 code 码从段起始值依次递增（如 40301 → 40302 → 40303）
3. **三语言一致**：同一 key 在 `zh-CN.ts` / `en-US.ts` / `zh-HK.ts` 中的 `code` 值**必须相同**
4. **禁止跨段**：如 blog 模块的 code 不得使用 401xx 段
5. **禁止复用**：已使用的 code 码即使对应的 key 被移除，也**不得**分配给新 key

### 完整示例：Blog 模块 code 码分配

| 错误码 Key | code | 类型 | 触发场景 |
|-----------|:----:|:----:|---------|
| `BLOG_NOT_FOUND` | 40301 | 业务 | 文章不存在、已删除 |
| `BLOG_FORBIDDEN` | 40302 | 业务 | 非文章作者操作 |
| `BLOG_TAGS_LIMIT` | 40303 | 业务 | 标签数量超过 10 个 |
| `BLOG_CREATE_FAILED` | 50301 | 系统 | 文章创建数据库写入失败 |
| `BLOG_UPDATE_FAILED` | 50302 | 系统 | 文章更新数据库写入失败 |
| `BLOG_DELETE_FAILED` | 50303 | 系统 | 文章删除数据库写入失败 |
| `BLOG_LIST_FAILED` | 50304 | 系统 | 文章列表查询异常 |
| `BLOG_QUERY_FAILED` | 50305 | 系统 | 文章详情查询异常 |

### 完整示例：Trip Square 模块 code 码分配

| 错误码 Key | code | 类型 | 触发场景 |
|-----------|:----:|:----:|---------|
| `TRIP_NOT_FOUND` | 40201 | 业务 | 行程不存在或已删除 |
| `TRIP_NOT_LISTED` | 40202 | 业务 | 行程未上架到广场 |
| `RATE_LIMIT_EXCEEDED` | 40203 | 业务 | 触发限流规则 |

## 📄 index.ts 合并导出参考

`config/dsl/index.ts` 需要同时满足两个消费方：

```typescript
// config/dsl/index.ts
import * as path from 'path';

// 导入各子目录的 locale 文件
import { zhCN as coreZhCN } from './core/zh-CN';
import { enUS as coreEnUS } from './core/en-US';
import { zhHK as coreZhHK } from './core/zh-HK';

import { zhCN as tripSquareZhCN } from './trip_square/zh-CN';
import { enUS as tripSquareEnUS } from './trip_square/en-US';
import { zhHK as tripSquareZhHK } from './trip_square/zh-HK';

import { zhCN as blogZhCN } from './blog/zh-CN';
import { enUS as blogEnUS } from './blog/en-US';
import { zhHK as blogZhHK } from './blog/zh-HK';

// 消费方 1：供 app.ts 的 dsl.config() 使用（目录路径列表，自动扫描）
export const dslDirs = [
    path.join(__dirname, 'core'),
    path.join(__dirname, 'trip_square'),
    path.join(__dirname, 'blog'),
];

// 消费方 2：供 error_catch.ts 使用（合并后的 locale 对象）
export const zhCNLocale = { ...coreZhCN, ...tripSquareZhCN, ...blogZhCN };
export const enUSLocale = { ...coreEnUS, ...tripSquareEnUS, ...blogEnUS };
export const zhHKLocale = { ...coreZhHK, ...tripSquareZhHK, ...blogZhHK };
```

> **新增模块时**：在此文件中追加 import 和合并导出，同时将新目录路径加入 `dslDirs` 数组。

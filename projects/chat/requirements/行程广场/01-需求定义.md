# 行程广场功能需求定义

> **任务类型**: 需求开发（01-requirement-dev）— 完整模式
> **需求来源**: 用户提出行程广场 5 项子需求
> **涉及项目**: chat 服务（Egg.js + TypeScript + MongoDB）

---

## 一、业务背景

当前行程售卖功能已支持用户配置售卖参数（价格、预览天数、权限等），买家可通过分享链接预览并购买行程。但缺少一个**公开的行程展示入口**——即「行程广场」，让所有用户（含未登录访客）能浏览、搜索、发现优质行程。

本次需求围绕「行程广场」核心场景，新增 5 项子功能。

## 三、逐项需求定义

### REQ-1: 售卖配置新增「是否上架到广场」字段

**业务目标**: 让卖家控制行程是否对外展示在广场中，实现"开启售卖 ≠ 上架广场"的灵活配置。

**功能需求**:

1. 在行程售卖配置（`sell_config`）中新增 `listed_in_square` 布尔字段，默认为 `false`
2. 该字段仅在 `sell_config.enabled = true`（开启售卖）时有意义
3. 当 `enabled` 设为 `false` 时，`listed_in_square` 应强制设为 `false`
4. 保存售卖配置接口需新增该字段的 DSL 验证和写入逻辑
5. 国际化 label 需同步新增（中文/英文/繁体中文）

**验收标准**:

- [ ] `PUT /home/trip_project/trip/:tripId/sell-config` 接口支持 `listed_in_square` 参数
- [ ] `listed_in_square` 默认值为 `false`
- [ ] `enabled=false` 时保存，`listed_in_square` 被强制设为 `false`
- [ ] `GET /home/trip_project/trip/:tripId/sell-config` 返回值中包含 `listed_in_square`
- [ ] 三语言包均包含 `label.listed_in_square`

---

### REQ-2: 售卖价格支持配置为 0（免费行程）

**业务目标**: 允许卖家将行程价格设为 0，支持免费行程场景。

**功能需求**:

1. 现金价格（`price`）最小值从 100（即 $1.00）改为 0
2. 积分价格（`credit_price`）最小值从 1 改为 0
3. Model 层 `sell_config.price.min` 从 `0.99` 改为 `0`
4. Model 层 `sell_config.credit_price.min` 从 `1` 改为 `0`
5. Controller 层 DSL 验证范围从 `number:100-999999!` 改为 `number:0-999999!`
6. **修复已知 Bug**: `updateData` 中 `form.price &&` 使用 truthy 判断，导致 `price=0` 时不写入数据库，必须改为 `form.price !== undefined`

**前提说明**:

> `price=0` 的业务语义（免费购买 vs 跳过支付）由 payment 微服务侧决定。本次仅修改 chat 服务侧的验证和存储逻辑，payment 侧另行处理。

**验收标准**:

- [ ] `price=0` 时 DSL 验证通过
- [ ] `credit_price=0` 时 DSL 验证通过
- [ ] `price=0` 成功写入数据库（不被 truthy 判断跳过）
- [ ] `credit_price=0` 成功写入数据库
- [ ] MongoDB Schema 的 `min` 约束允许 0

---

### REQ-3: 行程广场列表（免登录 + 城市筛选 + 搜索）

**业务目标**: 提供一个公开的行程广场列表接口，让任何人（含未登录访客）都能浏览和搜索已上架的行程。

**功能需求**:

1. 新增 `GET /open/trip_square/list` 接口，无需登录鉴权
2. 接口放在 `/open/*` 路由组下（使用 `openRequestValid` 中间件）
3. 查询条件：
   - 固定过滤：`del_flag=0` + `sell_config.enabled=true` + `sell_config.listed_in_square=true`
   - 可选参数 `keyword`：模糊匹配 `trip_name`（标题）和 `destination`（目的地）
   - 可选参数 `city`：模糊匹配 `destination` 字段（`$regex`, 忽略大小写）
4. 支持分页：`page`（默认 1）、`page_size`（默认 20）
5. 返回字段：`_id`, `trip_name`, `trip_cover`, `trip_brief`, `destination`, `start_date`, `end_date`, `like_count`, `sell_config.price`, `sell_config.payment_type`, `sell_config.credit_price`, `created_at`
6. 暂不返回卖家信息（后续可追加）
7. 暂不返回当前用户点赞状态（免登录接口无 user_id）

**数据源说明**:

- 查询 `trips` 集合（`app/model/trip.ts`）
- `destination` 字段为字符串类型（如 "Tokyo, Japan"），使用正则模糊匹配

**验收标准**:

- [ ] `GET /open/trip_square/list` 无需登录即可访问
- [ ] 只返回 `enabled=true` 且 `listed_in_square=true` 且 `del_flag=0` 的行程
- [ ] `keyword` 参数能同时匹配标题和目的地
- [ ] `city` 参数能过滤目的地城市（模糊匹配，不区分大小写）
- [ ] 分页正常工作，返回 `total` 总数
- [ ] 空数据时返回空数组而非报错

### REQ-4: 行程点赞功能

**业务目标**: 允许已登录用户对广场中的行程点赞/取消点赞，为后续排序提供数据支撑，同时提升用户参与感。

**功能需求**:

1. 新建 `trip_likes` 集合，记录点赞关系（`trip_id` + `user_id`）
2. 联合唯一索引 `{ trip_id: 1, user_id: 1 }` 防止重复点赞
3. `trips` 集合顶层新增 `like_count` 冗余字段（默认 0），用于排序加速
4. 点赞时 `like_count` 使用 `$inc: +1` 原子操作
5. 取消点赞时 `like_count` 使用 `$inc: -1`，并保护不低于 0
6. 提供三个接口（均需登录，放在 `/home/*` 路由组）：
   - `POST /home/trip_like/:tripId` — 点赞
   - `DELETE /home/trip_like/:tripId` — 取消点赞
   - `GET /home/trip_like/:tripId/status` — 查询当前用户是否已点赞
7. 点赞前校验：行程必须存在、未删除、已上架广场

**验收标准**:

- [ ] 首次点赞成功，`like_count` +1
- [ ] 重复点赞幂等处理（不报错，不重复计数）
- [ ] 取消点赞成功，`like_count` -1
- [ ] 未点赞时取消点赞，幂等处理
- [ ] `like_count` 不会变为负数
- [ ] 查询点赞状态返回 `{ liked: true/false }`
- [ ] 未登录时调用点赞接口返回 401

---

### REQ-5: 广场搜索复合排序规则

**业务目标**: 让广场列表搜索结果按"城市匹配优先 → 标题匹配次之 → 点赞量 → 发布时间"的复合规则排序，提升搜索体验。

**功能需求**:

1. 排序规则定义：

   | 场景 | 排序规则 |
   |------|---------|
   | 无关键词 | `like_count DESC` → `created_at DESC` |
   | 关键词同时命中标题和城市 | 匹配评分 3 → `like_count DESC` → `created_at DESC` |
   | 关键词只命中城市 | 匹配评分 2 → `like_count DESC` → `created_at DESC` |
   | 关键词只命中标题 | 匹配评分 1 → `like_count DESC` → `created_at DESC` |

2. 匹配评分计算规则：
   - 城市 + 标题都命中 → `score = 3`
   - 只命中城市 → `score = 2`（城市优先于标题）
   - 只命中标题 → `score = 1`
   - 无匹配 → `score = 0`（过滤掉）

3. 使用 MongoDB Aggregation Pipeline 实现（`$addFields` + `$switch` + `$sort`）

**验收标准**:

- [ ] 无关键词时按点赞量降序 → 时间降序排列
- [ ] 有关键词时，城市命中的结果排在标题命中之前
- [ ] 同时命中城市和标题的结果排在最前
- [ ] 同匹配级别内按点赞量降序排列
- [ ] 有关键词时不返回未匹配的结果

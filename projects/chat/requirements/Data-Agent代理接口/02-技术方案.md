## 一、方案概述

在 chat 服务的 `home` 路由组（`userJwt + singleLogin` 鉴权）新增一个代理接口，
直接将请求参数透传给 Data Agent 服务，并原样返回响应。

**核心设计原则**：最薄代理层，不做数据转换，不引入新模型/新中间件。

---

## 二、接口设计

### 提交链接解析

| 项目 | 内容 |
|------|------|
| **方法** | `POST` |
| **路径** | `/home/data_agent/link/submit` |
| **鉴权** | `userJwt + singleLogin`（basic） |
| **请求体** | `{ "link": "https://www.xiaohongshu.com/explore/xxxxxx" }` |
| **透传目标** | `POST {data_agent.url}/v1/link/submit` |
| **响应** | 原样透传 Data Agent 响应 |

请求体字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `link` | string | ✅ | 分享链接（小红书等） |

## 四、核心代码设计

### 4.1 Controller — `app/controller/home/DataAgentController.ts`

参数校验内联在 Controller，调用逻辑直接在 Controller 中完成（无需单独 Service/Validator 文件）。

```typescript
import { Controller } from 'egg';
import * as _ from 'lodash';

export default class DataAgentController extends Controller {
    logPrefix = '[DataAgentController]';

    // POST /home/data_agent/link/submit
    public async submitLink() {
        const { ctx } = this;

        // 参数校验：link 必填字符串
        ctx.validate(
            {
                link: { type: 'string', required: true, max: 2000, trim: true },
            },
            ctx.request.body,
        );

        const body = _.pick(ctx.request.body, ['link']);
        const microService: any = _.get(ctx.app.config.custom, 'microservices.data_agent');
        const apiUrl = `${microService.url}/v1/link/submit`;

        ctx.logger.info(`${this.logPrefix} submitLink apiUrl.`, apiUrl);

        const result = await ctx.http.postJSON(
            `${this.logPrefix}.submitLink`,
            apiUrl,
            body,
            { timeoutMs: 60000, retries: 0 },
        );

        // 原样透传 Data Agent 响应（{ code, message, data }）
        ctx.body = result;
    }
}
```

### 4.2 路由 — `app/routes/home/data_agent.ts`

```typescript
import { Application } from 'egg';
import { RouterGroup } from 'egg-router-group';

export default (app: Application, groupRouter: RouterGroup) => {
    groupRouter.group({
        name: 'Data Agent 代理',
        prefix: '/data_agent',
    }, (g: RouterGroup) => {
        const ctrl = app.controller.home.dataAgentController;
        g.post('/link/submit', ctrl.submitLink);
    });
};
```

### 4.3 注册路由 — `app/routes/home/index.ts`（修改部分）


## 六、时序图

```
前端/后端
  │
  │  POST /home/data_agent/link/submit  { link }
  ▼
chat service（userJwt + singleLogin 鉴权）
  │  DataAgentValidator.submitLink()   — 校验 link 必填/URL格式
  │  DataAgentService.submitLink()
  │
  │  POST {microservices.data_agent.url}/v1/link/submit  { link }
  ▼
Data Agent 服务（Nacos 注入地址）
  │
  │  { code: 200, data: { uuid, platform, biz_id, status, ... } }
  ▼
chat service → ctx.success(data)
  ▼
前端/后端（拿到 uuid，可再自行调用 Data Agent GET 接口查询完整结果）
```

## 七、风险与注意事项

| 风险 | 处理方式 |
|------|---------|
| Data Agent 超时（LLM 调用较慢） | `timeoutMs: 60000`，不重试（`retries: 0`） |
| `data_agent.url` 未在 Nacos 配置 | `_.get` 返回 undefined，`${undefined}/v1/...` 会导致 curl 报错，由 httpHelper 异常捕获返回 5xx |
| Data Agent 返回 `code: 500` | httpHelper 层面请求成功（HTTP 200），原始 JSON 透传给前端；前端按 `code` 字段判断 |

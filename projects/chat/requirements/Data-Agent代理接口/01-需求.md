---

## 一、需求背景

Data Agent 服务提供链接提取与内容解析能力（如小红书笔记解析为结构化行程/美食 JSON）。
前端和后端需要通过 chat 服务的**已鉴权接口**访问该能力，而不是直接调用 Data Agent，原因：

- 统一鉴权入口，避免 Data Agent 直接暴露给外部
- 利用 chat 服务已有的 JWT 鉴权体系（userJwt + singleLogin）
- 便于后续在 chat 侧做日志、限流、监控等横切逻辑

---

## 二、需求内容

### 功能需求

- [ ] **提交解析链接**：`POST /home/data_agent/link/submit`
  - 接收前端传入的 `link` 参数（必填，URL 格式）
  - 透传请求体到 Data Agent `POST /v1/link/submit`
  - 将 Data Agent 响应原样返回给客户端

- [ ] **鉴权**：需通过 `userJwt + singleLogin`（basic 级别）鉴权

### 非功能需求

- 超时：60s（Data Agent 内部调用 LLM，耗时较长）
- 重试：不重试（`retries: 0`）
- 错误处理：上游错误透传，不做额外转换

---

## 三、验收标准

| 验收项 | 标准 |
|--------|------|
| 未登录访问 | 返回 401 |
| POST 提交合法链接 | 返回 Data Agent 成功响应（含 `uuid`、`platform`、`status` 等字段） |
| `link` 字段缺失 | 返回参数校验错误 |
| Data Agent 异常 | 透传原始响应或返回 `EXTERNAL_API_FAILED` |

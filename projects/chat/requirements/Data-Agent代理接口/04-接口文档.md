## 📝 接口详情

### 1. 提交链接解析

#### 基本信息

| 项目 | 内容 |
|------|------|
| **URL** | `POST /home/data_agent/link/submit` |
| **Content-Type** | `application/json` |
| **鉴权** | 需要登录 |
| **超时** | 60s（内部调用 LLM，耗时较长，请勿提前断开连接） |

#### 请求参数

| 参数 | 位置 | 类型 | 必填 | 最大长度 | 说明 |
|------|------|------|------|---------|------|
| `link` | body | string | ✅ | 2000 | 分享链接，目前主要支持小红书笔记链接 |

#### 请求示例

```http
POST /home/data_agent/link/submit HTTP/1.1
Host: sit-aita.guardforceai.com
Authorization: Bearer eyJhbGciOiJIUzI1NiIs...
Content-Type: application/json

{
  "link": "https://www.xiaohongshu.com/explore/65f1f9a4000000000102abcd"
}
```

#### 成功响应（HTTP 200）

> ⚠️ **注意**：chat 服务直接透传 Data Agent 的原始响应，`code` 字段为 **Data Agent 业务码**（200/500），非 chat 标准码（0/-1）。

```json
{
  "code": 200,
  "message": "success",
  "data": {
    "uuid": "f5b9e88c-9d1a-4a0e-9e8d-6e8e9f5b9e88",
    "platform": "xiaohongshu",
    "biz_id": "65f1f9a4000000000102abcd",
    "original_link": "https://www.xiaohongshu.com/explore/65f1f9a4000000000102abcd",
    "status": "DONE"
  }
}
```

#### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | int | Data Agent 业务码：`200` 成功，`500` 内部错误 |
| `message` | string | 描述信息 |
| `data` | object \| null | 任务结果，失败时为 `null` |
| `data.uuid` | string | 任务唯一 ID，后续可用于查询完整结果 |
| `data.platform` | string | 识别出的平台，如 `"xiaohongshu"` |
| `data.biz_id` | string \| null | 平台内业务 ID（如小红书 `note_id`） |
| `data.original_link` | string | 原始链接 |
| `data.status` | string | 任务状态：`DONE` / `FAILED`（同步模式一般直接返回 `DONE`） |

#### 错误响应

**解析失败（HTTP 200，Data Agent code 500）**

```json
{
  "code": 500,
  "message": "some error message from backend",
  "data": null
}
```

**鉴权失败（HTTP 401）**

```json
{
  "code": 401,
  "message": "Unauthorized"
}
```

**参数校验失败（HTTP 422）**

```json
{
  "code": 422,
  "message": "Validation Failed",
  "errors": [
    { "field": "link", "message": "required" }
  ]
}
```

## 典型调用流程

```
1. 用户粘贴分享链接
   ↓
2. POST /home/data_agent/link/submit → 拿到 uuid（等待约 3~30s）
   ↓
3. 判断 data.status
   - DONE   → 用 uuid 调用 Data Agent GET /v1/link/result/{uuid} 获取 extracted_json
   - FAILED → 读取 error_message 提示用户重试
```

> **注意**：获取完整结果（`extracted_json`）需直接请求 Data Agent，chat 服务未暴露该查询接口。  
> Data Agent GET 接口：`GET {data_agent_base_url}/v1/link/result/{uuid}`  
> Data Agent API 文档详见附件 `api.md`。

---

## 错误处理建议

| 情况 | 处理方式 |
|------|---------|
| `code: 200` + `status: DONE` | 正常，继续获取结果 |
| `code: 200` + `status: FAILED` | 解析失败，提示用户"内容解析失败，请重试" |
| `code: 500` | Data Agent 内部错误（第三方接口或 LLM 异常），提示用户稍后重试 |
| HTTP 超时（> 60s） | 极少发生，可提示用户网络异常并重试 |
| HTTP 401 | token 过期，引导重新登录 |

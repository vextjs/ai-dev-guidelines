## 一、概览

### 1.1 接口列表

| # | 方法 | 路径 | 鉴权 | 分页 | 说明 |
|:-:|:----:|------|:----:|:---:|------|
| 1 | POST | `/home/blog` | JWT | — | 创建文章 |
| 2 | GET | `/open/blog` | 无 | 游标分页 | 公开文章列表 |
| 3 | GET | `/open/blog/:blogId` | 无 | — | 文章详情 |
| 4 | PUT | `/home/blog/:blogId` | JWT | — | 更新文章 |
| 5 | DELETE | `/home/blog/:blogId` | JWT | — | 软删除文章 |
| 6 | GET | `/home/blog/admin/list` | dbToken | 游标分页 | 管理员文章列表 |
| 7 | PUT | `/home/blog/admin/:blogId/status` | dbToken | — | 管理员上下架 |

### 1.2 通用响应格式

所有接口均遵循统一响应结构：

**成功**：
```json
{
  "code": 0,
  "data": { ... }
}
```

**失败**：
```json
{
  "code": 40301,
  "message": "文章不存在"
}
```

> `code: 0` 表示成功，`code !== 0` 表示失败，失败时读取 `message` 字段展示给用户。

## 二、鉴权说明

### 2.1 用户 JWT（User 端接口）

用于 `POST /home/blog`、`PUT /home/blog/:blogId`、`DELETE /home/blog/:blogId`。

**请求头**：

```
Authorization: Bearer <user_login_token>
```

> Token 通过用户登录接口获取，过期后需重新登录刷新。

**缺少 / 过期时响应**：

```json
{
  "code": 401,
  "message": "Unauthorized"
}
```

### 2.2 无需鉴权（Open 端接口）

`GET /open/blog` 和 `GET /open/blog/:blogId` 无需携带任何 Token，任何客户端均可直接访问。

### 2.3 管理后台 Token（Admin 端接口）

用于 `GET /home/blog/admin/list`、`PUT /home/blog/admin/:blogId/status`。

**请求头**：

```
x-action-token: <db_token>
```

> dbToken 由后台管理平台颁发，仅供内部管理系统使用，不对 C 端暴露。

**缺少 / 无效时响应**：

```json
{
  "code": 401,
  "message": "Unauthorized"
}
```

## 三、接口详情

### 3.1 创建文章

```
POST /home/blog
Authorization: Bearer <token>
Content-Type: application/json
```

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `title` | string | ✅ | 文章标题，1-200 字符 |
| `content` | string | ✅ | 文章正文，无长度限制，支持 Markdown |
| `summary` | string | — | 文章摘要，0-500 字符；不传则自动截取正文前 200 字符 |
| `cover_image` | string | — | 封面图 URL |
| `tags` | string[] | — | 标签数组，最多 10 个 |
| `status` | string | — | `"draft"`（默认）或 `"published"` |

**请求示例**：

```json
{
  "title": "我的第一篇博客",
  "content": "# 正文标题\n\n这是正文内容，支持 Markdown 格式。",
  "summary": "这是文章摘要",
  "cover_image": "https://example.com/cover.jpg",
  "tags": ["旅行", "攻略"],
  "status": "draft"
}
```

**响应（200）**：

```json
{
  "code": 0,
  "data": {
    "_id": "65f1a2b3c4d5e6f7a8b9c0d1",
    "title": "我的第一篇博客",
    "content": "# 正文标题\n\n这是正文内容，支持 Markdown 格式。",
    "summary": "这是文章摘要",
    "cover_image": "https://example.com/cover.jpg",
    "tags": ["旅行", "攻略"],
    "status": "draft",
    "author_id": "65f000000000000000000001",
    "view_count": 0,
    "del_flag": 0,
    "published_at": null,
    "created_at": "2026-03-06T10:00:00.000Z",
    "updated_at": "2026-03-06T10:00:00.000Z"
  }
}
```

> `published_at`：仅当 `status: "published"` 时才有值，草稿为 `null`。

**错误码**：

| code | 说明 |
|:----:|------|
| 40303 | 标签数量超过 10 个 |
| 50301 | 服务器创建失败（系统错误） |

### 3.2 公开文章列表（游标分页）

```
GET /open/blog
```

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `limit` | int | — | 每页条数，范围 1-50，默认 20 |
| `after` | string | — | 游标值，获取该游标**之后**的数据（下一页）|
| `before` | string | — | 游标值，获取该游标**之前**的数据（上一页）|
| `page` | int | — | 页码（1 起），服务端跳页模式，范围 1-10000 |
| `keyword` | string | — | 关键词，同时模糊搜索标题和摘要，最长 100 字符 |
| `tag` | string | — | 标签精确过滤，最长 20 字符 |
| `author_id` | string | — | 按作者 ID 过滤（ObjectId 格式） |

> **注意**：`page` 与 `after`/`before` **互斥**，不能同时传入；首次请求三者都不传（等同于 `page=1`）。

**请求示例**：

```
# 首次请求
GET /open/blog?limit=20

# 下一页（游标翻页）
GET /open/blog?limit=20&after=eyJzIjp7InB1Ymxpc2hlZF9hdCI6LTF9fQ==

# 跳到第 5 页（服务端跳页）
GET /open/blog?limit=20&page=5

# 关键词搜索
GET /open/blog?keyword=旅行&limit=10

# 标签过滤
GET /open/blog?tag=攻略&limit=10
```

**响应（200）**：

```json
{
  "code": 0,
  "data": {
    "items": [
      {
        "_id": "65f1a2b3c4d5e6f7a8b9c0d1",
        "lang": "zh",
        "title": "我的第一篇博客",
        "summary": "这是摘要...",
        "cover_image": "https://example.com/cover.jpg",
        "tags": ["旅行", "攻略"],
        "status": "published",
        "author_id": "65f000000000000000000001",
        "view_count": 128,
        "published_at": "2026-03-06T10:00:00.000Z",
        "created_at": "2026-03-06T10:00:00.000Z"
      }
    ],
    "pageInfo": {
      "hasNext": true,
      "hasPrev": false,
      "startCursor": "eyJzIjp7InB1Ymxpc2hlZF9hdCI6LTF9LCJhIjp7fX0=",
      "endCursor": "eyJzIjp7InB1Ymxpc2hlZF9hdCI6LTF9LCJhIjp7fX0="
    },
    "totals": {
      "mode": "async",
      "total": 100,
      "totalPages": 5,
      "ts": 1709625600000
    },
    "currentPage": 1
  }
}
```

**响应字段说明**：

| 字段 | 说明 |
|------|------|
| `items` | 当页文章列表，**不含 `content` 字段**（详情才有） |
| `items[].lang` | 语言标识（`zh` / `en` / `hk`），与请求头 `language` 对应 |
| `items[].summary` | 摘要；创建时未填写则自动取正文**第一段落**（按 `\n\n` 分割，最多 500 字符） |
| `items[].cover_image` | 封面图 URL；创建时未填写则自动从正文提取**第一张图片**（Markdown `![](url)` 或 HTML `<img src>`），无图时为空字符串 |
| `pageInfo.hasNext` | 是否有下一页，用于控制"加载更多"按钮显示 |
| `pageInfo.hasPrev` | 是否有上一页 |
| `pageInfo.startCursor` | 当页第一条的游标值，用于"上一页"请求的 `before` 参数 |
| `pageInfo.endCursor` | 当页最后一条的游标值，用于"下一页"请求的 `after` 参数 |
| `pageInfo.currentPage` | 当前页码（仅使用 `page` 参数跳页时返回） |
| `totals.total` | 总条数（async 模式下首次请求可能为 `null`，后续返回缓存值） |
| `totals.totalPages` | 总页数 |

**错误码**：

| code | 说明 |
|:----:|------|
| 50304 | 服务器查询失败（系统错误） |

### 3.3 文章详情


```
GET /open/blog/65f1a2b3c4d5e6f7a8b9c0d1
```

**响应（200）**：

```json
{
  "code": 0,
  "data": {
    "_id": "65f1a2b3c4d5e6f7a8b9c0d1",
    "lang": "zh",
    "title": "我的第一篇博客",
    "content": "# 正文标题\n\n这是完整正文内容...",
    "summary": "这是摘要...",
    "cover_image": "https://example.com/cover.jpg",
    "tags": ["旅行", "攻略"],
    "status": "published",
    "author_id": "65f000000000000000000001",
    "view_count": 129,
    "del_flag": 0,
    "published_at": "2026-03-06T10:00:00.000Z",
    "created_at": "2026-03-06T10:00:00.000Z",
    "updated_at": "2026-03-06T10:30:00.000Z"
  }
}
```

> **注意**：
> - 详情接口返回完整 `content` 字段（列表接口不返回）
> - 每次调用后 `view_count` 会异步 +1，响应中的值可能有短暂延迟（最终一致，非实时）
> - 草稿文章（`status: "draft"`）通过此接口访问时返回 `BLOG_NOT_FOUND`（40301）

**错误码**：

| code | 说明 |
|:----:|------|
| 40301 | 文章不存在、已删除或为草稿 |
| 50305 | 服务器查询失败（系统错误） |

### 3.4 更新文章

```
PUT /home/blog/:blogId
Authorization: Bearer <token>
Content-Type: application/json
```

**路径参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `blogId` | string | ✅ | 文章 ID（MongoDB ObjectId 格式） |

**请求体**（所有字段均可选，仅传入需要更新的字段）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `title` | string | 文章标题，1-200 字符 |
| `content` | string | 文章正文，无长度限制 |
| `summary` | string | 文章摘要，0-500 字符 |
| `cover_image` | string | 封面图 URL |
| `tags` | string[] | 标签数组，最多 10 个 |
| `status` | string | `"draft"` 或 `"published"` |

**请求示例**：

```json
{
  "title": "更新后的标题",
  "status": "published"
}
```

**响应（200）**：

```json
{
  "code": 0,
  "data": {
    "_id": "65f1a2b3c4d5e6f7a8b9c0d1",
    "title": "更新后的标题",
    "content": "...",
    "summary": "...",
    "cover_image": "https://example.com/cover.jpg",
    "tags": ["旅行"],
    "status": "published",
    "author_id": "65f000000000000000000001",
    "view_count": 0,
    "del_flag": 0,
    "published_at": "2026-03-06T11:00:00.000Z",
    "created_at": "2026-03-06T10:00:00.000Z",
    "updated_at": "2026-03-06T11:00:00.000Z"
  }
}
```

> **状态变更规则**：
> - `draft → published`：若为首次发布（`published_at` 为空），自动写入 `published_at`
> - `published → draft`：`published_at` 保留不清除
> - **仅文章作者可更新**，非作者操作返回 `BLOG_FORBIDDEN`（40302）

**错误码**：

| code | 说明 |
|:----:|------|
| 40301 | 文章不存在或已删除 |
| 40302 | 非文章作者，无权操作 |
| 40303 | 标签数量超过 10 个 |
| 50302 | 服务器更新失败（系统错误） |

### 3.5 删除文章（软删除）

```
DELETE /home/blog/:blogId
Authorization: Bearer <token>
```

**路径参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `blogId` | string | ✅ | 文章 ID（MongoDB ObjectId 格式） |

**请求示例**：

```
DELETE /home/blog/65f1a2b3c4d5e6f7a8b9c0d1
```

**响应（200）**：

```json
{
  "code": 0,
  "data": {
    "success": true
  }
}
```

> **说明**：
> - 为软删除，数据不会物理删除，`del_flag` 设为 1
> - **接口具有幂等性**：已删除的文章重复调用也返回 `success: true`，前端无需额外处理
> - **仅文章作者可删除**，非作者操作返回 `BLOG_FORBIDDEN`（40302）

**错误码**：

| code | 说明 |
|:----:|------|
| 40301 | 文章不存在 |
| 40302 | 非文章作者，无权操作 |
| 50303 | 服务器删除失败（系统错误） |

### 3.6 管理员文章列表（游标分页）

```
GET /home/blog/admin/list
x-action-token: <db_token>
```

**Query 参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `limit` | int | — | 每页条数，范围 1-100，默认 20 |
| `after` | string | — | 游标值，下一页 |
| `before` | string | — | 游标值，上一页 |
| `page` | int | — | 页码（1 起），服务端跳页模式，范围 1-10000 |
| `status` | string | — | 按状态过滤：`"draft"` / `"published"`，不传则返回全部 |
| `del_flag` | int | — | 按删除状态过滤：`0`=正常 / `1`=已删除，不传则返回全部 |
| `author_id` | string | — | 按作者 ID 过滤（ObjectId 格式） |
| `keyword` | string | — | 标题关键词搜索，最长 100 字符 |

**请求示例**：

```
# 全量列表（含草稿和已删除）
GET /home/blog/admin/list?limit=20

# 只看已发布
GET /home/blog/admin/list?status=published

# 跳到第 3 页（服务端跳页）
GET /home/blog/admin/list?limit=20&page=3

# 关键词搜索草稿
GET /home/blog/admin/list?status=draft&keyword=测试

# 已删除文章
GET /home/blog/admin/list?del_flag=1
```

**响应（200）**：

```json
{
  "code": 0,
  "data": {
    "items": [
      {
        "_id": "65f1a2b3c4d5e6f7a8b9c0d1",
        "title": "我的第一篇博客",
        "summary": "这是摘要...",
        "tags": ["旅行"],
        "status": "draft",
        "author_id": "65f000000000000000000001",
        "view_count": 0,
        "del_flag": 0,
        "published_at": null,
        "deleted_at": null,
        "created_at": "2026-03-06T10:00:00.000Z",
        "updated_at": "2026-03-06T10:00:00.000Z"
      }
    ],
    "pageInfo": {
      "hasNext": true,
      "hasPrev": false,
      "startCursor": "...",
      "endCursor": "..."
    },
    "totals": {
      "mode": "sync",
      "total": 200,
      "totalPages": 10,
      "ts": 1709625600000
    }
  }
}
```

> **与公开列表的差异**：
> - 返回全量文章（含草稿、已删除），字段包含 `del_flag`、`deleted_at`、`updated_at`
> - `totals` 使用 `sync` 模式，`total` 始终有准确值（不会为 `null`）
> - 按 `created_at` 降序排列（公开列表按 `published_at`）
> - 不启用缓存，每次返回实时数据

**错误码**：

| code | 说明 |
|:----:|------|
| 50304 | 服务器查询失败（系统错误） |

### 3.7 管理员上下架

```
PUT /home/blog/admin/:blogId/status
x-action-token: <db_token>
Content-Type: application/json
```

**路径参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `blogId` | string | ✅ | 文章 ID（MongoDB ObjectId 格式） |

**请求体**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|:----:|------|
| `status` | string | ✅ | 目标状态：`"published"`（上架）或 `"draft"`（下架） |

**请求示例（上架）**：

```json
{
  "status": "published"
}
```

**请求示例（下架）**：

```json
{
  "status": "draft"
}
```

**响应（200）**：

```json
{
  "code": 0,
  "data": {
    "success": true
  }
}
```

> **说明**：
> - 管理员可操作**任意文章**，不受作者权限限制
> - `draft → published` 且该文章从未发布过时，自动写入 `published_at`

**错误码**：

| code | 说明 |
|:----:|------|
| 40301 | 文章不存在 |

## 四、游标分页指南

### 4.1 原理

游标分页（Cursor-based Pagination）基于排序字段的"位置"而非"偏移量"进行翻页，性能稳定，不受数据量增长影响（O(1)）。

- **公开列表**：按 `published_at` 降序排列
- **管理员列表**：按 `created_at` 降序排列
- 游标值由服务端生成，前端只需透传，不需解析内容

### 4.2.1 跳页方案对比

服务端支持两种分页模式，可按场景选用：

#### 方案 A：服务端 `page` 跳页（✅ 推荐）

直接传 `page` 参数，服务端利用 monsqlize 内置的**书签缓存机制**实现跳页，前端无需额外逻辑。

**原理**：服务端每 10 页自动缓存一个书签（游标快照），跳页时先定位到最近的书签，再向前连续翻页至目标页。书签缓存 TTL 6 小时，同一查询条件下重复跳页效率极高。

```javascript
// 跳到第 5 页 — 一个请求搞定
const res = await get('/open/blog', { limit: 10, page: 5 });
// res.data.pageInfo.currentPage === 5

// 跳到第 12 页
const res2 = await get('/open/blog', { limit: 10, page: 12 });
// 服务端自动利用第 10 页书签 → 仅需 2 次内部翻页

// 回到第 1 页
const res3 = await get('/open/blog', { limit: 10, page: 1 });
```

**前端页码导航完整示例**：

```javascript
const limit = 10;
let currentPage = 1;
let totalPages = null;

// 加载指定页
async function loadPage(page) {
  const res = await get('/open/blog', { limit, page });
  currentPage = res.data.pageInfo.currentPage || page;
  totalPages = res.data.totals?.totalPages ?? totalPages;
  updateView(res.data);
  renderPagination(currentPage, totalPages);
}

// 页码按钮点击
function onPageClick(page) {
  loadPage(page);
}

// 首次加载
await loadPage(1);
```

**`page` 与 `after`/`before` 互斥**：

| 参数组合 | 行为 |
|---------|------|
| 无参数 | 返回首页（等同 `page=1`） |
| `after=xxx` | 游标翻页 — 下一页 |
| `before=xxx` | 游标翻页 — 上一页 |
| `page=5` | 跳页 — 直接返回第 5 页 |
| `page=3&after=xxx` | ❌ 报错（互斥） |

**服务端跳页限制**：

| 限制项 | 默认值 | 说明 |
|--------|:------:|------|
| `maxHops` | 20 | 单次跳页最大内部翻页次数；超出报 `JUMP_TOO_FAR` 错误 |
| `step` | 10 | 书签间隔（每 10 页缓存一个书签） |
| `page` 上限 | 10000 | 超过 10000 页不记录书签 |
| 书签 TTL | 6 小时 | 书签过期后首次跳页稍慢（需重建书签链） |

> ⚠️ 跳页幅度超过 `maxHops`（默认 20 × limit 条记录的跨度）会返回 `JUMP_TOO_FAR` 错误。在无书签缓存的冷启动场景下，跳到第 21 页以上可能触发此限制。连续使用后书签自动建立，后续跳页不受此影响。

#### 方案 B：客户端游标缓存跳页（备选）

如需在前端完全控制分页逻辑（例如离线缓存场景），也可以通过缓存每页游标来实现跳页：

```javascript
// 游标缓存：下标即页码（从 0 开始），值为该页的 after 游标
const cursors = [null];  // cursors[0] = null 表示第 1 页
let currentPage = 0;
const limit = 10;

async function jumpToPage(targetPage) {
  if (targetPage < 0) return;

  // 目标页游标已缓存 → 直接跳
  if (cursors[targetPage] !== undefined) {
    const res = await get('/open/blog', {
      limit,
      after: cursors[targetPage] ?? undefined,
    });
    currentPage = targetPage;
    updateView(res.data);
    return;
  }

  // 未缓存 → 从最后已知游标连续翻页
  let page = cursors.length - 1;
  let cursor = cursors[page];
  while (page < targetPage) {
    const res = await get('/open/blog', { limit, after: cursor ?? undefined });
    page++;
    cursor = res.data.pageInfo.endCursor;
    cursors[page] = cursor;
    if (!res.data.pageInfo.hasNext) break;
  }

  const finalRes = await get('/open/blog', {
    limit,
    after: cursors[targetPage] ?? undefined,
  });
  currentPage = targetPage;
  updateView(finalRes.data);
}
```

> ⚠️ 客户端方案在数据变更后游标可能失效，需在刷新或路由切换时清空缓存。

#### 场景推荐

| 场景 | 推荐方案 | 说明 |
|------|:--------:|------|
| PC 端页码导航 | **A（服务端 page）** | 一个请求直达目标页，最简单 |
| 管理后台分页 | **A（服务端 page）** | 支持跳页 + sync 模式准确总数 |
| 无限滚动（移动端） | 游标 `after` | 只维护 `endCursor`，每次追加 |
| 上/下翻页（无页码） | 游标 `after`/`before` | 不需要跳页，连续翻页最高效 |
| 离线/PWA 场景 | **B（客户端缓存）** | 前端控制缓存，断网可用 |

### 4.3 前端状态管理建议

```javascript
// 推荐状态结构
const state = {
  items: [],
  pageInfo: {
    hasNext: false,
    hasPrev: false,
    startCursor: null,
    endCursor: null,
  },
  loading: false,
};

// 首次加载
async function loadFirst(limit = 20) {
  const res = await get('/open/blog', { limit });
  state.items = res.data.items;
  state.pageInfo = res.data.pageInfo;
}

// 加载下一页
async function loadNext() {
  if (!state.pageInfo.hasNext) return;
  const res = await get('/open/blog', {
    limit: 20,
    after: state.pageInfo.endCursor,
  });
  state.items = [...state.items, ...res.data.items]; // 追加（无限滚动）
  // 或 state.items = res.data.items;               // 替换（分页切换）
  state.pageInfo = res.data.pageInfo;
}

// 加载上一页
async function loadPrev() {
  if (!state.pageInfo.hasPrev) return;
  const res = await get('/open/blog', {
    limit: 20,
    before: state.pageInfo.startCursor,
  });
  state.items = res.data.items;
  state.pageInfo = res.data.pageInfo;
}
```

## 六、多语言切换

本系统使用**自定义 `language` 请求头**（非标准 `Accept-Language`）控制语言，影响两个层面：

1. **文章内容语言**：返回对应语言版本的文章（`title`/`content`/`summary` 均为对应语言）
2. **错误消息语言**：接口错误时 `desc` 字段返回对应语言的提示文字

> ⚠️ **注意**：请勿使用标准 HTTP `Accept-Language` 头，服务端不解析该字段。必须使用自定义 `language` 头。

---

### 6.1 语言标识对照

| `language` 值 | 语言 | 说明 |
|:---:|------|------|
| `en`（默认） | 英文 | 不传 `language` 时的默认语言 |
| `zh` | 简体中文 | — |
| `hk` | 繁体中文 | — |

---

### 6.2 文章内容多语言机制

同一篇文章的中/英/繁体版本在数据库中为**独立记录**，通过 `lang` 字段区分（与项目 `feature_modules` 等模块一致）。

- **查询**：Service 层自动从 `language` 请求头读取语言，过滤对应 `lang` 的文章记录，前端无需额外传参
- **创建**：POST 请求时 `lang` 由后端从请求头自动写入，每次调用创建该语言的独立记录
- **独立性**：三语言版本的 `status`、`del_flag`、`view_count` 互相独立，可单独上下架

## 七、注意事项

### 7.1 view_count 浏览量

- 每次调用详情接口（`GET /open/blog/:blogId`）后，`view_count` 会**异步 +1**
- 异步更新存在短暂延迟（最多 120 秒，与缓存 TTL 一致），响应中的值不是实时最新值
- 这是设计行为（最终一致），无需前端额外处理

### 7.2 缓存说明

| 接口 | 缓存 TTL | 说明 |
|------|:-------:|------|
| `GET /open/blog`（列表） | 60 秒 | 相同查询参数命中缓存 |
| `GET /open/blog/:blogId`（详情） | 120 秒 | 相同 blogId 命中缓存 |
| `GET /home/blog/admin/list`（管理列表） | 无缓存 | 实时数据 |

> 写操作（创建/更新/删除/上下架）成功后，服务端会自动清除相关缓存，前端无需特殊处理。

### 7.3 文章状态枚举

| 值 | 说明 | 前端展示建议 |
|:--:|------|------------|
| `"draft"` | 草稿，未发布 | 显示"草稿"标签，不对外展示 |
| `"published"` | 已发布，公开可见 | 正常展示 |

### 7.4 软删除说明

文章删除后 `del_flag = 1`，公开接口不再返回该文章（返回 `BLOG_NOT_FOUND`）。管理员接口可通过 `del_flag=1` 参数查询已删除文章，但无法在此版本恢复删除。

### 7.5 列表与详情的字段差异

| 字段 | 公开列表 | 文章详情 | 管理列表 |
|------|:-------:|:-------:|:-------:|
| `_id` | ✅ | ✅ | ✅ |
| `title` | ✅ | ✅ | ✅ |
| `content` | ❌ | ✅ | ❌ |
| `summary` | ✅ | ✅ | ✅ |
| `cover_image` | ✅ | ✅ | — |
| `tags` | ✅ | ✅ | ✅ |
| `status` | ✅ | ✅ | ✅ |
| `author_id` | ✅ | ✅ | ✅ |
| `view_count` | ✅ | ✅ | ✅ |
| `del_flag` | ❌ | ✅ | ✅ |
| `published_at` | ✅ | ✅ | ✅ |

# Blog 博客功能技术方案

## 零、需求概览

> **上游文档**: [01-需求定义.md](./01-需求定义.md)
> **本次修正**: 基于 vscode-copilot 产出的 v1 方案，根据用户审查意见进行三项修正
> **修正人**: zed-copilot（2026-03-06）

### 业务目标

在 chat 服务内新增「Blog 博客」模块，支持用户创作和发布博客文章，提供完整的增删查改（CRUD）接口。

### 需求清单

| REQ | 名称 | 接口 | 优先级 |
|:---:|------|------|:------:|
| REQ-1 | 文章创建 | `POST /home/blog` | P0 |
| REQ-2 | 文章列表（游标分页） | `GET /open/blog` | P0 |
| REQ-3 | 文章详情 | `GET /open/blog/:blogId` | P0 |
| REQ-4 | 文章更新 | `PUT /home/blog/:blogId` | P0 |
| REQ-5 | 文章删除（软删除） | `DELETE /home/blog/:blogId` | P0 |
| REQ-6 | 管理后台（列表 + 上下架） | `GET /home/blog/admin/list` + `PUT /home/blog/admin/:blogId/status` | P1 |

### 关键约束

| 约束 | 说明 |
|------|------|
| 参数校验 | Controller 内联 `ctx.dsl()` + `ctx.validateDsl()`，不创建独立 Validator 文件 |
| 分页查询 | 使用 monSQLize `findPage` 游标分页，响应格式遵循 monSQLize 原生结构 |
| 错误码 | DSL 子目录 `config/dsl/blog/{zh-CN,en-US,zh-HK}.ts`，`{ code, message }` 对象格式 |
| code 码段 | 业务 40301-40399，系统 50301-50399，禁止与其他模块重复 |
| 读写分离 | 查询用 `ctx.msq`，写入用 `ctx.model`，写操作后调 `invalidate()` |
| 多语言模式 | 同一篇文章按 `lang`（`zh`/`en`/`hk`）存为独立记录；查询时按请求头 `language` 自动过滤；与项目 `feature_modules`/`trip_destination` 模式一致 |
| 错误处理 | `ctx.dsl.error.throw()` / `ctx.dsl.error.create()`，禁止 `throw new Error()` |
| 成功响应 | `ctx.success(data)`，禁止 `ctx.body = {...}` |

### v1 → v2 修正对照

| # | v1 方案 | v2 修正 | 原因 |
|:-:|---------|---------|------|
| 1 | 独立 Validator 文件（`app/validator/open/blog_validator.ts` 等） | Controller 内联 `ctx.dsl()` + `ctx.validateDsl()` | 项目新代码约定在 Controller 内校验，不创建独立 Validator |
| 2 | 追加到 `config/dsl/zh.ts` 扁平文件 + 简单字符串格式 `'KEY': '消息'` | 子目录 `config/dsl/blog/` + `{ code, message }` 对象格式 | 参考行程广场方案 + 规范要求 code 码不重复 |
| 3 | `find()` + `count()` 传统分页（`page` + `page_size` → `{ total, list }`） | `findPage()` 游标分页（`limit` + `after`/`before` → `{ items, pageInfo, totals }`） | 用户明确要求游标分页，使用 monSQLize findPage 原生响应格式 |

### v2 → v3 多语言修正对照

| # | v2 方案 | v3 修正 | 原因 |
|:-:|---------|---------|------|
| 1 | `blogs` 集合无语言字段，所有语言共用同一条记录 | 新增 `lang` 字段（`'zh'`/`'en'`/`'hk'`），同一文章按语言存为独立记录 | 遗漏多语言需求；与项目 `feature_modules`/`trip_destination` 既有模式对齐（方案 B2） |
| 2 | 列表/详情查询不做语言过滤 | 查询时从 `ctx.i18n.getLanguage()` 读取语言，自动追加 `lang` 过滤条件 | 按请求头 `language`（zh/en/hk）返回对应语言文章 |
| 3 | 管理端列表无 `lang` 参数 | 管理端列表新增可选 `lang` query 参数，支持按语言过滤（不传则查全部语言） | 管理员需要能按语言维度管理文章 |
| 4 | 索引 `{ status, del_flag, published_at }` | 索引改为 `{ lang, status, del_flag, published_at }`，lang 置首位利用索引前缀 | 所有查询均携带 lang 条件，lang 在前可充分利用复合索引 |

## 一、架构设计

### 1.1 整体架构

```
┌──────────────────────────────────────────────────────────────────────┐
│                            Routes 路由层                              │
├──────────────────┬─────────────────────────┬─────────────────────────┤
│  /open/*         │  /home/*（用户鉴权）     │  /home/blog/admin/*     │
│  (无鉴权)        │  [userJwt, singleLogin]  │  (dbToken)              │
│  GET /blog       │  POST /blog              │  GET /blog/admin/list   │
│  GET /blog/:id   │  PUT /blog/:id           │  PUT /blog/admin/:id/   │
│                  │  DELETE /blog/:id        │      status             │
├──────────────────┴─────────────────────────┴─────────────────────────┤
│                      Controller 控制层（含内联参数校验）                  │
│  open/BlogController  │  home/BlogController  │  home/BlogAdminCtrl   │
│  ctx.dsl() + validateDsl() 内联校验 → 调用 Service → ctx.success()     │
├───────────────────────┴───────────────────────┴───────────────────────┤
│                            Service 业务层                              │
│                         blog/BlogService                               │
├────────────────────────────────────────────────────────────────────────┤
│                     数据访问层（读写分离）                                 │
│  ctx.msq.collection('blogs').findPage/findOne — 查询+缓存              │
│  ctx.model.Blog — 写操作 + invalidateCache()                          │
├────────────────────────────────────────────────────────────────────────┤
│                           MongoDB: blogs 集合                          │
└────────────────────────────────────────────────────────────────────────┘
```

### 1.2 模块划分

| 模块 | 文件路径 | 职责 |
|------|---------|------|
| Model | `app/model/blog.ts` | blogs 集合 Mongoose Schema 定义 + 索引 |
| Service | `app/service/blog/BlogService.ts` | 博客 CRUD 业务逻辑、权限校验、缓存查询 |
| Controller | `app/controller/open/BlogController.ts` | 公开接口（列表、详情）+ 内联参数校验 |
| Controller | `app/controller/home/BlogController.ts` | 用户端接口（创建、更新、删除）+ 内联参数校验 |
| Controller | `app/controller/home/BlogAdminController.ts` | 管理端接口（全量列表、上下架）+ 内联参数校验 |
| Route | `app/routes/open/blog.ts` | 公开路由注册 |
| Route | `app/routes/home/blog.ts` | 用户端 + 管理端路由注册 |
| DSL | `config/dsl/blog/{zh-CN,en-US,zh-HK}.ts` | 错误码多语言注册（子目录结构） |

### 1.3 核心数据流

**公开列表请求（GET /open/blog — 游标分页）**:
```
前端请求（?limit=20&after=xxx&keyword=旅行）
  → open/BlogController.index()
      → ctx.dsl() + ctx.validateDsl(schema, 'query')（内联校验）
      → BlogService.getBlogList(params)
          → ctx.msq.collection('blogs').findPage({
              query: filter,
              sort: { published_at: -1 },
              limit, after,
              totals: { mode: 'async' },
              cache: 60000
            })
          → 返回 { items, pageInfo, totals }
      → ctx.success({ items, pageInfo, totals })
```

**创建文章请求（POST /home/blog）**:
```
前端请求（携带 JWT）
  → [userJwt, singleLogin] 中间件（JWT 解析 + 单点登录校验）
  → home/BlogController.create()
      → ctx.dsl() + ctx.validateDsl(schema, 'body')（内联校验）
      → BlogService.createBlog(form, userId)
          → ctx.model.Blog.create(doc)
          → ctx.msq.collection('blogs').invalidateCache()
      → ctx.success(result)
```

**详情请求（GET /open/blog/:blogId）**:
```
前端请求
  → open/BlogController.detail()
      → ctx.dsl() + ctx.validateDsl(schema, 'params')（内联校验）
      → BlogService.getBlogDetail(blogId, userId?)
          → ctx.msq.collection('blogs').findOne(filter, { cache: 120000 })
          → 异步 fire-and-forget：ctx.model.Blog.updateOne({ $inc: { view_count: 1 } })
      → ctx.success(result)
```

**管理端列表（GET /home/blog/admin/list — 游标分页）**:
```
前端请求（携带 dbToken）
  → dbToken 中间件
  → home/BlogAdminController.index()
      → ctx.dsl() + ctx.validateDsl(schema, 'query')（内联校验）
      → BlogService.adminGetBlogList(params)
          → ctx.msq.collection('blogs').findPage({
              query: filter,
              sort: { created_at: -1 },
              limit, after,
              totals: { mode: 'sync' }
              // 不启用缓存
            })
      → ctx.success({ items, pageInfo, totals })
```

## 二、数据模型设计

### 2.1 blogs 集合（新建）

**文件**: `app/model/blog.ts`

| 字段 | 类型 | 必填 | 默认值 | 说明 |
|------|------|:----:|--------|------|
| `_id` | ObjectId | — | 自动 | 主键 |
| `lang` | String | — | `'en'` | 语言标识：`zh`=简体中文 / `en`=英文 / `hk`=繁体中文。同一文章的不同语言版本为独立记录，查询时按请求头 `language` 过滤。与 `feature_modules`/`trip_destination` 模式一致 |
| `title` | String | ✅ | — | 文章标题（1-200 字符） |
| `content` | String | ✅ | — | 文章正文（无长度限制） |
| `summary` | String | — | `''` | 文章摘要（最多 500 字符，不填则自动截取正文前 200 字符） |
| `cover_image` | String | — | `''` | 封面图 URL |
| `tags` | [String] | — | `[]` | 标签数组（最多 10 个，每个 1-20 字符） |
| `status` | String | ✅ | `'draft'` | 文章状态：`draft` / `published` |
| `author_id` | ObjectId | ✅ | — | 作者用户 ID |
| `view_count` | Number | — | `0` | 浏览量（原子自增，异步更新；各语言版本独立计数） |
| `del_flag` | Number | — | `0` | 软删除标记：`0`=正常，`1`=已删除 |
| `published_at` | Date | — | — | 首次发布时间 |
| `deleted_at` | Date | — | — | 软删除时间 |
| `created_at` | Date | — | `Date.now` | 创建时间 |
| `updated_at` | Date | — | `Date.now` | 最后更新时间 |

> **多语言说明**：同一篇文章的中/英/繁体版本在 `blogs` 集合中各为一条独立记录，通过 `lang` 字段区分。
> 三个版本的 `status`、`del_flag`、`view_count` 互相独立，各语言版本可单独上下架。
> 创建文章时 `lang` 由 Service 层从 `ctx.i18n.getLanguage()` 自动读取（即请求头 `language: zh/en/hk`），无需前端显式传入。

**TypeScript 接口签名**（仅作类型参考）:

```typescript
interface BlogDocument {
  _id: Types.ObjectId;
  title: string;
  content: string;
  summary: string;
  cover_image: string;
  tags: string[];
  status: 'draft' | 'published';
  author_id: Types.ObjectId;
  view_count: number;
  del_flag: 0 | 1;
  published_at?: Date;
  deleted_at?: Date;
  created_at: Date;
  updated_at: Date;
}
```

### 2.2 索引设计

| 索引名 | 字段组合 | 类型 | 用途 |
|--------|---------|:----:|------|
| `idx_lang_status_del_published` | `{ lang: 1, status: 1, del_flag: 1, published_at: -1 }` | 普通复合 | 公开列表主查询（`lang` 置首位，覆盖语言过滤 + 状态过滤 + 排序）+ findPage 游标分页 |
| `idx_author_del_created` | `{ author_id: 1, del_flag: 1, created_at: -1 }` | 普通复合 | 按作者查询（权限校验 + 作者筛选） |
| `idx_tags_lang_status_del` | `{ tags: 1, lang: 1, status: 1, del_flag: 1 }` | 普通复合 | 按标签过滤查询（加 `lang` 覆盖语言维度） |

> **索引与 findPage 的关系**：monSQLize findPage 游标分页会在排序字段末尾自动追加 `_id` 确保稳定排序，
> 复合索引的排序字段应与 findPage 的 `sort` 参数一致（如 `published_at: -1`），`_id` 无需显式加入索引。
>
> **lang 置首位的原因**：公开列表查询始终携带 `lang` 过滤条件（精确匹配），将其放在索引前缀位置可最大化利用索引选择性，减少扫描范围。

### 3.1 POST /home/blog — 创建文章

**鉴权**: `userAuth({ level: 'basic' })`（`ctx.state.user._id` 为作者）

**请求体**:

| 参数 | 类型 | 必填 | DSL 校验规则 | 说明 |
|------|------|:----:|------------|------|
| `title` | string | ✅ | `string:1-200!` | 文章标题（对应当前语言版本） |

### 3.2 GET /open/blog — 文章列表（游标分页）

**鉴权**: 无（`open` 分组）

**请求参数（query）**:

| 参数 | 类型 | 必填 | DSL 校验规则 | 说明 |
|------|------|:----:|------------|------|
| `limit` | int | — | `number:1-50` | 每页条数，默认 20 |
| `after` | string | — | `string` | 游标值，获取该游标之后的数据（下一页） |
| `before` | string | — | `string` | 游标值，获取该游标之前的数据（上一页） |
| `keyword` | string | — | `string:0-100` | 关键词（同时匹配 title + summary，正则转义防注入） |
| `tag` | string | — | `string:0-20` | 标签精确过滤 |
| `author_id` | string | — | `objectId` | 按作者 ID 过滤 |

> **互斥规则**：`after` 和 `before` 不能同时传入（monSQLize 会抛出 `VALIDATION_ERROR`）。
> 首次请求不传 `after`/`before`，获取第一页数据。

**固定过滤条件**: `{ lang: <请求头language>, del_flag: 0, status: 'published' }`

> **语言过滤**：Service 层从 `ctx.i18n.getLanguage()` 自动读取语言，无需前端在 query 中传 `lang`。
> 请求头 `language: zh` → 返回简体中文文章；`language: en`（默认）→ 返回英文文章；`language: hk` → 返回繁体中文文章。

**排序**: `{ published_at: -1 }`（monSQLize 自动追加 `_id` 确保稳定排序）

**缓存策略**: `cache: 60000`（60 秒 TTL）

**响应（200）**:
```json
{
  "code": 0,
  "data": {
    "items": [
      {
        "_id": "65f1a2b3c4d5e6f7a8b9c0d1",
        "title": "我的第一篇博客",
        "summary": "这是摘要...",
        "cover_image": "https://example.com/cover.jpg",
        "tags": ["旅行", "攻略"],
        "status": "published",
        "author_id": "65f000000000000000000001",
        "view_count": 128,
        "published_at": "2026-03-05T11:00:00.000Z",
        "created_at": "2026-03-05T10:00:00.000Z"
      }
    ],
    "pageInfo": {
      "hasNext": true,
      "hasPrev": false,
      "startCursor": "eyJzIjp7InB1Ymxpc2hlZF9hdCI6LTEsIl9pZCI6MX0sImEiOnt9fQ==",
      "endCursor": "eyJzIjp7InB1Ymxpc2hlZF9hdCI6LTEsIl9pZCI6MX0sImEiOnt9fQ=="
    },
    "totals": {
      "mode": "async",
      "total": 100,
      "totalPages": 5,
      "ts": 1709625600000
    }
  }
}
```

> **响应格式说明**：
> - `items`：当页数据列表（列表接口不返回 `content` 字段，通过 findPage 的 `pipeline` 中 `$project` 排除）
> - `pageInfo.hasNext` / `pageInfo.hasPrev`：是否有下一页/上一页
> - `pageInfo.startCursor` / `pageInfo.endCursor`：游标值，用于前端翻页请求
> - `totals`：总数统计（async 模式，首次返回可能为 null，后续请求返回缓存结果）
> - 前端通过 `pageInfo.hasNext` 判断是否显示"加载更多"，通过 `endCursor` 作为下次请求的 `after` 参数

**错误码**:

| 错误码 Key | code | 触发条件 |
|-----------|:----:|---------|
| `BLOG_LIST_FAILED` | 50304 | 查询异常（catch 块） |

### 3.3 GET /open/blog/:blogId — 文章详情

**鉴权**: 无（`open` 分组）

**路径参数**:

| 参数 | 类型 | 必填 | DSL 校验规则 |
|------|------|:----:|------------|
| `blogId` | ObjectId | ✅ | `objectId!` |

**访问规则**:
- 默认只可访问 `lang=<请求语言>` + `status=published` + `del_flag=0` 的文章
- 例外：`/home/blog/:blogId` 路径由已登录用户调用，且 `author_id === ctx.state.user._id`，可访问自己的草稿（通过 home 路由实现，而非 open 路由，详见§七）
- 每次访问后异步对 `view_count` 原子 `+1`（不阻塞响应，不触发缓存失效）

## 四、模块设计

### 4.1 BlogService（`app/service/blog/BlogService.ts`）

> **多语言处理约定（v2.3 新增）**：
> - Service 内部通过私有方法 `_getLang()` 读取 `ctx.i18n.getLanguage()`（由 `language` 中间件注入），返回 `'zh'` / `'en'` / `'hk'`，默认 `'en'`
> - `createBlog`：存入 `lang = this._getLang()`，无需前端传入
> - `getBlogList` / `getBlogDetail`：查询时自动追加 `lang` 过滤条件
> - `updateBlog` / `deleteBlog`：通过 `_id` 直接定位，不加 `lang` 过滤（`_id` 创建时已与 `lang` 绑定）
> - `adminGetBlogList`：`lang` 为可选过滤参数，不传则查全部语言

**核心方法**:

| 方法签名 | 数据访问 | 说明 |
|---------|:-------:|------|
| `createBlog(form, authorId)` | `ctx.model` | 创建文章，处理 summary 自动截取，tags 数量校验，invalidateCache |
| `getBlogList(params)` | `ctx.msq` findPage + cache | 查询已发布列表（游标分页），排除 content，缓存 60s |
| `getBlogDetail(blogId, userId?)` | `ctx.msq` findOne + cache | 查询文章详情（含 content），缓存 120s；异步更新 view_count |
| `updateBlog(blogId, form, userId)` | `ctx.model` | 更新文章，校验作者权限，处理 published_at 逻辑，invalidateCache |
| `deleteBlog(blogId, userId)` | `ctx.model` | 软删除，校验作者权限，幂等处理，invalidateCache |
| `adminGetBlogList(params)` | `ctx.msq` findPage (no cache) | 管理员全量查询（游标分页），不启用缓存 |
| `adminUpdateBlogStatus(blogId, status)` | `ctx.model` | 管理员强制修改状态，invalidateCache |
| `_buildListFilter(params)` | — | 内部辅助：构建查询 filter 对象，含 keyword 正则转义 |
| `_autoSummary(content)` | — | 内部辅助：截取 content 前 200 字符作为摘要 |

**关键实现要点**:

1. **keyword 正则转义**（防注入）:
   ```
   const escaped = keyword.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
   { $or: [{ title: new RegExp(escaped, 'i') }, { summary: new RegExp(escaped, 'i') }] }
   ```

2. **summary 自动截取**:
   ```
   form.summary = form.summary || form.content.slice(0, 200);
   ```

3. **view_count 异步更新**（不触发缓存失效）:
   ```
   // 先返回查询结果
   const blog = await ctx.msq.collection('blogs').findOne(filter, { cache: 120000 });
   // 异步更新，不 await，不调用 invalidateCache
   ctx.model.Blog.updateOne({ _id: blogId }, { $inc: { view_count: 1 } })
     .catch(err => ctx.logger.error('[BlogService] view_count 更新失败', { blogId, err }));
   return blog;
   ```

4. **作者权限校验**（统一在 Service 层）:
   ```
   if (doc.author_id.toString() !== userId.toString()) {
     ctx.dsl.error.throw('BLOG_FORBIDDEN');
   }
   ```

5. **tags 数量校验**（create 和 update 复用）:
   ```
   if (form.tags && form.tags.length > 10) {
     ctx.dsl.error.throw('BLOG_TAGS_LIMIT');
   }
   ```

6. **游标分页列表查询**（findPage 调用）:
   ```
   const result = await ctx.msq.collection('blogs').findPage({
     query: filter,
     sort: { published_at: -1 },
     limit: params.limit || 20,
     after: params.after,
     before: params.before,
     totals: { mode: 'async' },
     cache: 60000,
     pipeline: [
       { $project: { content: 0 } }  // 列表排除 content 字段
     ],
   });
   return result;  // 直接返回 { items, pageInfo, totals }
   ```

### 5.1 ctx.msq 实际调用方式

```typescript
// ✅ 正确 — 列表查询（游标分页，带缓存 60s，排除 content 字段）
const result = await ctx.msq.collection('blogs').findPage({
  query: { status: 'published', del_flag: 0, ...extraFilter },
  sort: { published_at: -1 },
  limit: params.limit || 20,
  after: params.after,       // 游标：下一页
  before: params.before,     // 游标：上一页
  totals: { mode: 'async' }, // 异步总数统计
  cache: 60000,              // 缓存 60s
  pipeline: [
    { $project: { content: 0 } }  // 排除 content 字段
  ],
});
// result = { items: [...], pageInfo: { hasNext, hasPrev, startCursor, endCursor }, totals: { total, totalPages, ... } }

// ✅ 正确 — 详情查询（带缓存 120s，返回全部字段）
const blog = await ctx.msq.collection('blogs').findOne(
  { _id: blogId, status: 'published', del_flag: 0 },
  { cache: 120000 }
);

// ✅ 正确 — 管理端列表（游标分页，不使用缓存）
const result = await ctx.msq.collection('blogs').findPage({
  query: filter,
  sort: { created_at: -1 },
  limit: params.limit || 20,
  after: params.after,
  before: params.before,
  totals: { mode: 'sync' },  // 管理端同步获取总数
  // 不设置 cache — 管理端需要实时数据
});

// ✅ 正确 — 手动失效集合缓存（写操作后调用）
await ctx.msq.collection('blogs').invalidateCache();
```

> ❌ **错误写法**（不存在此 API）：
> ```typescript
> ctx.msq('Blog').find(...)          // 错误：ctx.msq 是对象，不是函数
> ctx.msq.find(...)                  // 错误：缺少 .collection('blogs')
> ctx.msq.collection('blogs').find({ ... }, { skip, limit })  // 错误：列表分页应用 findPage
> ```

### 5.2 缓存 TTL 策略

| 接口 | TTL | 说明 |
|------|:---:|------|
| GET /open/blog（列表 findPage） | 60s | 公开列表允许 60s 内短暂不一致 |
| GET /open/blog/:blogId（详情 findOne） | 120s | 详情内容变化频率低，TTL 更长 |
| GET /home/blog/admin/list（管理列表 findPage） | 禁用 | 管理端需要实时数据 |

### 5.3 缓存失效策略

> ⚠️ **重要说明**：写操作使用 `ctx.model.Blog`（Mongoose），不会自动触发 monSQLize 缓存失效。需在 Service 层每次写操作完成后，手动调用 `ctx.msq.collection('blogs').invalidateCache()`（集合级失效）。

| 操作 | 失效时机 | 是否调用 invalidateCache |
|------|---------|:---------------------:|
| 创建文章（`createBlog`） | `ctx.model.Blog.create()` 成功后 | ✅ |
| 更新文章（`updateBlog`） | `ctx.model.Blog.updateOne()` 成功后 | ✅ |
| 软删除（`deleteBlog`） | `ctx.model.Blog.updateOne()` 成功后 | ✅ |
| 管理员上下架（`adminUpdateBlogStatus`） | `ctx.model.Blog.updateOne()` 成功后 | ✅ |
| view_count +1（异步） | 异步执行，**不调用** | ❌（允许 view_count 最终一致） |

> **findPage 缓存失效补充**：`invalidateCache()` 会同时失效 `find`、`findOne`、`findPage` 等所有 monSQLize 查询的缓存（键前缀 `fp:` / `f:` 均失效），无需单独处理。

## 六、鉴权与权限设计

### 6.1 userAuth 中间件说明

根据 `app/middleware/userAuth.ts` 源码，`userAuth` 支持的 level 仅有：

| level | 说明 |

### 6.2 各接口鉴权方式

### 6.3 文章级权限校验（Service 层强制）

| 操作 | 校验逻辑 | 失败响应 |
|------|---------|---------|
| 更新文章（`updateBlog`） | `doc.author_id.toString() !== userId.toString()` | `BLOG_FORBIDDEN`（40302） |

### 8.2 游标分页前端对接指南

**首次请求**：
```
GET /open/blog?limit=20
→ 不传 after/before，获取第一页
```

**加载下一页**：
```
GET /open/blog?limit=20&after={上次响应的 data.pageInfo.endCursor}
```

**加载上一页**：
```
GET /open/blog?limit=20&before={上次响应的 data.pageInfo.startCursor}
```

**判断是否还有更多**：
```
data.pageInfo.hasNext === false → 已到最后一页
data.pageInfo.hasPrev === false → 已到第一页
```

**响应统一格式**（列表接口）：
```json
{
  "code": 0,
  "data": {
    "items": [...],
    "pageInfo": {
      "hasNext": true,
      "hasPrev": false,
      "startCursor": "...",
      "endCursor": "..."
    },
    "totals": {
      "total": 100,
      "totalPages": 5
    }
  }
}
```

> **注意**：`totals` 使用 async 模式，首次请求 `total` 可能为 `null`（后台计算中），刷新后返回缓存值。
> 管理端使用 sync 模式，`total` 始终有值。

## 十、影响评估

### 10.1 风险评估

| 风险 | 级别 | 缓解措施 |
|------|:----:|---------|
| 路由冲突 `/home/blog/admin/*` vs `/home/blog/:blogId` | 中 | 管理端路由先于用户端注册 |
| view_count 缓存不一致 | 低 | 可接受（最终一致，缓存最多 120s 偏差） |
| findPage 游标失效 | 低 | 前端捕获 `CURSOR_INVALID` 错误后回退到首页重新查询 |
| DSL 子目录迁移未完成时的兼容性 | 低 | Blog DSL 已独立实现（spread merge 追加），不依赖行程广场 Phase 0 迁移 |
| 大数据量 totals 统计超时 | 低 | 公开列表使用 async 模式（不阻塞），管理端使用 sync + hint 索引 |

### 10.2 性能设计

| 设计 | 说明 |
|------|------|
| 游标分页 O(1) | findPage 基于排序字段 + 游标定位，性能不随数据量增长而下降 |
| 列表缓存 60s | monSQLize L1 内存缓存，相同 query + sort + limit + cursor 组合命中缓存 |
| 详情缓存 120s | 详情内容变化频率低，更长 TTL |
| view_count 异步 | 不阻塞响应，不触发缓存失效，避免缓存抖动 |
| 排除 content | 列表接口通过 pipeline `$project` 排除 content 字段，减少数据传输 |
| 复合索引覆盖 | 三个索引覆盖列表、作者、标签三种核心查询 |

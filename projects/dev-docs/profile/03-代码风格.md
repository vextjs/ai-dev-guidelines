## 📋 文档格式规范

### 缩进和格式

- **缩进**: 2 空格（Markdown 列表嵌套、YAML 文件）
- **编码**: UTF-8（所有文件）
- **换行**: LF（Unix 风格）
- **路径分隔符**: 统一使用 `/`（不使用 `\`）
- **表格**: 必须有表头行和分隔行
- **代码块**: 必须指定语言标识（如 ```yaml、```markdown）
- **链接**: 使用相对路径（不使用绝对路径）

### 语言规范

- **主体语言**: 中文
- **技术术语**: 可使用英文原文（如 Agent、SSOT、glob）
- **标点**: 中文文档使用中文标点，代码/配置中使用英文标点

## 📁 命名规范

### 目录命名

| 类型 | 规范 | 示例 |
|-----|------|------|
| 工作流目录 | `NN-kebab-case/` | `00-pre-check/`、`01-requirement-dev/` |
| 模板目录 | kebab-case/ | `lite/`、`core/`、`extended/` |

### 文件命名

| 类型 | 规范 | 示例 |
|-----|------|------|
| 入口/索引文件 | UPPER_CASE.md | `README.md`、`STATUS.md`、`CHANGELOG.md` |
| 内容文件 | kebab-case.md | `error-handling.md`、`task-memory.md` |
| 模板文件（核心） | xxx-template.md | `requirement-template.md` |
| 模板文件（精简） | xxx-lite.md | `technical-lite.md` |
| 配置文件 | kebab-case.yaml | `decision-tree.yaml` |
| 工具脚本 | kebab-case.js | `validate-links.js` |
| 输出报告 | YYYYMMDD-NN-描述.md | `20260227-01-analysis-xxx.md` |
| 记忆文件 | YYYYMMDD.md（每天一文件，v1.7） | `20260227.md` |
| Profile 模块 | NN-中文名.md | `01-项目信息.md`、`02-架构约束.md` |

## 📝 文档结构规范

### 标准文档结构

```markdown
# 标题

> **关键元数据**: 值  
> **版本**: vX.Y  
> **最后更新**: YYYY-MM-DD

---

## 📋 章节1

内容...

---

## 🎯 章节2

内容...

---

**最后更新**: YYYY-MM-DD
```

### 必须包含的元素

- ✅ 一级标题（`#`）作为文件标题
- ✅ 元数据块（版本、更新日期等）
- ✅ 分隔线（`---`）分隔主要章节
- ✅ Emoji 前缀标记章节类型（📋、🎯、⚠️、🔗 等）

### 表格规范

| 要求 | 说明 |
|-----|------|
| 必须有表头 | 第一行为列名 |
| 必须有分隔行 | `|---|---|` 格式 |
| 对齐方式 | 左对齐（默认）或居中（`:---:` 用于编号/状态列） |

---

## 🔤 YAML 风格（decision-tree.yaml 等）

- **缩进**: 2 空格
- **字符串**: 简单值不加引号，含特殊字符时使用双引号
- **数组**: 短数组使用流式 `[a, b, c]`，长数组使用块式 `- item`
- **注释**: `#` 开头，与代码间隔 1 空格
- **版本号**: 带引号 `version: "2.9.0"`

---

## 🔤 JavaScript 风格（tools/ 脚本）

- **缩进**: 2 空格
- **引号**: 单引号
- **分号**: 必须
- **命名**: camelCase（变量/函数）、UPPER_SNAKE_CASE（常量）

---

## 📐 严重程度/优先级标记

| 标记 | 含义 | 使用场景 |
|-----|------|---------|
| 🔴 | 严重 Bug / 强制规则 | 必须修复、禁止违反 |
| 🟡 | 警告 / 待办 | 需要关注但不阻塞 |
| 💡 | 建议 / 可选优化 | 改进建议 |
| ❌ | 失败 / 已废弃 | 验证未通过、已弃用 |
| ✅ | 通过 / 完成 | 验证通过、任务完成 |
| ⚠️ | 注意 / 有条件 | 需要特别注意的事项 |

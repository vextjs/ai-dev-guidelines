# 目录结构重构

> **项目**: ai-dev-guidelines (dev-docs)

> **优先级**: P1
> 
> **关联**: 本文档 → [02-技术方案](./02-技术方案.md) → [03-实施方案/](./03-实施方案/README.md)

---

## 一、需求背景

ai-dev-guidelines 当前有 9 个平级子目录（workflows/、standards/、templates/、tools/、spec-self-fix/、best-practices/、examples/、docs/、changelogs/），外加 projects/。目录层级过于扁平，缺乏语义分组，AI 和用户在浏览时难以快速理解各目录的职能归属关系。

需要将目录收拢为以受众和职能为核心的分层结构：
- `core/` — AI 执行内核（规则、流程、工具、模板、自修复）
- `docs/` — 文档区（用户指南、最佳实践、示例）
- `projects/` — 项目产出（不变）
- `changelogs/` — 版本变更（不变）

---

## 二、需求内容

### 功能需求

- [ ] F1: 创建 `core/` 目录，将 `workflows/`、`standards/`、`templates/`、`tools/` 移入
- [ ] F2: 将 `spec-self-fix/` 移入 `core/` 并重命名为 `self-fix/`
- [ ] F3: 将原 `docs/` 下所有内容移入 `docs/guides/`
- [ ] F4: 将 `best-practices/` 移入 `docs/`
- [ ] F5: 将 `examples/` 移入 `docs/`
- [ ] F6: 顶级文件（README.md、QUICK-REFERENCE.md、CONSTRAINTS.md、ONBOARDING.md、STATUS.md、META.yaml、CHANGELOG.md）位置不变
- [ ] F7: `projects/` 和 `changelogs/` 位置不变
- [ ] F8: 所有内部路径引用同步更新（P0 入口文件 + P1 内部文件）
- [ ] F9: 工具脚本（JS 文件）中的路径常量更新
- [ ] F10: `validate-structure.js` 的 EXPECTED_STRUCTURE 更新

### 非功能需求

- 兼容性要求: 历史文件（changelogs/、reports/、.ai-memory/）中的旧路径不修改，保持历史快照完整性
- 可回滚: 所有变更通过 git 管理，可 `git reset --hard` 回退

# 04 - 错误处理

> **模块**: 错误处理规范  
> **版本**: v1.0.0  
> **最后更新**: 2026-02-24  
> **视角**: 本文档使用"你"指代 AI 助手（规范执行者）

---

## 📍 错误系统位置

- **错误码定义 + 工厂函数**: `lib/errors.js`
- **锁专属错误类**: `lib/lock/errors.js`（例外，见下方）
- **错误对象结构**: `{ message, code, details?, cause? }`

---

## 🚫 强制规则

1. ❌ **禁止** 直接 `throw new Error('...')` 不带 `code`
2. ❌ **禁止** 在公开 API 中透传 MongoDB 原生错误（必须包装）
3. ❌ **禁止** 硬编码错误码字符串（如 `error.code = 'VALIDATION_ERROR'`）
4. ✅ **必须** 使用 `lib/errors.js` 的工厂函数创建错误
5. ✅ **必须** 在 `ErrorCodes` 对象中注册新错误码

---

## 🛠️ 错误工厂函数

### `createError(code, message, details?, cause?)`

通用错误创建：

```javascript
const { createError, ErrorCodes } = require('../errors');

throw createError(
    ErrorCodes.INVALID_ARGUMENT,
    'maxTimeMS 必须在 1 ~ 300000 之间',
    [{ field: 'maxTimeMS', value: -1 }]  // details 可选
);
```

### `createValidationError(messageOrDetails, details?)`

Schema 或参数验证失败时使用：

```javascript
// 新用法（推荐）
throw createValidationError('用户名不能为空');

// 带详细信息（兼容旧用法：第一个参数也可以是 details 数组）
throw createValidationError('参数校验失败', [
    { field: 'username', message: '不能为空' }
]);
```

### 完整快捷工厂列表（`lib/errors.js` 导出）

| 函数 | 签名 | 适用场景 |
|------|------|---------|
| `createValidationError` | `(messageOrDetails, details?)` | 参数/Schema 校验失败 |
| `createCursorError` | `(message, details?)` | 游标/分页错误 |
| `createConnectionError` | `(message, cause?)` | 连接失败 |
| `createQueryTimeoutError` | `(timeoutMs, cause?)` | 查询超时（自动拼接 `"查询超时 (Xms)"` 消息） |
| `createWriteError` | `(operation, message, cause?)` | 写操作失败 |

---

## ⚠️ 例外：锁错误类

`lib/lock/errors.js` 中的 `LockAcquireError` 和 `LockTimeoutError` 是**独立的命名错误类**，不通过 `createError` 工厂创建。这是历史设计决策，保持现状：

```javascript
// EXCEPTION: 锁错误类是独立命名类，保持现状以避免破坏已有 instanceof 捕获逻辑
const { LockAcquireError, LockTimeoutError } = require('./lock/errors');
throw new LockAcquireError('获取锁失败：资源被占用');
```

调用方可通过 `instanceof` 或 `err.code === 'LOCK_ACQUIRE_FAILED'` 判断。  
**新增**的锁相关错误应继续沿用独立命名错误类模式，保持一致。

## 📋 错误码表

### 当前已定义的错误码

| 分类 | 错误码 | 说明 |
|------|--------|------|
| **验证** | `VALIDATION_ERROR` | 参数或 Schema 校验失败 |
| **验证** | `INVALID_ARGUMENT` | 非法参数值 |
| **验证** | `INVALID_COLLECTION_NAME` | 集合名称无效 |
| **验证** | `INVALID_DATABASE_NAME` | 数据库名称无效 |
| **游标** | `INVALID_CURSOR` | 游标无效 |
| **游标** | `CURSOR_SORT_MISMATCH` | 游标排序不一致 |
| **分页** | `JUMP_TOO_FAR` | 跳页过远 |
| **分页** | `STREAM_NO_JUMP` | 流式分页不支持跳页 |
| **分页** | `STREAM_NO_TOTALS` | 流式分页不支持总数 |
| **分页** | `STREAM_NO_EXPLAIN` | 流式分页不支持 explain |
| **连接** | `CONNECTION_TIMEOUT` | 连接超时 |
| **连接** | `CONNECTION_FAILED` | 连接失败 |
| **连接** | `CONNECTION_CLOSED` | 连接已关闭 |
| **数据库** | `DATABASE_ERROR` | 数据库操作失败 |
| **数据库** | `QUERY_TIMEOUT` | 查询超时 |
| **缓存** | `CACHE_ERROR` | 缓存操作失败 |
| **缓存** | `CACHE_TIMEOUT` | 缓存超时 |
| **配置** | `INVALID_CONFIG` | 配置无效 |
| **配置** | `UNSUPPORTED_DATABASE` | 不支持的数据库类型 |
| **写操作** | `WRITE_ERROR` | 写操作失败 |
| **写操作** | `DOCUMENT_REQUIRED` | 缺少文档参数 |
| **写操作** | `DOCUMENTS_REQUIRED` | 缺少文档数组参数 |
| **写操作** | `DUPLICATE_KEY` | 唯一键冲突 |
| **写操作** | `WRITE_CONFLICT` | 写冲突 |
| **锁**（独立类） | `LOCK_ACQUIRE_FAILED` | 锁获取失败（见例外说明） |
| **锁**（独立类） | `LOCK_TIMEOUT` | 锁超时（见例外说明） |

### 添加新错误码

1. 在 `lib/errors.js` 的 `ErrorCodes` 对象中添加新项
2. 在 `docs/error-codes.md` 中同步更新文档
3. 在对应分类下排列

## 🔄 包装 MongoDB 原生错误

```javascript
try {
    await collection.insertOne(doc);
} catch (err) {
    if (err.code === 11000) {
        // 唯一键冲突 - 转换为标准错误
        throw createError(ErrorCodes.DUPLICATE_KEY, '文档已存在', null, err);
    }
    // 其他数据库错误
    throw createError(ErrorCodes.DATABASE_ERROR, err.message, null, err);
}
```

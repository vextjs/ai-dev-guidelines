## 🚫 架构约束（强制）

### 1. 禁止破坏 API 向后兼容性

monSQLize 是 npm 公开包，**所有公开 API 变更必须向后兼容**。

- ❌ 禁止删除或重命名已发布的公开方法
- ❌ 禁止修改已有方法的参数顺序或必填性
- ❌ 禁止修改已有方法的返回值结构（除非版本升级且有 migration guide）
- ✅ 新增参数时必须设置合理的默认值
- ✅ 废弃 API 应先标记 `@deprecated` 并保留至少一个版本

### 2. 禁止将可选功能变为强制依赖

- ❌ 禁止在核心路径中直接 `require('ioredis')`（Redis 是可选依赖）
- ✅ 可选依赖必须使用 `try/catch` 懒加载，并给出友好安装提示

```javascript
// ✅ 正确：可选依赖懒加载模式
let ioredis;
try {
    ioredis = require('ioredis');
} catch (e) {
    // ioredis 未安装，Redis 缓存不可用
}
```

> 注：`schema-dsl` 虽然是 Model 层专用，但它已列为 `dependencies`（非 `optionalDependencies`），会随包安装，无需懒加载。

### 3. 禁止绕过统一错误系统

- ❌ 禁止直接 `throw new Error('...')`（无错误码）
- ❌ 禁止在公开 API 中暴露原始 MongoDB 驱动错误（必须包装）
- ✅ 必须使用 `lib/errors.js` 中的 `createError(code, message)` 工厂函数
- ✅ 错误码必须在 `ErrorCodes` 中定义，禁止硬编码字符串

```javascript
// ❌ 错误
throw new Error('集合名称无效');

// ✅ 正确
const { createError, ErrorCodes } = require('./errors');
throw createError(ErrorCodes.INVALID_COLLECTION_NAME, '集合名称无效');
```

> ⚠️ **例外**：`lib/lock/errors.js` 的 `LockAcquireError` / `LockTimeoutError` 是历史遗留的独立命名错误类，保持现状。新增的锁错误沿用该模式，其他模块不得效仿。

### 4. 禁止在 `lib/mongodb/` 以外的代码中直接操作 MongoDB Collection

- ❌ 禁止在 `lib/model/`、`lib/saga/` 等模块中直接调用 MongoDB 原生方法（`collection.find()` 等）
- ✅ 必须通过 `lib/mongodb/` 层提供的封装方法（或传入的 collection 封装对象）操作数据库

### 5. 禁止在核心模块中直接使用 `console.log` 输出

- ❌ 禁止在 `lib/` 目录下使用 `console.log` / `console.warn` / `console.error` 直接打印
- ✅ 必须使用 `lib/logger.js` 的 Logger 实例（通过 `this.logger` 访问）
- ⚠️ 例外：模块安装提示（`require` 失败时的 `console.error` 安装说明允许保留）

### 6. 禁止硬编码连接信息

- ❌ 禁止在代码中硬编码 MongoDB 连接字符串、IP、端口、用户名密码
- ✅ 所有连接信息通过构造函数 `config` 参数传入

### 7. 禁止修改 MongoDB 原生方法的返回值格式

monSQLize 承诺"100% API 兼容"，原生方法的返回结构不可改变：

- ❌ 禁止在 `find()`、`findOne()` 等原生方法的包装中修改返回格式
- ✅ 新增的便利方法（如 `findPage()`）可以定义自己的返回格式

### 8. 禁止在写操作中忽略缓存失效

- ❌ 新增写操作方法时，禁止遗漏缓存失效逻辑
- ✅ 所有写操作（insert/update/delete/upsert/replace）完成后，必须触发对应集合的缓存失效
- ✅ 支持 `autoInvalidate` 配置的写操作，还需触发精准失效逻辑

详见 [06-缓存规范.md](./06-缓存规范.md)。

### 9. 禁止破坏 CommonJS / ESM 双模块支持

- ❌ 禁止在 `lib/` 核心代码中使用 ESM 语法（`import`/`export`）
- ❌ 禁止删除或修改 `index.mjs`（ESM 入口）
- ✅ `lib/` 全部使用 CommonJS，`index.mjs` 负责 ESM 适配

### 10. 禁止降低测试覆盖率门槛

- ❌ 禁止在 `package.json` 的 `nyc` 配置中降低覆盖率阈值（当前：lines/statements/functions ≥ 70%，branches ≥ 65%）
- ✅ 新功能必须配套单元测试，目标覆盖率 ≥ 90%

### 11. 新功能必须加入验证清单

- ❌ 禁止新增功能（代码 / 文档 / 测试 / 类型）后不更新验证清单
- ✅ 新增功能时，**必须**在验证清单（`dev-docs/projects/monSQLize/requirements/REQ-001-全功能验证/01-requirement.md`）中添加对应条目
- ✅ 条目必须包含：编号、功能名、代码文件、文档、测试文件、验证文档名、状态
- ✅ 同步更新统计总览中的功能总数

> **目的**：确保验证清单始终覆盖 100% 的项目功能，不因新增功能而出现验证盲区。

所有例外必须在代码中添加注释：

```javascript
// EXCEPTION: [原因说明]
```

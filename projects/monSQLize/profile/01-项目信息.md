## 🧩 核心模块说明

### 主类（`lib/index.js`）

`MonSQLize` 是唯一的外部入口类：
- 构造函数接收完整配置（连接、缓存、日志、连接池等）
- `.connect()` 建立 MongoDB 连接，返回 `{ collection: Function, db: Function }`
- `.collection(name)` 通过 `connect()` 返回的 `collection` 函数获取集合操作对象
- `.model(name)` 通过 Model 层获取增强后的集合对象（需先 `Model.define`）

**构造函数重要配置项**（除基础的 `type`/`config`/`cache`/`logger` 外）：

| 配置项 | 说明 |
|-------|------|
| `maxTimeMS` | 全局查询超时（默认 2000ms） |
| `findLimit` | find 默认返回条数上限（默认 10） |
| `findPageMaxLimit` | 分页最大 limit（默认 500） |
| `slowQueryMs` | 慢查询阈值（默认 500ms，-1 禁用） |
| `slowQueryLog` | 慢查询持久化存储配置（v1.3.1+） |
| `autoConvertObjectId` | 自动 ObjectId 转换配置 |
| `pools` | 多连接池配置数组（v1.0.8+） |
| `poolStrategy` | 连接池选择策略（`auto` / `round-robin` 等） |
| `models` | Model 自动加载配置（v1.4.0+） |
| `sync` | Change Stream 同步配置（v1.0.8+） |
| `namespace` | 命名空间（缓存隔离用，默认 `{ scope: 'database' }`） |
| `cursorSecret` | 深分页游标加密密钥 |
| `countQueue` | Count 高并发队列配置 |

### 缓存架构

| 层级 | 实现 | 说明 |
|------|------|------|
| L1 | 内存缓存（LRU） | `cache.js`，进程级，最快 |
| L2 | Redis（ioredis） | `redis-cache-adapter.js`，跨进程共享，可选依赖 |
| 精准失效 | `cache-invalidation.js` | v1.1.6 新增，只清除受写操作影响的缓存键 |
| 分布式失效 | `distributed-cache-invalidator.js` | 多实例部署时同步失效 |

### 表达式系统（`lib/expression/`）

- 支持 122 个 MongoDB 操作符（v1.1.0 达到 100%）
- 允许用类 SQL 语法编写 MongoDB 聚合查询

### Saga 分布式事务（`lib/saga/`）

- `SagaOrchestrator` 负责 Saga 定义注册与执行触发
- `SagaExecutor` 负责顺序执行步骤 + 失败时触发补偿回滚
- `SagaContext` 持有每次执行的上下文数据

### Model 层（`lib/model/`）

- 基于 `schema-dsl` 进行 Schema 定义与验证
- 支持 Hooks（before/after）、自定义方法、关系（Populate / hasOne / hasMany / belongsTo）
- `schema-dsl` 是 `dependencies`（非 `optionalDependencies`），会随包一起安装
- 不调用 Model API 时不影响性能，但不能从 package 中卸除

### 锁系统（`lib/lock/`）

- 基于 `async-lock` 实现本地锁
- 专属错误类：`LockAcquireError` / `LockTimeoutError`（`lib/lock/errors.js`）
- 这两个错误类**不走** `createError` 工厂，是独立命名的错误类（见 [04-错误处理.md](./04-错误处理.md) 例外说明）

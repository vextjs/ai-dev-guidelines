## 🏗️ 缓存架构

```
查询请求
    ↓
[L1 内存缓存 (LRU)]  ← lib/cache.js
    ↓ miss
[L2 Redis 缓存]      ← lib/redis-cache-adapter.js（可选）
    ↓ miss
[MongoDB 数据库]
    ↑
写操作 → 触发缓存失效
         ├── 集合级失效（默认）
         └── 精准失效（autoInvalidate: true）
```

## ⚙️ 配置约定

### 构造函数级别配置（实例全局）

```javascript
const { createRedisCacheAdapter } = require('monsqlize/lib/redis-cache-adapter');

// 仅 L1 内存缓存（默认）
const msq = new MonSQLize({
    type: 'mongodb',
    config: { ... },
    cache: {
        maxSize: 100000,    // 最大缓存条目数（默认 100000）
        maxMemory: 0,       // 最大内存字节数，0 = 无限制

        // 精准缓存失效（v1.1.6+，默认 false）
        autoInvalidate: false,

        // 分布式缓存失效（多实例部署）
        distributed: {
            enabled: false
        }
    }
});

// L1 + L2 多级缓存（启用 Redis）
const msq = new MonSQLize({
    type: 'mongodb',
    config: { ... },
    cache: {
        multiLevel: true,
        remote: createRedisCacheAdapter('redis://localhost:6379/0'),  // L2 Redis
        // local 配置（可选，同单级缓存配置）
        local: { maxSize: 10000 },
        policy: { writePolicy: 'both' }  // 'both' | 'local-first-async-remote'
    }
});
```

### 查询级别配置（优先于实例配置）

```javascript
// cache 为毫秒数 = 启用缓存，TTL 为该值
await db.collection('users').find(
    { status: 'active' },
    {
        cache: 60000,           // 缓存 60 秒
        autoInvalidate: true,   // 覆盖实例级别配置
    }
);

// cache: false = 禁用此次查询的缓存
await db.collection('users').find(query, { cache: false });
```

## 🔄 缓存失效规范

### 写操作必须触发缓存失效

**所有写操作完成后，必须调用缓存失效逻辑**：

| 方法 | 必须失效 | 支持精准失效 |
|------|---------|------------|
| `insertOne` | ✅ | ✅ |
| `insertMany` | ✅ | ✅ |
| `updateOne` | ✅ | ✅ |
| `updateMany` | ✅ | ✅ |
| `deleteOne` | ✅ | ✅ |
| `deleteMany` | ✅ | ✅ |
| `upsertOne` | ✅ | ✅ |
| `replaceOne` | ✅ | ✅ |
| `incrementOne` | ✅ | ✅ |
| `findOneAndUpdate` | ✅ | ⚠️ 部分支持 |
| `findOneAndDelete` | ✅ | ⚠️ 部分支持 |
| `findOneAndReplace` | ✅ | ⚠️ 部分支持 |
| `insertBatch` | ✅ | - |
| `updateBatch` | ✅ | - |
| `deleteBatch` | ✅ | - |

> ⚠️ **新增写操作方法时**，必须参考 `lib/mongodb/writes/` 中现有实现，确保包含缓存失效调用。

### 精准缓存失效（v1.1.6+）

精准失效只清除"受写操作影响"的缓存键，而非整个集合的缓存。

**支持的操作符**：等值、`$in`、`$gt/$gte/$lt/$lte`、`$ne`、`$exists`

**自动跳过**（降级为集合级失效）：`$or`、`$nor`、`$expr`、`$where`、`$text`、`$regex` 等复杂查询

**实现位置**：`lib/cache-invalidation.js`（精准失效与集合级失效均在此文件，`lib/cache-invalidation/` 目录当前为空）

---

## 🚫 缓存相关禁止项

1. ❌ **禁止**新增写操作方法时忽略缓存失效（见架构约束第 8 条）
2. ❌ **禁止**在缓存键生成中直接使用 ObjectId 对象（必须序列化为字符串）
3. ❌ **禁止**修改缓存键生成规则（会破坏现有缓存的一致性）
4. ❌ **禁止**在 L2 Redis 不可用时导致查询失败（Redis 失败应降级到 L1）

---

## 🔑 缓存键约定

- 缓存键由 `lib/mongodb/common/accessor-helpers.js` 的 `mongoKeyBuilder` 生成
- ObjectId 字段在生成缓存键时**统一序列化为字符串**（v1.1.6 保证）
- 命名空间（`namespace.instanceId`）用于多实例缓存隔离

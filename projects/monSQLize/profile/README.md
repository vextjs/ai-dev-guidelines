## 🚨 AI 助手规范遵循声明（强制）

### 强制要求

**当你收到涉及 monSQLize 项目的任何开发任务时，必须严格遵守以下规则**：

1. **🔴 优先级**: 本 Profile 规范 > 通用规范 > 原有项目代码模式

2. **🔴 禁止行为**:
   - ❌ **禁止模仿原有项目代码中的旧模式**：如有不符合规范的现存代码，不得模仿
   - ❌ **禁止"保持一致性"**：不得以"与现有代码风格保持一致"为由违反规范
   - ❌ **禁止跳过验证**：不得跳过架构约束检查（尤其是 [02-架构约束.md](./02-架构约束.md)）
   - ❌ **禁止随意添加依赖**：新增核心依赖必须评估兼容性（Node 16+、mongodb 4.x/5.x/6.x/7.x）

3. **✅ 正确行为**:
   - ✅ **遵循新规范**：所有新代码必须符合本 Profile 规范
   - ✅ **重构旧代码**：修改旧代码时，修改部分必须按新规范重构
   - ✅ **遵循架构约束**：严格遵守 [02-架构约束.md](./02-架构约束.md) 的禁止项
   - ✅ **保持 API 兼容**：公开 API 变更前必须评估向后兼容性

### 新旧代码区分

| 场景 | 处理方式 |
|------|---------|
| 新增代码 | 🔴 必须完全遵循本规范 |
| 修改旧代码 | 🔴 修改部分必须按新规范重构 |
| 仅阅读旧代码 | ⚠️ 不要被旧模式误导 |

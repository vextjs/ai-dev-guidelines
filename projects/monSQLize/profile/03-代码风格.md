## 📋 基础格式

### 缩进和格式

| 规则 | 值 |
|------|---|
| 缩进 | 4 空格（项目约定，ESLint `indent` 规则当前为 `off`，靠团队约定维护） |
| 引号 | 单引号（`'`），字符串内含单引号时允许双引号 |
| 分号 | **必须**（`semi: error`） |
| 模块系统 | CommonJS（`lib/` 目录全部使用 `require/module.exports`） |
| 语法版本 | ES2021（`ecmaVersion: 2021`） |
| 行尾空格 | 禁止 |

### 变量声明

- ✅ 优先使用 `const`（`prefer-const: warn`）
- ✅ 禁止使用 `var`（`no-var: warn`）
- ✅ 优先使用对象简写（`object-shorthand: warn`）

## 📝 命名规范

| 类型 | 规范 | 示例 |
|-----|------|------|
| 类名 | PascalCase | `ConnectionPoolManager`, `SagaOrchestrator` |
| 文件名（类文件） | PascalCase | `ConnectionPoolManager.js`, `SagaExecutor.js` |
| 文件名（功能模块） | kebab-case | `cache-invalidation.js`, `redis-cache-adapter.js` |
| 函数/方法名 | camelCase | `createError`, `findOneById` |
| 私有属性/方法 | 下划线前缀 | `this._poolManager`, `this._cacheConfig` |
| 常量（模块级） | UPPER_SNAKE_CASE | `ErrorCodes.VALIDATION_ERROR` |
| 工厂函数 | `create` 前缀 | `createFindOps`, `createInsertOneOps` |
| 操作类方法（返回 ops 对象） | `create` + 功能 + `Ops` | `createFindPageOps`, `createUpdateOneOps` |

## 📚 JSDoc 规范

所有公开方法（`lib/mongodb/` 的写操作、查询操作）和主类的公开方法**必须**编写 JSDoc 注释。

### 必填项

```javascript
/**
 * [方法简短描述]
 * @param {类型} 参数名 - 参数描述
 * @param {Object} [可选参数] - 可选参数描述
 * @returns {Promise<类型>} 返回值描述
 * @throws {Error} 抛出条件描述（有错误时必填）
 * @example
 * // 用法示例（推荐但非强制）
 */
```

### 版本注解

新增功能需标注版本：

```javascript
/**
 * @since 1.1.6
 */
```

## 🔤 注释风格

- 使用**中文**注释（项目现有风格，面向中文团队）
- 行内注释使用 `//`，块注释使用 `/* */` 或 JSDoc `/** */`
- 重要修复或变更注释格式：`// 🔧 修复：说明` / `// 🆕 v1.x.x: 说明`
- 临时性代码禁止使用 `// TODO` 超过一个版本周期

---

## 📂 文件组织原则

- 每个模块负责单一职责（Single Responsibility）
- 工厂函数模式（`createXxxOps`）用于延迟绑定，避免循环依赖
- `lib/common/` 放纯工具函数（无副作用，不依赖 MongoDB 实例）
- `lib/utils/` 放通用工具函数（当前仅含 ObjectId 序列化工具，新工具按需添加）

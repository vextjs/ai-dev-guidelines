## 🧪 测试框架

| 工具 | 用途 |
|------|------|
| **自定义运行器** | `test/run-tests.js`（自实现 describe/it/before/after，通过 `node` 直接运行） |
| **assert** | Node.js 内置断言库（`require('assert')`） |
| **Sinon** | Mock / Stub / Spy |
| **内置内存 MongoDB** | 通过 `config: { useMemoryServer: true }` 启用（无需手动管理 MongoMemoryServer 生命周期） |
| **nyc** | 代码覆盖率 |
| **tsd** | TypeScript 类型定义测试 |

> ⚠️ **重要**：本项目**不使用** `mocha` 命令运行测试，也**不使用** `chai`。测试文件通过 `node test/run-tests.js` 执行，断言使用 Node.js 内置 `assert`。

---

## 📊 覆盖率要求

| 指标 | 最低要求 | 目标 |
|------|---------|------|
| Lines | **70%** | ≥ 90% |
| Statements | **70%** | ≥ 90% |
| Functions | **70%** | ≥ 90% |
| Branches | **65%** | ≥ 85% |

> ⚠️ **禁止降低** `package.json` 中 `nyc.check-coverage` 的阈值。

## 📐 测试规范

### 1. 文件命名

- 测试文件统一命名：`<功能名>.test.js`
- 放置位置：`test/unit/features/`（API 功能）或 `test/unit/infrastructure/`（底层模块）

### 2. 测试结构

```javascript
const assert = require('assert');        // ✅ 使用 Node.js 内置 assert
const MonSQLize = require('../../../lib');

describe('功能名 测试套件', function () {
    this.timeout(30000);  // 设置超时

    let msq;
    let collection;

    before(async function () {
        msq = new MonSQLize({
            type: 'mongodb',
            databaseName: 'test_xxx',
            config: { useMemoryServer: true }  // ✅ 内置内存 MongoDB，无需手动管理
        });
        const conn = await msq.connect();
        collection = conn.collection;
    });

    after(async function () {
        await msq.close();
    });

    describe('方法名', () => {
        it('应该 [正常行为描述]', async () => {
            const result = await collection('users').findOne({ name: 'test' });
            assert.strictEqual(result.name, 'test');       // ✅ assert.strictEqual
            assert.ok(result._id);                         // ✅ assert.ok
        });

        it('当 [异常条件] 时，应该抛出 [错误码]', async () => {
            await assert.rejects(                          // ✅ assert.rejects
                () => someMethod(invalidParam),
                (err) => {
                    assert.strictEqual(err.code, 'INVALID_ARGUMENT');
                    return true;
                }
            );
        });
    });
});
```

### 3. 测试 IT 描述规范

- ✅ 使用中文描述（项目现有风格）
- ✅ 格式：`应该 [做什么]` 或 `当 [条件] 时，应该 [结果]`
- ✅ 每个 `it` 只测试一个行为

### 4. 内存 MongoDB 使用（内置方式）

```javascript
// ✅ 正确：通过 config.useMemoryServer 内置启用（单例共享，~3s 首次启动）
msq = new MonSQLize({
    type: 'mongodb',
    databaseName: 'test_xxx',
    config: { useMemoryServer: true }
});
await msq.connect();  // 自动使用内存 MongoDB

// ❌ 错误：不要手动 new MongoMemoryServer()，项目已内置集成
```

> **性能说明**：`useMemoryServer` 使用单例模式，所有测试共享同一内存服务器实例，首次启动约 3 秒，后续 < 1ms。

### 5. 新功能测试要求

- **每个新功能必须配套单元测试**
- **Bug 修复必须包含回归测试**（证明 Bug 已修复，且不再复现）
- **公开 API 变更必须更新类型定义测试（tsd）**

### 6. 禁止在测试中硬编码真实数据库连接信息

- ❌ 禁止测试代码中含有真实的 MongoDB URI、IP、密码
- ✅ 使用 `config: { useMemoryServer: true }` 或环境变量

## 🚀 运行测试

```bash
# 运行全部测试
npm test
# 等价于: node test/run-tests.js

# 运行单个测试文件（传文件名关键字）
node test/run-tests.js find
node test/run-tests.js findPage

# 带覆盖率（生成 lcov + text 报告）
npm run test:coverage

# 生成 HTML 覆盖率报告
npm run coverage
# 报告在 coverage/index.html
```
